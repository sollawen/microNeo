# aibp-claude 分发 / 安装方式分析

> 目标：让用户**真正把 aibp-claude 安装进 Claude**（持久、可更新、免 `--plugin-dir` flag、免 clone 仓库），**不走 Claude 官方 plugin-market**。
>
> **结论：npm 包（与 aibp-pi / aibp-opencode 一致）+ 一个极小的 marketplace 目录文件。**
> npm 负责「交付与版本」（家族统一），marketplace 是 Claude 唯一的「安装入口」——两层缺一不可，原因见 §2。
>
> ✅ **已选定：变体 B**（marketplace 目录放独立小仓库 `sollawen/microNeo-plugins`，microNeo 根目录零侵入）。落地清单见 §7。

---

## 0. 现状速览（已具备的条件）

| 项 | 现状 | 说明 |
|---|---|---|
| 家族 npm 渠道 | `aibp-pi@1.0.4`、`aibp-opencode@1.0.4` 已发布 | 安装：`pi install npm:aibp-pi`；opencode 同理 |
| `aibp-claude` npm 名 | **已占位 `0.0.0`**（placeholder 包，无 files/bin） | 名字已 reserved，下次 `npm publish` 出 `1.0.0` 即可 |
| 包内容已 publish-ready | `package.json` 的 `files` 已含 `.claude-plugin/plugin.json`、`index.ts`、`package.json`、`README.md`；零依赖 | `npm publish` 直接发，无需 build |
| 插件可被 cache 复制 | monitor 命令 `bun "${CLAUDE_PLUGIN_ROOT}/index.ts"`；`index.ts` 用 `import.meta.url` 推 `__dirname` 读 `package.json` | 无写死源码路径，复制到 cache 照常跑 |
| 运行时依赖 | **用户机器需有 `bun`**（monitor 用它跑 `.ts`） | 与交付方式无关：npm / git / zip 都一样要 bun |

## 1. 关键概念澄清

### 1.1 「官方 plugin-market」≠「自建 marketplace」

| | Claude 官方 marketplace | 自建 marketplace（custom marketplace） |
|---|---|---|
| 是什么 | Anthropic 的 `claude-plugins-official` / `-community`，需审核 | **任何带 `.claude-plugin/marketplace.json` 的 git 仓库** |
| 谁控制 | Anthropic | 你自己（GitHub 仓库即目录） |
| 是否「官方 plugin-market」 | ✅ 是（要排除的） | ❌ 不是 |

「不走官方 market」**不**排除自建 marketplace——它就是「在 microNeo GitHub 仓库里放一个插件目录文件」。

### 1.2 Claude 与 pi/opencode 的插件模型差异（重点）

这是 npm 路在 Claude 上**形态不同**的根本原因：

| | pi / opencode | Claude Code |
|---|---|---|
| host 原生认 npm？ | ✅ `pi install npm:aibp-pi`，host 自己解析包并加载 | ❌ **没有「直接从 npm 装」的命令** |
| npm 在体系里的角色 | 交付 **+** 安装入口（二合一） | 仅 **交付**（plugin `source` 类型之一） |
| 安装入口 | 包名 specifier（`npm:xxx`） | **必须经 marketplace**：`claude plugin install <p>@<mkt>` |

> 官方原文：`claude plugin install` 的参数是 `plugin-name@marketplace-name`，「Install a plugin from available **marketplaces**」。npm 在 Claude 里只是 marketplace 条目里的一种 `source`（`{source:"npm", package:"aibp-claude"}`），不是独立安装通道。

**结论**：Claude 上必须 **npm 包（交付）+ marketplace 目录（入口）两层**。发布流程（`npm publish`）与 pi/opencode 完全一致；只是 Claude 的安装多一层 marketplace。

## 2. 推荐方案：npm 包 + marketplace（首选）

### 2.1 两层各自干什么

```
┌─ 交付层（npm，与 pi/opencode 一致）──────────────────┐
│  包名 aibp-claude，npm publish 发布                  │
│  内含：index.ts / package.json /                    │
│        .claude-plugin/plugin.json / README.md       │
│  版本：随 microNeo release bump（1.0.0 → 1.0.1 …）   │
└──────────────────────────────────────────────────────┘
                         ▲ source:{source:"npm",package:"aibp-claude"}
┌─ 目录层（marketplace，Claude 专属）──────────────────┐
│  一个 marketplace.json，条目用 npm source 指向上面的包 │
│  放哪？见 §2.2 两个变体                              │
└──────────────────────────────────────────────────────┘
```

### 2.2 marketplace 目录文件放哪（二选一）

#### 变体 A：放在 microNeo 仓库根（最少文件）

microNeo 根新增 `.claude-plugin/marketplace.json`（**仅此一个文件**，插件代码不动）：

```jsonc
{
  "name": "microNeo-plugins",
  "owner": { "name": "sollawen", "url": "https://github.com/sollawen" },
  "plugins": [
    {
      "name": "aibp-claude",
      "source": { "source": "npm", "package": "aibp-claude" },
      "description": "AIBP receiver for Claude Code — microNeo 选区/消息递送并立即触发响应",
      "homepage": "https://github.com/sollawen/microNeo/tree/main/internal/aibp/aibp-agents/claude",
      "repository": "https://github.com/sollawen/microNeo"
    }
  ]
}
```

> `source.version` 省略 = 跟最新；要锁定写 `"version": "^1.0.0"`。
> 该文件运行时**完全惰性**——只有用户显式 `marketplace add` 才读，不会让在 microNeo 里工作的 Claude 自动加载任何东西。

#### 变体 B：独立小仓库（根目录零侵入）

建 `sollawen/microNeo-plugins`（只有这一个 JSON），内容同上。microNeo 根目录保持 100% 干净，代价是多维护一个仓库。

### 2.3 用户安装（两条命令，永久）

```bash
# 一次性：登记 marketplace（变体 A 用 microNeo 仓库；变体 B 用 microNeo-plugins 仓库）
claude plugin marketplace add sollawen/microNeo

# 一次性：安装（默认 user scope，持久进 enabledPlugins）
claude plugin install aibp-claude@microNeo-plugins
```

之后**任何** `claude` / `ccmm` / `ccaibp` 启动自动加载，`ccaibp` 退化成纯 env 中转（去掉 `--plugin-dir`）。

### 2.4 发布与更新流程（与 pi/opencode 一致）

```bash
cd internal/aibp/aibp-agents/claude
# 改 package.json 的 version（1.0.0 → 1.0.1）
npm publish          # 交付层：发新版到 npm
# market 层无需改动（source 无 version = 自动跟最新）
```

> 若 `source` 里 pin 了 `version`，还要 bump 那一项再让用户 `/plugin update`。建议**省略 source.version**，发布即生效，省一步。

### 2.5 优缺点

| ✅ 优点 | ⚠️ 代价 |
|---|---|
| **与 aibp-pi/aibp-opencode 同一发布渠道与版本节奏** | Claude 侧多一层 marketplace 目录文件（npm 在 Claude 不能直接装） |
| npm 版本管理成熟（语义化、历史、回滚） | 用户机器仍需 bun（runtime，与交付无关） |
| 不 clone microNeo、不经官方 market | 占位包 `0.0.0` 要升级到 `1.0.0`（一次性） |
| user-scope 安装 → monitor 无限制加载（绕开 `--plugin-dir` 作用域存疑） | — |

## 3. 其它途径（对比用，均不主推）

| # | 途径 | 与 npm 方案比 |
|---|---|---|
| 0 | `--plugin-dir` 本地路径（当前） | 仅 session、要 clone、要 wrapper——开发期用，分发不用 |
| 1 | `--plugin-url` 远程 zip | 仅 session、每次带 flag；且要额外托管 zip，不如 npm |
| 2 | marketplace + **git/github/git-subdir source**（不经 npm） | 可行，但**脱离家族 npm 渠道**，版本与 pi/opencode 不同步 |
| 3 | `~/.claude/skills/` 就地（`plugin init`） | 仅本机，不能经 GitHub/npm 分发 |
| 4 | `extraKnownMarketplaces`（microNeo 的 `.claude/settings.json`） | 给 microNeo 贡献者用；外部用户不在 microNeo 仓库里起 Claude，无意义 |
| 5 | 官方 `@claude-plugins-official` / `-community` | **排除**（用户要求 + 要审核） |

> git-subdir 适合「不想发 npm」的场景；既然家族已走 npm，没必要另起一套。

## 4. 实施前要确认 / 处理的小项

1. **占位包升级**：当前 npm 上 `aibp-claude@0.0.0` 是空占位。把 `package.json` version 设 `1.0.0`，`npm publish` 覆盖出正式版（首次发布后 0.0.0 可忽略，或 `npm deprecate`）。
2. **version 策略**：`package.json` 设显式 version，随 release bump；marketplace 的 `source.version` **省略**（自动跟最新），发布即生效。
3. **`bin` 字段**：当前 `package.json` 有 `"bin": {"aibp-daemon":"./index.ts"}`。Claude 提取 tarball 到 cache 时不做全局 link，此字段对 Claude **惰性**；保留无害（便于将来 `npx` 独立跑 daemon 调试）。
4. **README 安装段**：当前只写 `--plugin-dir`。补「正式安装」一节，给 §2.3 两条命令 +「需要 bun」前置。
5. **`ccaibp` wrapper**：marketplace 装好后去掉 `--plugin-dir`，退化为纯 env 中转，README 同步。
6. **Bash 权限**：marketplace 安装的 monitor 通常预信任；若弹，仍用 README 现有 allow 规则。
7. **先实测再公开**：`marketplace add` → `install` → 起 Claude 确认 monitor 自动 spawn + `/plugin list` 见 `aibp-claude@microNeo-plugins` + microNeo Alt-Enter 端到端触发响应。

---

## 5. 可选的未来优化（非现在做）

**去 bun 依赖**：当前 monitor 靠 `bun index.ts`。既然走 npm，理论上可 `bun build` 把 `index.ts` 预编译成单文件 `index.js`，monitor 改 `node index.js`，用户就只需 node（Claude 自带/常见）。代价：加 build 步骤、与 pi/opencode 的「直接发 .ts」不再完全对齐。**当前不建议**——保持家族一致、零 build 优先；等真有用户反馈 bun 是负担再做。

---

## 6. 结论（已决策）

- **方案：npm 包 + marketplace**（§2）。交付层与 aibp-pi/aibp-opencode 完全一致（`npm publish`），Claude 侧多一个 marketplace 目录文件作为安装入口。
- ✅ **marketplace 目录文件放独立小仓库 `sollawen/microNeo-plugins`（变体 B）**：microNeo 根目录零侵入；代价仅多维护一个只有 1 个 JSON 的仓库。变体 A（并入 microNeo 根）不采用。
- 不触碰 Claude 官方 marketplace，完全自托管。

## 7. 落地清单（变体 B）

### 7.1 三处位置各放什么

| 位置 | 内容 | 谁读/拉 |
|---|---|---|
| ① microNeo 仓库 `internal/aibp/aibp-agents/claude/` | aibp-claude 全部源文件（**不动**） | 只有开发者；`npm publish` 的源头 |
| ② npm registry | 发布的 `aibp-claude` 包 | 用户安装/更新时 Claude 拉 |
| ③ `sollawen/microNeo-plugins` 仓库（**新建**） | 仅 `.claude-plugin/marketplace.json`（+可选 README） | 用户 `marketplace add` 时 Claude 拉 |

### 7.2 ③ 仓库的唯一文件 `.claude-plugin/marketplace.json`

```jsonc
{
  "name": "microNeo-plugins",
  "owner": { "name": "sollawen", "url": "https://github.com/sollawen" },
  "plugins": [
    {
      "name": "aibp-claude",
      "source": { "source": "npm", "package": "aibp-claude" },
      "description": "AIBP receiver for Claude Code — microNeo 选区/消息递送并立即触发响应",
      "homepage": "https://github.com/sollawen/microNeo/tree/main/internal/aibp/aibp-agents/claude",
      "repository": "https://github.com/sollawen/microNeo"
    }
  ]
}
```

> `source.version` 省略 = 跟最新（`npm publish` 新版即生效，无需改此文件）。

### 7.3 建 ③ 仓库（需手动执行）

```bash
mkdir microNeo-plugins && cd microNeo-plugins
mkdir -p .claude-plugin
# 把 §7.2 的 JSON 写到 .claude-plugin/marketplace.json
# （可选）写个 README.md 说明这是 microNeo 的 Claude plugin 目录

git init && git add . && git commit -m "init: microNeo-plugins marketplace (aibp-claude via npm)"
git branch -M main
git remote add origin git@github.com:sollawen/microNeo-plugins.git
git push -u origin main
# 注意：GitHub 上先建「空」仓库 sollawen/microNeo-plugins（不勾 README/.gitignore）
```

### 7.4 发布 npm 包 ②（需手动执行）

```bash
cd <microNeo>/internal/aibp/aibp-agents/claude
# package.json version 已是 1.0.0；确认 npm whoami 已登录
npm publish      # 占位包 0.0.0 → 正式 1.0.0
```

### 7.5 验证（端到端）

```bash
claude plugin marketplace add sollawen/microNeo-plugins
claude plugin install aibp-claude@microNeo-plugins
claude           # /plugin list 见 aibp-claude@microNeo-plugins
                 # task panel/monitor 列表见 aibp-daemon
                 # microNeo Alt-Enter 发消息 → Claude 立即 streaming 响应
```

### 7.6 后续发版

```bash
cd <microNeo>/internal/aibp/aibp-agents/claude
# bump package.json: 1.0.0 → 1.0.1
npm publish
# 用户：claude plugin update aibp-claude@microNeo-plugins（或已开 auto-update 自动拉）
```
