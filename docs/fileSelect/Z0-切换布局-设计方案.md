# 切换布局 —— 设计方案

需求来源：`Z0-切换布局.md`。本文是落地设计。

---

## 1. 现状（已查证）

**键位链路**
- `defaults_*.go` 里 `Ctrl-t` 原生绑 `AddTab`，但 `command_neo.go::InitNeoCommands()` 已把它重绑成 `HSplit`，并替换 `BufKeyActions["HSplit"] = neoHSplitAction`。所以现在按 Ctrl-t = 上下分屏，新 pane 在底部（`splitbottom=true` 默认）且立即激活。
- 分屏动作还有 `VSplit`（左右）和命令 `:vsplit` / `:hsplit`，都已经有 neo wrapper（`neoVSplitAction` / `neoVSplitCmd` / `neoHSplitCmd`）。

**分屏调用图（关键，决定守卫位置）**

所有「开一个 pane」的入口，最终都汇聚到 `BufPane.VSplitBuf` / `HSplitBuf`（bufpane.go:689 / 694）→ `VSplitIndex` / `HSplitIndex`（bufpane.go:661 / 675，真正建 pane 的地方）。已核实的全部入口：

| 入口 | 链路 |
|---|---|
| 键位 Ctrl-t / VSplit | `neoHSplitAction`/`neoVSplitAction` → `HSplitAction`/`VSplitAction`（actions.go:2032 / 2025）→ `HSplitBuf`/`VSplitBuf` |
| `:hsplit` / `:vsplit` 无参 | `neoHSplitCmd`/`neoVSplitCmd` → `HSplitCmd`/`VSplitCmd`（command.go:528 / 508）无参分支 → `HSplitAction`/`VSplitAction` → `HSplitBuf`/`VSplitBuf` |
| `:hsplit a b c` / `:vsplit a b c` 多文件 | `HSplitCmd`/`VSplitCmd` 的 `for _, a := range args` 循环，每文件调一次 `HSplitBuf`/`VSplitBuf` |
| `:help t1 t2 ...` 多主题 | `HelpCmd`（command.go:454）`for _, topic := range topics`（command.go:493）→ `openHelp`（command.go:430）→ `HSplitBuf`/`VSplitBuf` |
| `:openlog` / `:togglelog` | `OpenLogBuf`（globals.go:36）→ `HSplitBuf(LogBuf)` |
| 启动期多文件 `micro a b c`（`multiopen != "tab"`） | `InitTabs`（tab.go:209-216）`for _, b := range bufs[1:]` → `MainTab().CurPane().HSplitBuf`/`VSplitBuf` |
| Lua 插件 | 导出的 `HSplitBuf`/`VSplitBuf` 直接调 |

结论：**一处卡口（`VSplitBuf`/`HSplitBuf`）即可覆盖所有 split 入口**，这是本方案守卫位置的选择依据，也决定了「不逐入口加守卫、只堵汇聚点」的实现方式。

**分屏树 `views.Node`（internal/views/splits.go）**
- 每个分屏 pane 对应树上一个叶子节点，叶子 id = pane 的 `splitID`。
- `HSplit` 后：root 节点 `Kind = STVert`（纵向堆叠），下挂 2 个叶子（上、下）。
- `VSplit` 后：root 节点 `Kind = STHoriz`（横向并排），下挂 2 个叶子（左、右）。
- `Node.ResizeSplit(size)`：把当前叶子重排到 `size`（高度或宽度），兄弟节点吃掉剩余空间；末尾调 `markSizes()` 同步 `propH/propW`，所以随后任何重排都保留新比例。

**现成可复用的方法（重要：不新写）**
- `BufPane.ResizePane(size int)`（bufpane.go:325）：内部就是 `GetNode(splitID).ResizeSplit(size)` + `h.tab.Resize()`。这正是鼠标拖拽分隔线调比例走的同一条路径（tab.go:295-296），已验证可用。
- `Tab.Resize()`（tab.go:380）：遍历 panes，把每个 pane 的 view/BWindow 对齐到其节点几何。`ResizePane` 末尾已调它。
- `InfoBar.Message(string...)`：全项目大量先例（actions.go:1051 等）。

**Tab.Panes / multiopen**
- `Tab.Panes []Pane` + `Tab.active int`，`len(Panes)` 就是当前 tab 的 pane 数。
- `multiopen`（settings.go:127，默认 `"tab"`）：`"tab"` → 多文件各开一个 tab（每 tab 1 pane）；`"hsplit"`/`"vsplit"` → 第一个文件进首 tab，后续文件全部 `HSplitBuf`/`VSplitBuf` 进**同一个 tab**。

## 2. 目标行为

**产品边界（先讲 why）**：microNeo 的分屏 = 「对照模式」，一个 tab 只允许两个 buffer 并排看/改；要同时开更多文件 → 开更多 tab（快速切换），要并排不同程序 → 交给 tmux/zellij。这和 file selector / birth selector / Z1 tab 切换 / quit selector 一脉相承，都是「专注一个 buffer + 快速切换」，而非「把屏幕切碎」。

1. **每 tab 最多 2 个 pane**，且这个约束是**不可绕过的不变量**——无论从键位、命令、help、log、还是命令行启动进入，第 3 个 pane 一律不创建。
2. **三档比例**：50:50、25:75、75:25（上下分屏是「上:下」，左右分屏是「左:右」）。
3. **Alt-,**：扩大当前 pane（向下一档更大的比例）；**Alt-.**：缩小当前 pane（向下一档更小的比例）。到顶/到底则 no-op + 提示。

## 3. 设计

### 3.1 每 tab 最多 2 个 pane —— 单点卡口 + wrapper 短路

**主卡口（一处覆盖全部入口）**：在 `VSplitBuf` / `HSplitBuf`（bufpane.go:689 / 694）顶部加守卫。由第 1 节调用图，所有 split 入口都过这里，所以这一处就能拦住键位、`:vsplit`/`:hsplit`（含多文件循环）、`:help` 多主题、`:openlog`、启动期 `InitTabs` 的 `hsplit`/`vsplit`、Lua 插件全部路径。

```go
func (h *BufPane) VSplitBuf(buf *buffer.Buffer) *BufPane {
    if h.tab != nil && len(h.tab.Panes) >= 2 {
        InfoBar.Message("already 2 panes in this tab")
        return h   // 命中卡口：返回当前 pane（非 nil），调用方不会 NPE；不建新 pane
    }
    return h.VSplitIndex(buf, h.Buf.Settings["splitright"].(bool))
}
// HSplitBuf 同构
```

返回 `h` 而非 `nil`：`OpenLogBuf`（globals.go:36）写法是 `LogBufPane = h.HSplitBuf(buffer.LogBuf); LogBufPane.CursorEnd()`，返回 nil 会 NPE；返回当前 pane 只在「2 pane 时还调 openlog」这个极边缘场景让 `LogBufPane` 临时指向当前 pane（语义略偏、不崩），见 5.6。

**4 个 neo wrapper 顶部加短路（只为不开 birth selector）**：主卡口已能阻止 split，但 `neoHSplitAction`/`neoVSplitAction`/`neoHSplitCmd`/`neoVSplitCmd` 在调完底层后还会 `OpenBirthSelectors(...)`（command_neo.go:115-143）。若只留主卡口，2 pane 时按 Ctrl-t 会：底层 split 被拦 + 提示，但 birth selector 仍弹出——体验混乱。所以这 4 个 wrapper 各加一个顶部短路，命中即提示 + return，连底层调用和 selector 都跳过：

```go
func (h *BufPane) neoHSplitAction() bool {
    if len(h.tab.Panes) >= 2 {
        InfoBar.Message("already 2 panes in this tab")
        return false
    }
    dir := birthDir(h)
    r := h.HSplitAction()
    OpenBirthSelectors(MainTab().CurPane(), dir)
    return r
}
```

注意：这 4 个短路**不是** 2-pane 约束的执行点（那是主卡口），它们只负责「交互入口的干净 UX」。仍需要它们、而不只留主卡口的原因：selector 弹不弹必须在 wrapper 这层决定，主卡口管不到 selector。

**不守卫** `neoAddTabAction` / `neoNewTabCmd`：它们开的是新 tab，不受单 tab 2-pane 限制。

**命令行多文件无需额外处理（已自洽）**：用户要求「命令行多文件多了丢弃」。由调用图，启动期 `InitTabs` 的 `hsplit`/`vsplit` 分支也走 `HSplitBuf`/`VSplitBuf`，主卡口直接生效：
- `multiopen=hsplit`（或 `vsplit`），`micro a b c`：首文件 a 进 tab1 → b 经 `HSplitBuf` 建第 2 pane（卡口放行）→ c 经 `HSplitBuf` 撞卡口被丢 + 提示。结果 tab1 = {a, b}，正是「最多 2、多了丢」。
- `multiopen=tab`（默认），`micro a b c`：3 个文件各开一个 tab，每 tab 1 pane，不违反 2-pane 约束，全部保留。

因此**不需要**改 `LoadInput`（cmd/micro/micro.go:150）截断文件参数：2-pane 不变量已由主卡口保证，`tab` 模式下多 tab 是合理且无害的（用户本意就是多 tab 快速切换，见产品边界）。若将来产品上想连 `tab` 模式也限制「命令行最多 2 文件」，再在 `LoadInput` 加截断即可，当前不做。

提示文案（简洁英文避免编码问题）：
- 分屏被拒：`"already 2 panes in this tab"`
- （3.3 的）resize 无效：`"pane already at max"` / `"pane already at min"`

### 3.2 三档比例 + Alt-,/Alt-. 步进

**比例集合**（当前 pane 自身占父容器的份额）：`S = {0.25, 0.5, 0.75}`。

**当前尺寸 cur**：用节点几何直接读，不维护额外状态（这样能兼容鼠标拖出的任意中间比例、Unsplit 后重建等情况）：

```
n := tab.GetNode(h.splitID)        // 当前 pane 的叶子节点
p := n.Parent()                    // ⚠ views.Node.parent 未导出，需加访问器 Parent()（见 §4）
若 p == nil（只有 1 个 pane，无兄弟）→ 不能 resize，提示后 return
若 p.Kind == STVert（上下分屏）→ cur = n.H,  total = p.H
否则（STHoriz，左右分屏）      → cur = n.W,  total = p.W
isFirst := (p.Children()[0] == n)  // 当前 pane 是上/左（第一个子节点）还是下/右
```

**为什么用「像素」比较，不用比例 cur/total**：pane 尺寸是整数像素，`ResizeSplit` 取整后实际落点会偏离名义档位——例如 total=82、目标 75%，下 pane 实际是 62/82 = 0.756（不是 0.75）。若拿比例去比 `0.756 > 0.75`，缩小键会把目标又指回 75%，而它已经在 75% → 卡死。所以档位要换算成「像素」，且必须**照搬 ResizeSplit 的取整方式**，让 cur 精确命中某个档位像素，`>` / `<` 严格比较即可，无需 epsilon：

```
stepPixel(r):
    if isFirst:  return int(r * total)                  // 上/左 pane：ResizeSplit 直接截断
    else:        return total - int((1-r) * total)      // 下/右 pane：取补
```

两个分支不同的原因：`ResizeSplit` 把 `size` 永远装给 `children[0]`（见下方「执行」），所以 children[0] 的尺寸是 `int(r*total)`（截断），children[1] 是 `total - int((1-r)*total)`；两者都 ≠ `round(r*total)`，且不能同时等于 round（两个 round 之和可能 ≠ total）。必须按当前 pane 是第几个分别算，否则差 1 像素就会卡住。

**步进规则**（单调夹紧，不回环）：
- Alt-,（扩大）：`target = min{ s ∈ S : stepPixel(s) > cur }`；不存在（已在最大档）→ 提示 "already at max"，return。
- Alt-.（缩小）：`target = max{ s ∈ S : stepPixel(s) < cur }`；不存在（已在最小档）→ 提示 "already at min"，return。

对中间像素（鼠标拖到任意位置）：两键都把分隔线吸附到最近的档位方向，符合直觉。

**为什么这套三档语义成立，前提就是 max-2**：单层 2 叶树里，「当前 pane 的兄弟」永远只有一个，`ResizeSplit` 才有无歧义的对端；树一旦嵌套，「当前 pane 的兄弟」可能不止一个，三档比例就崩了。所以「砍到 2」和「Alt-,/Alt-.」是配套的，不是两个独立特性。也正因如此，3.1 的主卡口必须真正不可绕过——一旦漏一个入口让 3 pane 出现，3.2 的档位语义就失效。

**执行**：`ResizeSplit(size)` 的 `size` 是「第一个子节点（上/左 pane）」的尺寸——对 2-pane，无论当前 pane 是哪个，传进去的 size 都落到 `children[0]`（`vResizeSplit`/`hResizeSplit` 恒把 `c1=children[0]` 设成 size）。所以先把「当前 pane 的目标比例」折算成「第一个 pane 的比例」，再调用：

```
ratio := target
if !isFirst { ratio = 1.0 - target }   // 当前 pane 是下/右（第二个）时取补
h.ResizePane(int(ratio * float64(total)))
```

`ResizePane` 内部已做 `ResizeSplit + tab.Resize()`，比例随 `markSizes()` 持久到 `propH/propW`，后续窗口缩放不丢。**不做这个取补会反向**：实测聚焦在下 pane 时反复按扩大键，下 pane 在 25%↔50% 振荡（因为 size 被错装到上 pane）。

### 3.3 新增两个 action + 键位绑定

仿照 `neoHSplitAction` 的写法，在 `command_neo.go` 加两个 BufKeyAction 方法：

- `func (h *BufPane) GrowPane() bool` —— Alt-, 的执行体（3.2 扩大逻辑）
- `func (h *BufPane) ShrinkPane() bool` —— Alt-. 的执行体（3.2 缩小逻辑）

注册 + 绑定（放 `InitNeoBindings()`，紧跟现有 Ctrl-q 绑定）：

```
BufKeyActions["GrowPane"]   = (*BufPane).GrowPane
BufKeyActions["ShrinkPane"] = (*BufPane).ShrinkPane
BindKey("Alt-,", "GrowPane",   Binder["buffer"])
BindKey("Alt-.", "ShrinkPane", Binder["buffer"])
```

`Alt-,` / `Alt-.` 是直键（无 Shift），`Alt + 单 rune` 在 tcell 下稳定匹配。**注意**：这两个键原是默认的标签页切换 `PreviousTab`/`NextTab`（defaults_darwin.go:56-57）；neo 的 `InitNeoBindings` 晚于默认加载执行 `BindKey`，会覆盖它们——标签页切换改由其它键承担（后续单独绑定，本方案不含）。选 `,`/`.` 而非原先的 `+`/`_` 是因为后者是 Shift 组合（`Shift==` / `Shift--`），实测 `Alt-Shift-=` 在该终端 tcell 报键与绑定 rune 不匹配、绑不上（按下去被当普通字符插入）。macOS 需终端把 Option 当 Meta（已有约定）。仍须在 `BufKeyActions[...]` 赋值之后再 `BindKey`（command_neo.go:26 注释强调此顺序坑）。

## 5. 边界 / 风险

1. **只有 1 个 pane 时按 Alt-,/Alt-.**：`n.parent == nil` → 提示 "no other split"（或静默 no-op，倾向提示），**然后 return，不得再调 `ResizePane`**。原因：单 pane 时 `GetNode(splitID)` 返回根叶子节点（`parent == nil`），而 `ResizeSplit`（splits.go:193）开头就是 `len(n.parent.children)` → nil 解引用 panic，**不会**返回 false。所以 `GrowPane`/`ShrinkPane` 里的 `n.Parent() == nil` 检查是**唯一保险**，必须先判后调，不能依赖 `ResizeSplit` 自身兜底。
2. **2-pane 不变量不可绕过**：主卡口在 `VSplitBuf`/`HSplitBuf`，覆盖键位 / `:vsplit`/`:hsplit`（含多文件循环）/ `:help` 多主题 / `:openlog` / 启动期 `InitTabs` 全部入口。microNeo 不持久化 split 布局（`internal/config` 无任何 session / 分屏树落盘，`Settings.Save` 只写 settings JSON），所以不存在「旧配置残留 3 pane」的启动路径。Lua 插件直接调导出的 `HSplitBuf`/`VSplitBuf` 同样撞卡口（这正是想要的）。
3. **比例集合硬编码 {0.25,0.5,0.75}**：未来若想加档（如 0.1/0.9），改一个常量切片即可，不必现在抽象成配置项（如非必要勿增实体）。
4. **`Alt-,` / `Alt-.` 覆盖默认标签页切换**：这两个键原绑 `PreviousTab`/`NextTab`（defaults_darwin.go:56-57）。`BindKey` 在 `InitNeoBindings`（晚于默认加载）执行，会覆盖。标签页切换功能改由其它键承担（后续单独任务），本方案不含。选 `,`/`.` 而非 `+`/`_` 的根因：`+`/`_` 是 Shift 组合键，`Alt-Shift-=` 实测绑不上（按下去插入普通字符），而 `,`/`.` 是直键、无此问题。
5. **最小高度**：`ResizeSplit` 拒绝 `size<=0` 且 `size>=total`；0.25*total 在合理终端高度（≥4 行）下合法。窗口极矮时 0.25 档可能撞下限，届时该次 resize 被 `ResizeSplit` 拒绝、比例不变，无副作用。
6. **2 pane 时调 `:openlog`（极边缘）**：主卡口返回当前 pane（非 nil），`LogBufPane` 临时指向当前 pane + 对它 `CursorEnd()`，不崩、语义略偏。`:togglelog` 正常路径是先 `Quit()` 关掉 log pane 再开，不触发此场景，不专门修。
7. **`Unsplit` / 关掉一个 pane 后的状态**：`Node.Unsplit`（splits.go:464）末尾 `flatten()` 在根只剩一个子节点时执行 `*n = *n.children[0]; n.parent = nil`，把剩余叶子提升为新根，回到单叶子状态：剩余 pane 的 `splitID` 仍 = `tab.ID()`、`root.parent == nil`。因此再按 Ctrl-t 重新分屏正常，Grow/Shrink 正确命中边界 1 的"无兄弟"提示→return。三档比例语义只在 2 叶子时成立，单叶子一律提示，不破坏不变量。

## 6. 实施顺序（出 PLAN 模式后）

1. `bufpane.go`：`VSplitBuf`/`HSplitBuf` 加主卡口 → `make build-quick`。
2. `command_neo.go`：4 个 wrapper 加短路 → 验证 2 pane 时 Ctrl-t / `:hsplit` / `:vsplit` 提示且 birth selector 不弹。
3. 验证主卡口覆盖全部入口（本方案的正确性核心）：
   - `:vsplit a b c`（从 1 pane）→ 只到 2 pane，c 被丢 + 提示。
   - `:help help commands`（多主题）→ 只到 2 pane。
   - `multiopen=hsplit` 下 `micro a b c` → tab1 = {a,b}，c 丢 + 提示；`multiopen=tab` 下 `micro a b c` → 3 个 tab 全保留。
4. `views/splits.go`：加 `Parent()` 一行访问器；`command_neo.go`：加 `GrowPane`/`ShrinkPane`（用 `n.Parent()` 判 nil、取 `Kind`/`H`/`W`）+ 注册 + 绑定 → 验证 Alt-,/Alt-. 在 2 pane 时三档切换、单 pane 时提示。
5. 其余验收：
   - 1 pane Ctrl-t → 2 pane 50:50；再 Ctrl-t → 提示、仍是 2 pane。
   - 2 pane Alt-,（当前=下）→ 上 25:下 75；Alt-, → 已到 max 提示。
   - Alt-. 连按 → 75:25 → 50:50 → 25:75 → 已到 min 提示（视当前 pane 位置）。
   - 鼠标拖到中间比例后 Alt-,/Alt-. 能吸附到最近档。
   - 窗口缩放后比例保持。
