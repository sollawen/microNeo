# 我提一个思路

## 1，把今天写的判断welcome mode之类的代码删掉
不再去判断有没有命令行文件名参数了。一视同仁，走标准的microNeo的开局。一直到 finishInitialize()完成，然后开始判断：

thisPane.isNoName=false
if this pane is noName then {
	thisPane.isNoName=true
}

if thisPane.isNoName then {
	fileSelector.open(isWelcome=false) 	// esc 就是关闭Selector，让用户真的编辑这个空buff
}

当这个pane创建完成后自动打开selector.open(isWelcome-false). 这个时候用户是只能呆在这个selector里面的，根本出不去。用户只有两条路可以走
- 选择一个文件，回车，于是这个pane进入了编辑这个文件的状态
- 在selector里面啥也不选，按esc退出selector。这时这个pane进入了真正编辑一个空文件的状态

关于 thisPane.isNoName
- 我希望 thisPane.isNoName 是个不变量，在这个pane创建的时候根据是不是noName来赋值。
- 后来用户在这个pane里面，不论多少次切换了别的文件，也不要改变这个变量的值
- 也就是说，这个变量就是这个Pane是怎么诞生的，它诞生的时候是不是noName
- 而且我以后想把用户在一个pane里面切换到另一个文件的所有功能点都关掉，根本不允许用户在一个pane里面切换文件。
- 用户只能按ctrl-p调出来fileSelector来切换文件

当microNeo启动后
-  如果用户是用不带文件名的方式运行的microNeo，那么第一个pane一定是noName，所以ctrl-q的时候一定会打开fileSelector
-  如果用户是用带文件名的方式运行的microNeo，那么第一个pane一定不是noName，所以ctrl-q的时候一定就退出程序了

## 2，当用户在某个pane里面按 ctrl-q 的时候

if not thisPane.isNoName then {
	走正常的，原来的那个 ctrl-q的处理
} else {
	fileSelector.open(isWelcome=true)
	// 打开文件选择器，
	// 当用户在文件选择器里面再次按了 ctrl-q 的时候，再走原来的那个ctrl-q，我记得是三级判断处理吧
}

当用户在这个pane里面的编辑状态的时候，按ctrl-q会打开selector.open(isWelcome-true),这个时候会把当前文件提示用户存盘的，然后才进入selector
- 用户选择了另外一个文件，回车，去编辑
- 用户没选择任何文件，按ctrl-q, 关闭这个pane

两个场景：
- 当这个pane创建时就带着文件名的时候，那么我希望ctrl-q就直接退出这个pane。
- 当这个pane创建的时候是noName的话，不论用户后来在这个pane里面切换了多少次文件，我希望用户在这个pane里面按ctrl-q的时候，不直接退出这个pane，而是进入selector(true)，通过在这个selector里面再按ctrl-q实现退出这个pane

# 分析与讨论

核心是把**全局 `WelcomeMode`（启动期、靠命令行参数判断）换成 per-pane 的 `isNoName`（任何 pane 只要当前 buffer 是 noName 就成立）**。这一步换对了——startup / vsplit / hsplit / 新 tab / 关文件回 welcome 全统一成同一件事，`DetectWelcomeMode`、`EnterWelcome`、`openWelcomeSelector`、`exitProgram` 可以整体删掉。

## 状态机制：

- **pane 就绪（finishInitialize）→ 若 noName → 开 selector(mode F = isWelcome=false)**：Esc 关选择器进 scratch 编辑、pick 开文件。
- **noName pane 里按 Ctrl-q → 开 selector(mode T = isWelcome=true)**：这次不退出，只是把选择器切到"退出模式"。
- **mode T 选择器里再按 Ctrl-q → 真退出（三级判断）**。




**B. `gotoWelcome`（关最后一个真实文件）走的是 `OpenBuffer` 换 buffer，不造新 pane，不触发 `finishInitialize`。**
所以"关掉最后文件 → 自动弹 selector"这条，finishInitialize 抓不到。要么 gotoWelcome 显式调同一个 `openNoNameSelector(h)`，要么把触发也挂到 `OpenBuffer`（换入 noName 时）。倾向前者：抽一个 neo 函数 `openNoNameSelector(h)`，finishInitialize 和 gotoWelcome 都调它。

> 我觉得就不存在gotoWelcome了
