# 需求

现在的microNeo，
- 一运行就会打开一个已有的文件或是一个新的空文件。
- 在编辑时，按ctrl-q就退出microNeo，回到shell

我的想法：
- 如果用户是用 microneo somefilename 来运行的，则直接打开这个文件进行编辑
- 如果用户是用 microneo 就回车了，那么
	- 什么文件都不打开，直接进入welcome状态，显示fileSelector
	- 如果用户在fileSelector里选择了某文件，则打开这个文件进入编辑
	- 如果用户在编辑中，按ctrl-q退出，则不是直接退出microNeo，而是退出并回到welcome状态，显示fileSelector

# 我再说详细些

## 只有不带文件名参数的运行microNeo，才进入这种工作模式

就是说，如果用户是使用带文件名的命令行来运行的，就是正常的现在的工作模式

如果用户没有带文件名参数就运行了microNeo，就是这种工作模式。我们起个名字吧，叫 File模式
- 这个File模式，一直持续到用户退出microNeo为止
- 在File模式里面，程序启动后，先进入welcome页面，或者叫welcome界面，或者叫welcome状态
- 用户结束编辑之后，不会退出microNeo，而是回到welcome界面
- 只有在welcome界面里面按ctrl-q，才真正结束程序

welcome界面
- 左40%，是fileSelector，用户可以在里面选择一个文件进入编辑
- 用户选择了文件后，welcome界面就消失了，进入了正常的编辑这个文件的界面
- 用户在关掉了最后一个打开的文件后，就退回到welcome界面里
- 右40%，我觉得先空着。我以后想显示一些帮助信息，新版本通知之类的。甚至未来可以在右边的区域里面预览当前选中的文件内容。
- 按 esc 或 ctrl-q 退出welcome界面，结束microNeo

你来研究一下，这个思路是否可行，工作量大吗？
