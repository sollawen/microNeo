# 需求

现在的microNeo，
- 一运行就会打开一个已有的文件或是一个新的空文件。
- 在编辑时，按ctrl-q就退出microNeo，回到shell

我的想法：
- 如果用户是用 microneo somefilename 来运行的，则直接打开这个文件进行编辑
- 如果用户是用 microneo 就回车了，那么
	- 什么文件都不打开，直接进入welcome状态，显示fileSelector
	- 如果用户在fileSelector里选择了某文件，则打开这个文件进入编辑
	- 如果用户在编辑中，按ctrl-o退出，则不是直接退出microNeo，而是退出并回到welcome状态，显示fileSelector
