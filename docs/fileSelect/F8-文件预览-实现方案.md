# F8 · FileSelector 文件预览 实现方案

**前置**：finder 已稳定（F0–F7），`fileselectwidth` 默认 0.4，finder box 只占 owner pane 左 40%。
**依据**：`internal/finder/session.go`（`Open` / `handleKey` / `Display` / `drawBorder` / `drawString`）、`internal/action/bufpane.go`（`HandleEvent` modal 转发 / `Display` overlay）、`internal/action/tab.go`（鼠标路由）、`internal/action/fileops.go`（`OpenFinder` 传 owner rect）。
**交付**：finder 打开期间，右侧 60% 区域显示光标所在**文本文件**的只读预览（超宽原始行右半截断、不画半字符），支持鼠标滚轮上下滚动；目录、二进制、读取失败等均给占位文案。
**原生侵入**：零。改动仅限 `internal/finder/`。

---

## 1. 背景与目标

### 1.1 现状

`Open` 用 `fileselectwidth`（默认 0.4）算出 `pickerW`，finder box 宽即 `pickerW`：

```go
fm.rect = Rect{X: rect.X, Y: rect.Y, W: pickerW, H: rect.H - 1}
```

owner pane 全宽是 `rect.W`，右边 `[rect.X+pickerW, rect.X+rect.W)` 这一整段（约 60%）**没有任何东西画**——`BufPane.Display` 先画编辑区（`BWindow.Display`），finder 只在自己 box 内 `SetContent` 清屏 + 绘制，右侧露出底层编辑区内容，视觉上既空又乱。

### 1.2 目标

把右侧空区改成一个**文件预览面板**：

- 只预览**文本文件**；目录、面包屑不预览（占位文案）。
- 只预览**文本**内容；二进制（含 NUL）不预览（占位文案）。
- **无语法高亮**、**无 softwrap**——超宽原始行右半截断不显示，仅供「一眼看个大概」；要看完整内容，Enter 进 BufPane。
- **只接鼠标滚轮**滚动；不支持键盘滚动、不支持点击定位、不支持编辑。

---

## 2. 关键发现

### 2.1 右侧区域天然可用

finder box 与预览区在 owner rect 内左右并列、区域不重叠。预览区每帧清屏天然盖住底层编辑区。`fileselectwidth` 继续只管 finder box 宽，预览宽 = 剩余宽度，二者解耦。

### 2.2 鼠标滚轮直达 finder，不触发失焦关闭

事件路由（finder 打开时，鼠标位于 owner pane 区域内）：

```
tab.go  Tab.HandleEvent
  ├─ Button1 按下 → SetActive(i) → 旧 pane onOwnerBlur → finder.NotifyBlur → close(Esc)
  └─ WheelUp/Down → default 分支：按坐标找到 owner pane → p.HandleEvent(event)，【不调 SetActive】
        → bufpane.go  HandleEvent: finder.IsOpen() → finder.HandleEvent(event)
```

即：**滚轮**不会触发失焦、不会关 finder，事件原样到达 `finder.HandleEvent`。finder 现在只认 `EventKey`：

```go
ev, ok := event.(*tcell.EventKey)
if !ok { return }   // *tcell.EventMouse 在此被静默丢弃
```

所以接滚轮是**纯增量**：在 `HandleEvent` 加 `*tcell.EventMouse` 分支，坐标落在预览矩形内且是 `WheelUp/Down` 时滚动即可。点击（Button1）仍走 tab 层 SetActive → 关 finder，是现有行为，不动。

### 2.3 不复用 BufWindow / buffer.Buffer

`display.BufWindow` 是完整编辑器窗口（MD 渲染 + screenBuffer + gutter + scrollbar + statusline + cursor + editMode），`buffer.Buffer` 是完整可编辑缓冲（SharedBuffer / EventHandler / 全局 `OpenBuffers` 注册表 / filetype·syntax 检测）。为只读纯文本预览强接它们会带来：

- 跑 filetype/syntax 检测（与「不要高亮、不要 softwrap」冲突）；
- 文件被塞进全局 `OpenBuffers`，用户随后真正打开该文件时会撞上 buffer 复用逻辑；
- 还要写胶水关掉 MD / gutter / scrollbar / cursor / statusline。

finder 本身就是「自画框、直接 `screen.Screen.SetContent` 写屏」的自洽 overlay（见 `drawBorder` / `drawContent`）。预览沿用同一模式才最小、最一致。

## 3. 布局

owner rect（`OpenFinder` 传入）= `{X: v.X, Y: v.Y, W: v.Width, H: v.Height}`，末行留给 statusLine。

| 区域 | 矩形 | 来源 |
|---|---|---|
| finder box（已有） | `{X: rect.X, Y: rect.Y, W: pickerW, H: rect.H-1}` | `Open` 已算 |
| 预览区（新增） | `{X: rect.X+pickerW, Y: rect.Y, W: rect.W-pickerW, H: rect.H-1}` | 本次新增字段缓存 |

预览区**不画边框**。finder box 的右边框（位于 `rect.X+pickerW-1` 的 `│`）暂时充当左右两栏的分隔线，预览区从 `rect.X+pickerW` 起、紧贴该竖线右侧；预览区的上、右、下三边暂无线条——这是过渡态。

**未来计划**：把 finder 与预览合并成**一个大外框**，中间一条竖线分隔左右两栏，标题统一嵌在上边框。本次只先把预览的内容画对、矩形字段留好，**不动 finder 现有 `drawBorder`**，避免影响已稳定的 finder 渲染；合并大框另开方案。

预览内容区 = 整个预览矩形（无边框，不减 2）。`drawString` 调用时宽度直接传 `pvRect.W`，超宽原始行右半截断不画（drawString 内部 `col+rw > x+w` break）。

`pickerW` 已在 `Open` 里做过 `fsMinWidth` 下限与 `rect.W` 上限保护；预览宽 = `rect.W - pickerW`。预览宽 < `previewMinWidth`（默认 10）时不绘预览（整段留给 finder），纯展示降级，不影响 finder 任何逻辑。

## 4. 数据结构

**新文件** `internal/finder/preview.go`，全部状态集中在一个结构体，挂在 `finderState` 上：

```go
// previewState 是文件预览的可变状态。仅主线程读写（reload / load 在 handleKey 链上、
// draw 在 Display 链上，均主线程；fetchGit 后台 goroutine 只碰 git 字段，不碰预览），
// 故无需 mu。
type previewState struct {
	path         string   // 当前预览文件的绝对路径；"" = 无预览
	previewLines []string // 按 '\n' 切的原始行（一原始行 = preview 一屏行）
	truncated    bool     // 文件超大被截断（仅读前 previewMaxBytes）
	binary       bool     // 判定为二进制（前 N 字节含 NUL）
	readErr      bool     // 读取失败（权限/IO 等）
	topLine      int      // 滚动位置：previewLines 数组下标（原始行号）
}
```

**常量**（同文件顶部）：

```go
const (
	previewMaxBytes      = 256 * 1024 // 单文件预览读取上限；超出截断
	previewBinProbeBytes = 8 * 1024   // 二进制探测窗口（前 N 字节含 NUL 即判二进制）
	previewMinWidth      = 10         // 预览区宽低于此则不绘预览
	previewScrollStep    = 3          // 每次滚轮滚动行数
)
```

**矩形字段**：`pvRect Rect` **挂在 `finderState` 上（不在 `previewState`）**——它是 finder 算出来的预览区域矩形，preview 只是个纯消费者。由 `finder.Open` 在每次进入时算一次：`{X: rect.X+pickerW, Y: rect.Y, W: rect.W-pickerW, H: rect.H-1}`。preview 包内的 `drawPreview` / `scrollPreview` / `handlePreviewWheel` 都直接读字段、不重算。运行中 resize 触发 finder 整体关闭→重开（§7），重开时重新计算并刷新该字段，不需单独适配。

## 5. 改动清单

### 5.1 `internal/finder/preview.go`（新增）

四个 `Session` 方法（与 `drawBorder` / `drawContent` 同级）：

- `refreshPreview()` — 用户输入触发的「按当前光标刷新预览」统一入口：光标在文件 → 拼绝对路径调 `loadFile`；目录/面包屑/越界 → `clearPreview`。moveCursor / chdirTo / toggleHidden 末尾各调一次。
- `loadFile(path string)` — 真正读盘、二进制探测、按 `'\n'` 切 `previewLines`、置标志；**path 未变则 no-op**（保留滚动位置）。仅 `refreshPreview` 调用。
- `drawPreview()` — 清预览矩形（盖底层编辑区）+ 按状态分支绘制（无文件/二进制/出错给居中占位文案，正文逐行 `drawString`，超宽右半截断）；**不画边框、不画标题**。
- `scrollPreview(delta int)` — `topLine += delta`，clamp 到 `[0, len(previewLines)-1]`。
- `clearPreview()` — 仅置 `pv.path = ""`、`pv.truncated/binary/readErr` 复位；`previewLines` 和 `topLine` **不清**——`loadFile` 入口会重置（读 path 第一个非空 buf 后重建 previewLines + `topLine = 0`），故 `clearPreview` 无需重复。

### 5.2 `internal/finder/session.go`（接线，5 处）

| # | 位置 | 改动 |
|---|---|---|
| 1 | `finderState` 结构体 | 加 `preview *previewState` 字段 + `pvRect Rect` |
| 2 | `Open` | 算 `pvRect = {X: rect.X+pickerW, Y: rect.Y, W: rect.W-pickerW, H: rect.H-1}` 并存；在 `newFinderState` 返回后赋值 `fm.state.preview = &previewState{}`；调 `fm.refreshPreview()` 让首帧有内容 |
| 3 | `moveCursor` / `chdirTo` / `toggleHidden` 末尾 | 各调一次 `fm.refreshPreview()`（具体位置：`moveCursor` 在 `fm.ensureVisible()` 后、`screen.Redraw()` 前；`chdirTo` 在 `s.mu.Unlock()` 后、`screen.Redraw()` 前；`toggleHidden` 在 `fm.ensureVisible()` 后、`screen.Redraw()` 前） |
| 4 | `Display` 末尾（`drawContent()` 之后） | 追加 `fm.drawPreview()` |
| 5 | `HandleEvent` | 在 `EventResize` 判断之后、`handleKey` 之前插入 `*tcell.EventMouse` 分支：坐标在 `pvRect` 内 + `WheelUp/Down` → `scrollPreview(∓previewScrollStep)` + `screen.Redraw()` |

**新文件 import**（`internal/finder/preview.go`）：`io`（`io.ReadFull`，session.go 未引入）、`bytes`（`bytes.ContainsRune`，未引入）；`os` / `strings` / `filepath` 在 session.go 已有。`screen` / `tcell` / `config` / `util` 已在 session.go 导入。

### 5.3 复用项（不新写）

| 能力 | 复用位置 |
|---|---|
| rune 显示列宽 | `internal/finder/strutil.go` 的 `runeWidth` / `stringWidth` |
| 限宽写字（CJK 双宽 + 超宽 break + 行尾 fill） | `session.go` `drawString(x, y, w, text, style)` —— 预览直接调用，**不再写 softwrap** |

## 6. 核心算法

### 6.1 refreshPreview + loadFile

```go
// refreshPreview 按当前光标刷新预览（统一入口）。moveCursor / chdirTo / toggleHidden
// 末尾各调一次。同步读盘 < 10ms（256KB 本地盘），path 未变时 loadFile 内部 no-op
// 保留滚动位置。
func (fm *Session) refreshPreview() {
	s := fm.state

	// gate 1：cursor 不合法
	if s.cursor < 1 || s.cursor-1 >= len(s.showEntries) {
		fm.clearPreview()
		return
	}
	e := s.showEntries[s.cursor-1]

	// gate 2：当前光标是目录
	if e.isDir {
		fm.clearPreview()
		return
	}

	// 真加载
	fm.loadFile(filepath.Join(s.currentDir, e.name))
}

// loadFile 载入一个文本文件用于预览。path 与当前相同则 no-op（保留滚动位置）。
// 二进制（前 previewBinProbeBytes 含 NUL）/ 读失败 / 截断分别置标志。文本内容按
// '\n' 切原始行存进 previewLines，运行时逐行 drawString，超宽右半自然截断。
// 在主线程 handleKey 链上同步调用（256KB 本地盘 < 10ms，不卡）。
func (fm *Session) loadFile(path string) {
	pv := fm.state.preview

	// gate 3：缓存命中（光标停在同一文件不动 / 重复 trigger）
	if pv.path == path {
		return
	}
	pv.path = path
	pv.previewLines = nil
	pv.truncated = false
	pv.binary = false
	pv.readErr = false
	pv.topLine = 0

	if path == "" {
		return // 空 path 不读盘（drawPreview 走占位文案）
	}

	f, err := os.Open(path)
	if err != nil {
		pv.readErr = true
		return
	}
	defer f.Close()

	// 多读 1 字节探截断：读满 N+1 字节 = 文件 > N（未截断时 ReadFull 在 EOF 返回
	// ErrUnexpectedEOF + 全量字节数，忽略错误）。
	buf := make([]byte, previewMaxBytes+1)
	n, _ := io.ReadFull(f, buf)
	buf = buf[:n]
	if n > previewMaxBytes {
		pv.truncated = true
		buf = buf[:previewMaxBytes]
	}

	probeEnd := len(buf)
	if probeEnd > previewBinProbeBytes {
		probeEnd = previewBinProbeBytes
	}
	if bytes.ContainsRune(buf[:probeEnd], 0) {
		pv.binary = true
		return // 不再 split：避免二进制内容被当文本一行超长
	}

	pv.previewLines = strings.Split(string(buf), "\n")
	if pv.truncated {
		pv.previewLines = append(pv.previewLines, " (truncated)") // 末行标记，画到屏自然出现
	}
	// 首行 BOM (U+FEFF) 暂不剔除（边缘 case）。设计取舍见 §8 #13。
}

func (fm *Session) clearPreview() {
	pv := fm.state.preview
	pv.path = ""
	pv.truncated = false
	pv.binary = false
	pv.readErr = false
}
```

### 6.2 运行时绘制

```go
// drawPreviewBody 在内容区 [x,y,w,h] 内从 topLine 起逐行绘制。previewLines 是按
// '\n' 切的原始行，每条可能远超 w 列——由 drawString 限宽 col+rw > x+w 时 break
// 截断（右半不画、不切半 CJK），尾部 fill space 清底层编辑区。
func (fm *Session) drawPreviewBody(x, y, w, h int, style tcell.Style) {
	pv := fm.state.preview
	sy := y
	for i := pv.topLine; i < len(pv.previewLines) && sy < y+h; i++ {
		fm.drawString(x, sy, w, pv.previewLines[i], style)
		sy++
	}
	for ; sy < y+h; sy++ {
		fm.clearRect(x, sy, w, 1, style)
	}
}
```

### 6.3 滚动

```go
// scrollPreview 按原始行滚动预览。delta 正向下、负向上；clamp 到 [0, len(previewLines)-1]。
func (fm *Session) scrollPreview(delta int) {
	pv := fm.state.preview
	if len(pv.previewLines) == 0 {
		return // 空预览 / load 未就绪不滚动
	}
	top := pv.topLine + delta
	if top < 0 {
		top = 0
	}
	if top > len(pv.previewLines)-1 {
		top = len(pv.previewLines) - 1
	}
	pv.topLine = top
}
```

### 6.4 drawPreview 入口 + handlePreviewWheel

```go
// drawPreview 在 Display 链调用：清屏 + 按状态分支绘制。
func (fm *Session) drawPreview() {
	pv := fm.state.preview
	r := fm.state.pvRect
	if r.W < previewMinWidth {
		return // 预览宽不够，整个预览区域不绘（让给 finder）
	}
	// 1) 清屏（盖底层编辑区）
	fm.clearRect(r.X, r.Y, r.W, r.H, config.DefStyle)
	// 2) 按状态分支
	x, y, w, h := r.X, r.Y, r.W, r.H
	switch {
	case pv.readErr:
		fm.drawCenteredText(x, y, w, h, "Unable to preview", config.DefStyle)
	case pv.binary:
		fm.drawCenteredText(x, y, w, h, "Binary file", config.DefStyle)
	case pv.path == "" || len(pv.previewLines) == 0:
		fm.drawCenteredText(x, y, w, h, "Select a file", config.DefStyle)
	default:
		fm.drawPreviewBody(x, y, w, h, config.DefStyle)
	}
}

// drawCenteredText 在 [x,y,w,h] 矩形内画一行居中文本。文本超出 w 时由 drawString 截断；
// h < 1 时 no-op。
func (fm *Session) drawCenteredText(x, y, w, h int, text string, style tcell.Style) {
	if w < 1 || h < 1 {
		return
	}
	tw := stringWidth(text)
	sx := x + (w-tw)/2
	if sx < x {
		sx = x
	}
	sy := y + h/2
	fm.drawString(sx, sy, w-(sx-x), text, style)
}

// HandleEvent 改动：在 EventResize 判断之后、handleKey 之前插入鼠标分支。
func (fm *Session) HandleEvent(event tcell.Event) {
	if !fm.isOpen {
		return
	}
	if _, ok := event.(*tcell.EventResize); ok {
		fm.close(Resize)
		return
	}
	if ev, ok := event.(*tcell.EventMouse); ok { // 新增
		fm.handlePreviewWheel(ev)
		return
	}
	fm.handleKey(event)
}

// handlePreviewWheel 仅处理预览区滚轮：坐标落在预览矩形内才滚动，其余鼠标事件丢弃。
func (fm *Session) handlePreviewWheel(ev *tcell.EventMouse) {
	mx, my := ev.Position()
	r := fm.state.pvRect
	if mx < r.X || mx >= r.X+r.W || my < r.Y || my >= r.Y+r.H {
		return // 不在预览区：丢弃（finder 列表区滚轮仍 no-op）
	}
	switch ev.Buttons() {
	case tcell.WheelUp:
		fm.scrollPreview(-previewScrollStep)
		screen.Redraw()
	case tcell.WheelDown:
		fm.scrollPreview(previewScrollStep)
		screen.Redraw()
	}
}
```

## 7. 边界行为

| 场景 | 行为 |
|---|---|
| 光标在文件行 | 预览该文件（path 不变则保留滚动位置） |
| 光标在目录 / 面包屑 / 越界 / 空目录 | 预览清空，占位 `Select a file` |
| 二进制文件（前 8KB 含 NUL） | 占位 `Binary file` |
| 文件 > 256KB | 读前 256KB，末行追加 `(truncated)` |
| 读取失败（权限/IO/不存在） | 占位 `Unable to preview` |
| 预览期间原文件被删除/移动 | `os.Open` 失败 → `readErr=true` → 占位 `Unable to preview`（与上一行同一路径） |
| 滚轮在预览区内 | 上下滚动 ±3 行，clamp 到 [0, len-1] |
| 滚轮在 finder 列表区内 | 维持现状（no-op） |
| 点击（Button1）任意位置 | 维持现状（tab 层 SetActive → finder 关） |
| owner pane 过窄（预览宽 < 10） | 不绘预览，整宽留给 finder |
| 运行中 resize | finder 已 `close(Resize)` 整体取消，预览随之销毁，无需单独处理 |
| CJK / 全角字符 | `drawString` 按 `runeWidth` 占 2 列；超宽右半截断，不画半字符 |
| 超长无换行行 | 右半截断；想看完整，Enter 进 BufPane |
| tab / 控制字符 | 直接写入，由 tcell 默认处理（与原文可能不一致，可接受） |
| **按住 ↓ 连扫文件** | 每次按键 reload → 同步读盘 < 10ms → 下一帧画新文件（按键节奏 ~30ms/次 富余 20ms）。**本地 SSD 场景下成立**；NFS/SMB 网络盘 / 机械盘随机读可能 200ms+，同步阻塞会卡 UI 帧（详见 §8 #9 前提） |
| **多 pane（VSplit/HSplit）** | 各 pane finder 互不串扰，预览状态独立 |
| **colorscheme 热切换** | 预览只读 DefStyle，不受影响 |

## 8. 设计取舍

| # | 决定 | 理由 |
|---|------|------|
| 1 | **自包含 preview.go，不复用 BufWindow/Buffer** | BufWindow 是带 MD/gutter/scrollbar/cursor 的完整编辑器窗口；Buffer 会跑 syntax 检测并污染全局 `OpenBuffers`。finder 本就是自画框 overlay，预览沿用同模式最一致、代码量最小。 |
| 2 | **不 softwrap，超宽右半截断** | 预览只为「眼个大概」；softwrap 需重写 CJK/tab/换行规则（容易切半字符），而 `session.drawString` 已天然限宽截断。超长行要看完整，Enter 进 BufPane 即满足。 |
| 3 | **二进制探测用「前 8KB 含 NUL」** | git diff / grep 的通用启发式，零依赖、对常见二进制（图片/压缩包/可执行）命中率高。 |
| 4 | **读取上限 256KB** | 预览非编辑，大文件截断无害；避免读 GB 级文件卡主线程（reload 同步在事件链上执行）。 |
| 5 | **path 不变则保留滚动位置** | `toggleHidden` / 重复落同一文件时不闪回顶部；loadFile 入口统一 guard（gate 3），零额外状态。 |
| 6 | **滚轮每次 3 行** | 手感常量（`previewScrollStep`）；嫌慢/嫌快改一处。 |
| 7 | **仅预览区接滚轮，列表区滚轮仍 no-op** | 用户只要求预览滚动；列表区滚轮属 F2 范畴（且当前 finder 未实现），留待以后。 |
| 8 | **三道 gate（学 yazi）** | `refreshPreview` 两道（cursor 合法、非目录）+ `loadFile` 一道（path 未变）。把分散在三个函数的判断统一为「gate」心智模型，参考 yazi `preview.go::go()`。 |
| 9 | **每次输入无脑触发，不 debounce** | 学 yazi：每次光标移 `peek` 无脑调，过 gate 才真干活。**前提**：本地 SSD 256KB 读盘 < 10ms，富余按键节奏 ~30ms/次；debounce 是伪问题。NFS/SMB 网络盘 / 机械盘随机读可 200ms+ 卡 UI 帧，该场景需未来引入异步 IO（§10 #6）。 |
| 10 | **预览仅主线程，不加 mu** | reload / load / draw 都在主线程；fetchGit 后台 goroutine 只写 git 字段。无数据竞争，省一把锁。 |
| 11 | **预览区不画边框、不画标题** | 过渡态：finder 右边框暂作左右分隔线。未来合并成「大框 + 中竖线」时再统一画外框与标题，本次不动 finder `drawBorder`，避免回归风险。当前预览文件由 finder 光标高亮行表明。 |
| 12 | **预览宽 < 10 不绘** | 极窄 pane 下毫无可读性；降级为整宽 finder，不报错。 |
| 13 | **首行 BOM (U+FEFF) 不剔除** | UTF-8 文本（Windows 创建）首行会带一个 ZWSP，肉眼可见但不影响阅读。后续可 `strings.TrimPrefix(previewLines[0], "\uFEFF")` 一行解决，本期留作边缘 case 不处理。 |

## 9. 验证

`make build` 后手测：

- [x] finder 打开，右侧出现预览区（无边框），正文逐原始行显示；当前预览文件 = finder 光标高亮行。
- [x] ↑↓ 移动光标到另一文件：预览内容随之切换。
- [x] 光标移到目录行 / 面包屑（cursor=0）：预览显 `Select a file`。
- [x] `. ` 切换 hidden 后光标仍落在同名文件：预览内容不变、滚动位置保留。
- [x] 进/出子目录（← → Enter）：预览按新光标文件刷新。
- [x] **滚轮在预览区内上下**：正文上下滚动，到顶/底 clamp 不越界。
- [x] 滚轮在 finder 列表区内：无反应（不影响光标）。
- [x] 滚动后再 ↑↓ 切到另一文件再切回：滚动位置因 path 复原而保留。
- [x] 二进制文件（如图片/可执行）：预览显 `Binary file`。
- [x] 超大文本文件（> 256KB）：预览显前 256KB，末行追加 `(truncated)`。
- [x] 无读权限文件：预览显 `Unable to preview`。
- [x] 含 CJK 的文件：双宽字符占 2 列，超宽右半截断不画半字符。
- [x] **超长无换行行**：右半截断不显示；想看完整内容，Enter 进 BufPane。
- [x] owner pane 极窄（预览宽 < 10）：不绘预览，finder 不受影响。
- [x] 运行中 resize：finder 整体关闭（既有行为），无残留。
- [x] 预览期间点 finder 列表/别处：finder 按既有行为关闭。
- [x] **多 pane（VSplit/HSplit）**：每 pane 的 finder 互不串扰，预览状态独立。
- [x] **colorscheme 热切换**：预览只读 DefStyle，切换 colorscheme 不影响预览内容、颜色不变。
- [x] **按住 ↓ 不松手连扫文件**（本地 SSD）：每按一次同步 reload + 读盘 < 10ms，下一帧切到新文件（读盘不拖按键节奏）。网络盘场景本期不覆盖。
- [x] **空目录 / 全 hidden 目录**：所有路径下都是 `Select a file` 占位。
