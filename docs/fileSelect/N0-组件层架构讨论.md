# 现状
floatFrame 是一个 modal 容器（主循环级，单实例），上面现有这些应用：
- `:theme` 打开主题选择器（SelectDialog）
- NotePane 的 Receiver 选择器（SelectDialog）

这些 Dialog 都是「open 后接管所有交互，关闭后才交回给调用者」的 modal 组件。

# 未来的需求

未来会需要两类组件，底层架构不同、语义清晰分开。

## 1. Dialog（modal 类）

**特征**：一次性、独占、数据进数据出。走 floatFrame 容器（主循环级，全局单实例）。

现有应用：
- SelectDialog：列表选择一个 item（`SelectPane` → `SelectDialog` rename 待执行）
  - `:theme` 主题选择器
  - NotePane Receiver 选择器

计划扩充：
- **InputDialog**：单行输入和编辑，返回 string。类似 InfoBar 的功能，但浮动在屏幕任意位置。finder 重命名文件/目录时使用。
- **ConfirmDialog**：确认对话框，一段文字 + OK/Cancel 两个按钮。finder 删除文件/目录时弹出。

## 2. Panel（长期存活类）

**特征**：长期存活、不阻塞、焦点驱动。嵌在 owner 矩形区域内（owner-local），多个 Panel 可并存，焦点在哪个 Panel 里交互就在哪个里。

底层架构：走 Pane 机制（owner 管理子 Panel 的布局、焦点、事件转发），不依赖 floatFrame。

典型应用场景（finder）：

- **PreviewPanel**：finder 右侧 60% 显示当前选中文件的预览。长期存活，跟随光标更新内容，支持滚动。

- **HistoryPanel**：finder 左侧文件列表下方显示访问历史。用户可选择历史快速跳转到目录/文件。与文件列表 Panel 之间可焦点切换。

注：finder 的核心文件列表也是 finder 自己维护的视图区域（不是独立 Panel），但可以拆成独立 Panel 以统一管理——视实现复杂度决定。
