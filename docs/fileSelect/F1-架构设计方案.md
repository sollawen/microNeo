### 2.1 分层

### 2.2 数据流（一次完整的"打开 → 选中 → 开文件"）

```
用户键入 :file
  → FileCmd(h)（command_neo.go:54）
       ├─ h.Buf.Modified() && !h.Buf.Shared()? → YNPrompt：y=保存后继续 / n/Esc=取消
       ├─ pane = 发起 :file 的那个 BufPane（h）
       ├─ 起始目录 = filepath.Dir(h.Buf.AbsPath)（无路径 fallback os.Getwd()）
       └─ NewFileSelector().Open(h, startDir, onSelect=打开文件)
            ├─ State 初始化（os.ReadDir，μs 级，同步）：列目录、排序、定位当前文件、dotfile 自动显
            ├─ computeLayout 预检（W<minWidth / avail<minHeight → InfoBar 提示并 onSelect(nil) 返回）
            ├─ buildSpec → TheFloatFrame.Open(FloatOpenSpec{
            │      Anchor=pane 左上角, ContentSize, Title="Open File",
            │      Display, HandleEvent, OnCancel, AutoExpand=false})
            ├─ Open 返回 false → onSelect(nil) 透明返回（picker 没开，不查 git）
            └─ Open 成功 → 后台启 git 查询（渐进显示）：
                  go fs.fetchGit(dir)
                    └─ m,ok := gitCache.statusFor(dir)   // 带 2s ctx
                       state.setGitStatus(m,ok); screen.Redraw()
                  // 首次 Display 时 gitStatus 已空 → 列表可见但无 git 标志；
                  //   git 回来 → Redraw → 列表补 git 标志（光标/滚动在 State，不受影响）

[运行期] 主循环把键事件交给 FloatFrame.HandleEvent
  → FloatFrame.handleEvent（FileSelector 的 Controller，fileselector.go:handleEvent）
       ├─ ↑↓ → 改 State.cursor / 滚动 → Redraw
       ├─ ←/→/Enter(面包屑)/Enter(目录)/Enter(文件) → 导航或选中
       ├─ . → 翻转 showHidden → 重列条目 → Redraw
       └─ Esc → Close + onSelect(nil)
  → FloatFrame.Display → FileSelector.display（View，读 State 画三段）

resize 到达 → FloatFrame 拦截 → Close + onCancel → onSelect(nil)

选中文件：
  Enter(文件行) → pick(): Close + onSelect(&absPath)
    → onSelect 闭包（command_neo.go:openSelector）：NewBufferFromFile → h.OpenBuffer(b)
       （与 :open 同路径，文件开进发起 :file 的那个 pane）
```

## 3. FloatFrame 容器现状（floatframe.go）

FloatFrame 是 microNeo 自有的统一浮窗容器：单例 `TheFloatFrame`、模态、单浮窗（同时全局最多一个）、画框、清屏、生命周期、事件路由、"后写胜出"（`micro.go` 主循环里 FloatFrame 最后画）。容器很薄，FileSelector 与 SelectPane（`:theme` 用）都是它的具体浮窗。

### 3.1 `FloatOpenSpec`（options 入参）

`Open` 经 options 模式收参，自文档化、加能力不 churn 签名：

```go
type FloatOpenSpec struct {
    Anchor      Pos                     // AutoExpand=true: 展开中心点; false: 外矩形左上角(含边框)
    ContentSize Size                    // 纯内容尺寸(不含边框); FloatFrame 内部派生 outerW/outerH
    Title       string                  // 嵌入上边框的标签; 空串=纯横线
    FrameColor  tcell.Style             // 边框色; 零值 = config.DefStyle
    Display     func(contentArea Rect)  // 画内容(收到的 area 已扣除边框)
    HandleEvent func(event tcell.Event) // 处理键事件(resize 不会到达, FloatFrame 已拦截)
    OnCancel    func()                  // FloatFrame 自身被关(resize)时回调; 具体浮窗在此清理业务回调
    AutoExpand  bool                    // true: 锚点自适应展开(SelectPane); false: 钉死 Anchor(FileSelector)
}
```

`Open(spec) bool`：返回 false = 没开成（已有浮窗在开 / 屏幕放不下），调用方应 `onSelect(nil)` 透明返回。

### 3.2 `AutoExpand` layout 开关

layout 按 `AutoExpand` 分叉：

| `AutoExpand` | `Anchor` 语义 | 行为 | 谁用 |
|---|---|---|---|
| `true` | 展开中心点（屏幕参考点） | FloatFrame 跑 `expandAnchor`，自适应上下左右 | SelectPane（贴光标/贴 statusLine 弹） |
| `false` | **外矩形左上角（含边框）** | FloatFrame 直接 `fx=Anchor.X, fy=Anchor.Y`，**跳过 `expandAnchor`** | FileSelector（pane 派生） |

职责划分：FloatFrame 只做失败前置检查（outer 是否放得下屏）+ 画框 + 生命周期 + 事件路由，**不碰 pane 边界**；FileSelector 自算 pane 派生的左上角 + 尺寸 + pane-local 预检，"不溢出到别的 pane" 100% 由 FileSelector 把 outer 尺寸钉成 pane 的 avail / pickerW 保证。`AutoExpand=false` 时 FloatFrame 不替调用者纠偏（不 clamp Anchor），位置合法性由调用者保证。

sentinel 细节：`Open` 内 `if spec.AutoExpand && ay < 0` 时把 `ay` 解释为"距 statusLine 的负偏移"（SelectPane 贴 statusLine 的约定），直接读入参 `spec.AutoExpand` 判定、不抄到容器字段（避免抄写时机的残留 bug）。`false` 路径 anchor 恒正（pane 左上角），完全不经过 sentinel。

### 3.3 resize 即关 + `OnCancel` 钩子

`HandleEvent` 收到 `EventResize` 时，FloatFrame **直接 Close + 调 onCancel，不再转发**给具体浮窗——所有浮窗统一"resize 即关"。关闭语义等同 Esc：业务回调收到 `onSelect(nil)`。

处理顺序（对齐"先关再回调"约定）：

```go
case *tcell.EventResize:
    cb := f.onCancel          // 先存
    f.Close()                 // 关容器(内部已清 onCancel)
    if cb != nil { cb() }     // 再触发业务取消回调
    return                    // 不再转发给具体浮窗
```

关键约束：`OnCancel` 内部**不得再调 `TheFloatFrame.Close()`**（重入，FloatFrame 已在关）。`OnCancel` 只做"调业务回调"这一件事。`Close()` 清空 `onCancel`（连同 `display`/`handleEvent`，断 GC 引用），幂等。

### 3.4 绘制

`Display()`：HideCursor → 清整个外矩形 → 4 角 + 上下左右边 → title 嵌入上边框（`──<title>──...─`）→ 委托 `f.display(contentArea)` 画内容（contentArea = `{fx+1, fy+1, contentSize.W, contentSize.H}`，已扣边框）。具体浮窗完全不需要知道边框存在。

## 4. FileSelector 主体（fileselector.go）

### 4.1 State / View / Controller 三层分离

| 层 | 落点 | 职责 | 不做什么 |
|----|------|------|---------|
| **State** `fileSelectorState` | 结构体字段 | 当前目录、条目切片（已排序/已过滤）、cursor、topIdx（滚动）、showHidden、pickerW、listH、git 状态、元数据缓存 | 不碰屏幕、不读键 |
| **View** `display(area)` | 方法 | 读 State，画面包屑（行0）+ 文件列表（视口）+ 末行元数据；左截断、Reverse 光标、颜色 | 不改 State |
| **Controller** `handleEvent(ev)` | 方法 | 读键 → 改 State（移动/导航/翻转显隐）或触发选中/取消 | 不直接画屏（只 `Redraw`） |

三层同处一个文件、共享 `FileSelector` 接收器，但职责清晰分离。v1.1 fuzzy 只需在 State 上加 `filter string` + 过滤后的条目视图，View 几乎不动，Controller 加"可打印字符 → 追加 filter"。

### 4.2 State 结构

```go
type fileSelectorState struct {
    currentDir string
    entries    []entry   // 已排序（目录优先 + 大小写不敏感字母序）、已按 showHidden 过滤
    cursor     int       // 0=面包屑行, 1..len(entries)=条目
    topIdx     int       // 文件列表视口顶（指向 entries 索引）
    showHidden bool
    pickerW    int       // 内容区宽（截断用）
    listH      int       // 文件列表可见行数（内容区高 - 面包屑 - 末行）

    // git 状态（异步、并发保护）：后台 goroutine 写、主循环 Display 读
    gitStatus map[string]statusKind
    gitOK     bool       // false=无 git 列（降级）
    mu        sync.RWMutex

    // 元数据缓存（按条目名缓存，末行用）
    metaName  string     // 缓存对应的条目名（""=面包屑/空/越界，无缓存）
    metaSize  int64
    metaMtime time.Time
    metaOK    bool       // false=stat 失败（断链/无权限），字段降级显 —
    metaIsDir bool       // = info.IsDir()（跟随符号链接）
}
```

`entry` 只存 `name`+`isDir`（来自 `os.ReadDir`，**不跟随**符号链接）。元数据按需 `os.Stat` 缓存进 state，不污染 entry。

git 并发访问契约：Display 读 git 一律走 `gitOf(name)`（RLock），绝不裸读 map；后台写走 `setGitStatus(m, ok)`（Lock）。

### 4.3 生命周期

**Open**（`fileselector.go:Open`）：
1. State init（`os.ReadDir`，μs 级，同步）→ gitStatus 空。光标起始停当前 buffer 文件上（F0 §5.3）；当前文件是 dotfile 且默认隐藏 → `showHidden=true` 后停其上；找不到/无路径 → 首条目。
2. `computeLayout` 预检 + 算 spec；预检不过 → `onSelect(nil)` 透明返回。
3. `TheFloatFrame.Open(spec)`；返回 false → `onSelect(nil)` 透明返回。
4. 后台 `go fs.fetchGit(dir)`——绝不阻塞首次渲染。`fetchGit` 内防竞态：仅当 `state.currentDir == dir`（未导航离开）时才应用结果。

**关闭**（三种路径，最终都触发恰好一次 `onSelect` 回调）：
- 选中文件（`pick`）/ Esc（`cancel`）：Controller 内走"存 onSelect → `TheFloatFrame.Close()` → 调 onSelect"。
- resize：FloatFrame 拦截 → `Close()` → 调 `OnCancel` → `onSelect(nil)`。

**回调契约**：每次 Open 恰好一次 onSelect 回调（与 SelectPane 一致）。选中 → `onSelect(&absPath)`；Esc / resize / 拒开 → `onSelect(nil)`。

### 4.4 Layout（pane-local，`computeLayout`）

```
view := pane.BWindow.GetView()       // *View{X,Y,Width,Height}
paneX, paneY := view.X, view.Y
paneW        := view.Width
avail        := view.Height - 1      // 全高模型（留 statusLine）

widthFrac := config.GetGlobalOption("fileselectwidth").(float64)  // 默认 0.4
pickerW   := int(widthFrac * paneW)
if pickerW < fsMinWidth { pickerW = fsMinWidth }
if pickerW > paneW     { pickerW = paneW }   // 防御：不溢出相邻 pane
```

预检（拒开条件）：
- `paneW < fsMinWidth(20)` → InfoBar 提示 pane 太窄，拒开。
- `avail < fsMinHeight(10)` → InfoBar 提示 pane 太矮，拒开。

换算 FloatOpenSpec：
```
Anchor      = {paneX, paneY}                    // AutoExpand=false 时 = 外矩形左上角（盖住 gutter）
ContentSize = {pickerW - 2, avail - 2}          // 减 FloatFrame 左右/上下边框
Title       = "Open File"
AutoExpand  = false                             // 钉死 pane 左上角，跳过 expandAnchor
OnCancel    = func(){ onSelect(nil) }           // resize 即关
```

常量：`fsMinWidth = 20`、`fsMinHeight = 10`（保证外高 ≥10 → 内容区 ≥8 行 = 面包屑 + ≥6 行文件列表 + 末行）。无退化分支——minHeight 保证三段布局始终放得下。

### 4.5 渲染（View，`display(area)`）

内容区高 = `area.H`（恒 ≥8），自上而下三段：

1. **行 0 面包屑**（`drawBreadcrumb`）：左截断全路径、恒保留"当前目录/"；根目录显 `/`；光标在此行时该行 Reverse（与文件行选中态一致）。截断在路径段边界，截断处用 `…/` 标记。
2. **行 1..H-2 文件列表**（`drawEntry`）：视口 `[topIdx, topIdx+visibleH)`；每行格式 `[标记] 名[/] [git]`：
   - 标记：目录 `▸`、文件空格（预留 1 字符位）。
   - 名：目录带 `/` 后缀；超长**左截断保留扩展名**（最后一个 `.` 之后，rune-safe；目录名里的 `.` 不当扩展名）。
   - git 状态：靠右、单字符、按颜色区分；无 git 时整列隐藏。
   - 选中行 Reverse；溢出时右侧画 `▲`/`▼`（style 跟随所在行，对齐 SelectPane 滚动指示范式）。
3. **末行元数据**（`buildMetaLine`，见 §4.6）。

配色：目录名用 colorscheme `type` 色；文件名用 `default`；git 状态按 §5.5 表；边框色传零值 → FloatFrame 用 `config.DefStyle`。

写屏一律按 CJK 显示列宽推进列坐标（`runeWidth`/`stringWidth`），放不下完整双宽字符则停（不写半截），避免中文被下一个字符覆盖。

### 4.6 末行：光标条目元数据（非按键 hint）

末行展示光标所在条目的实时元数据，**完全不显示** Enter / `.` / Esc 等按键提示：

| 光标位置 | 末行内容 |
|---|---|
| 文件 | `size   mtime`（如 `12.3K   01-15 14:30`） |
| 子目录 | size 段**留空但保留固定宽** + `mtime`（mtime 与文件行落在同一列） |
| 面包屑行（row 0） | 空白 |

**数据来源**：光标落到某条目时 `os.Stat(filepath.Join(currentDir, name))`（跟随符号链接，对齐 `ls`）。

**懒查 + 按条目名缓存**（`refreshMetaIfStale`）：guard 放在 `display` 画末行之前——名字没变则命中缓存、零 stat；变了才 stat 一次。display 是所有渲染的唯一入口，一处 guard 覆盖全部 cursor 变化路径（↑↓/chdir/toggleHidden/Open），无需在各 cursor 变化点分别埋刷新。命中时（绝大多数重绘）= 一次字符串比较；miss 时 = 1 次 stat（本地 SSD ~3µs）。重复 Redraw 不会变成 stat 风暴。

**size**（`humanSize`，对齐 `ls -lh`，1024 进制，输出恒 ≤5 列）：
- `< 1024` → 整数 + `B`（`405B`）
- `≥ 1024` → 除到 mantissa < 1024：mantissa `< 100` → 1 位小数（`3.4M`、`99.9K`）；mantissa `≥ 100` → 整数（`123K`），舍入后 ≥ 1024 则进位到下一单位（`1023.6K → 1.0M`）。

**mtime**（`formatMtime`，对齐 `ls -lh`）：同年 → `MM-DD HH:MM`（11 列）；跨年 → `YYYY-MM-DD`（10 列，省时分）。

**降级**：stat 失败（断链/无权限）→ 缺失字段显 `—`（U+2014，**显示宽度 2 列**）并补齐到字段定宽，不破坏 mtime 列对齐；光标在面包屑行 → 末行空白。

设计取舍：末行从原静态按键 hint 改为元数据——元数据是高频真需求（面对同名前缀文件靠 size/mtime 区分），按键提示是一次性 onboarding、常驻是视觉噪音。Esc 是 modal 通用退出约定、Enter 是通用激活、`.` 是 power-user 功能，靠交互自然发现。

### 4.7 Controller（`handleEvent`，键位表）

resize 不会到达这里（FloatFrame 已拦截）。

| 键 | 行为 |
|----|------|
| `↑` / `↓` | 移动光标（**不循环**，clamp `[0, len]`），滚动跟随 + Redraw |
| `Enter`（面包屑行） | 回父级目录 |
| `Enter`（目录条目） | 进入该目录 |
| `Enter`（文件条目） | Close + `onSelect(&absPath)` |
| `←` | 回父级目录（不移光标） |
| `→` | 光标在目录 → 进入 |
| `.` | 切换 dotfile 显隐（重列 + 光标 clamp：同名优先，否则按索引 clamp 到最近可见条目） |
| `Esc` | Close + `onSelect(nil)` |
| 其它键 | 吞掉（modal） |

`.` 的实现：浮窗模态拦截所有事件，在 `default` 分支里用 `ev.Key()==tcell.KeyRune && ev.Rune()=='.'` 直接匹配，绕过 micro 键位绑定（micro 主键位表里 `.` 可能已被占用）。

导航约定：进入子目录时光标落首条目；回父级时光标落在"刚离开的目录"上（ranger/lf 约定）。每次 chdir 重置 git（新目录首帧无 git 列）并后台重查。

## 5. git 状态子模块（fileselector_git.go）

git 状态是 FileSelector 里**唯一一项"外来"能力**——micro 全仓库无任何 git 调用经验。封装为独立、可降级的子模块，经接口解耦，任何环节失败都静默降级为"不显示状态列"，绝不阻塞浏览。

### 5.1 接口与实现

```go
type gitStatusCache interface {
    statusFor(dir string) (map[string]statusKind, bool)
    // false = 不可用（非仓库/超时/diffgutter 关），调用方降级为不显示
}
```

`*gitStatus` 是阻塞实现：持 `cache map[string]map[string]statusKind`（key=dir），`sync.Mutex` 保护。FileSelector 持 `gitCache gitStatusCache` 字段（`NewFileSelector()` 注入真实实现），便于 mock 测试。

**职责边界**：FileSelector 对 git 状态的全部工作就是 `os/exec` 起 git → 读 stdout → 解析两字符状态码 → 显示。**从不读文件内容、不算哈希、不碰 `.git/index`**——git 是黑盒，只消费它的输出文本。之所以关心 git 的内部成本，只因为它决定要**等多久**（所以有 2s 超时）。

### 5.2 机制：fork 系统 git + pathspec 限定

`statusFor(dir)` cache miss 时跑两条 git（都用 `-C <dir>`）：

```
git -C <dir> status --porcelain=v1 -z -- .      // 拿「哪些文件有变更」
git -C <dir> rev-parse --show-prefix             // 拿「当前目录相对仓库根」的前缀（亚毫秒）
```

- **`-- .`（pathspec）**：把工作限定到当前目录子树——不只过滤输出，未追踪扫描也只走这一棵子树（对齐 eza 快的做法）。大仓库不扫全仓。
- **`--porcelain=v1 -z`**：机器可解析、NUL 分隔（文件名含空格/换行也安全）。
- **用系统装的 git**：零新依赖、版本跟用户一致、自动尊重用户的 `config` / `.gitignore` / submodule。评估过 libgit2（cgo）/ go-git（纯 Go 几 MB），换来的收益是每次省 1–5ms 进程启动（人眼无感），不划算。

两条命令分工：status 给数据，rev-parse 给参照系——status 恒输出**仓库根相对**路径（与 cwd / `-z` / `-- .` / `status.relativePaths` 全都无关，实测铁律），需要 prefix 把它换算成**相对当前目录**的路径。

### 5.3 prefix 剥离 + 取顶层分量（子目录状态透传）

`parsePorcelain(out, prefix)` 的核心：

```go
rel := strings.TrimPrefix(path, prefix)          // 剥前缀 → 相对当前目录的路径
var name string
if i := strings.IndexByte(rel, '/'); i >= 0 {
    name = rel[:i]                                // 含 / → 变更在子目录内，取顶层目录名（列表里那个 dir 条目）
} else {
    name = rel                                    // 当前目录内的文件
}
```

效果：子目录内（任意深度）有变更文件时，当前列表里对应的顶层子目录条目亮状态——根目录与任意层子目录都对。例：

| 浏览位置 | prefix | porcelain 路径 | 剥后 (rel) | 顶层分量 (name) | 列表条目 |
|---|---|---|---|---|---|
| 根 | `""` | `docs/x.md` | `docs/x.md` | `docs` | `docs/` ✓ |
| `internal/` | `internal/` | `internal/action/a.go` | `action/a.go` | `action` | `action/` ✓ |
| `internal/` | `internal/` | `internal/flat.go` | `flat.go` | `flat.go` | `flat.go` ✓ |

`--show-prefix` 与 porcelain 路径**同一参照系（都仓库根相对）**，剥的是精确前缀，不靠猜，不存在误判。跨平台只切 `/`：git porcelain / rev-parse 在**所有平台（含 Windows）都发 `/`**；不要用 `\` / `filepath.Separator`（Unix 上 `\` 是合法文件名字符，`-z` 下 verbatim，双切会误伤）。

### 5.4 状态码映射与聚合优先级

porcelain 两字符 `XY`（索引状态 + 工作区状态），FileSelector 主要关心工作区状态（第二字符），也回退检查索引状态（如 `M ` 表示暂存区已改、工作区干净）：

| porcelain | 含义 | 显示 |
|---|---|---|
| `??` | 未追踪 | **U** |
| 含 `M`/`T`（` M` / `M ` / `MM` / `MT`…） | 已修改 / 类型变更 | **M** |
| `A ` / `AM` | 已暂存新增 | **A** |
| `D ` / ` D` | 删除 | **D** |
| `R ` | 重命名 | **R** |
| 全空格 | 干净 | 无标记 |

`I`（Ignored）需要额外 `--ignored`，开销大且 dotfile 默认隐藏时触发面小，**延后未做**（见 §8）。

多状态聚合：同一顶层目录下多个不同状态文件时，`st < existing`（枚举值最小 = 最高优先级）合并。当前枚举序为 `stModified(1) < stUntracked(2) < stAdded(3) < stDeleted(4) < stRenamed(5)`，即实际优先级是 **M > U > A > D > R**。

### 5.6 异步时序与渐进显示

git 查询**绝不阻塞首次渲染**：

```
1. Open() 同步：State init（os.ReadDir，μs 级）→ gitStatus = 空
2. FloatFrame.Open 成功 → 首次 Display：列表已可见、无 git 标志（用户可立即浏览）
3. go func() 后台：fork git status（带 2s ctx）
4. git 返回 → state.setGitStatus(...) → screen.Redraw()
5. 主循环下一帧 Display：列表补上 git 标志（光标/滚动在 State，重画不打断浏览）
```

降级 ≠ 干等 2s 才显示：超时降级指第 3 步的 goroutine 到 2s 没回来，列表在第 2 步**早已显示**、git 列始终没填上（静默隐藏）。`screen.Redraw()` 是 micro 现成机制（线程安全）。导航切目录按此时序重来，老结果作废。

### 5.7 降级链与缓存

显示 git 状态需**同时**满足：`diffgutter=true`；当前目录在 git 仓库内；`git status` 在 **2s** 内返回。任一不满足 → 整列静默隐藏（不报错、不阻塞）。2s 是病理安全网（巨型仓库 `.git/index` 冷加载 / 当前目录本身巨多文件 / NFS 放大），正常仓库几十到一两百 ms。超时降级后**会话内该目录不再重试**（cache 记录失败结果）。实现用 `exec.CommandContext` + 2s ctx。

缓存：以 `currentDir` 为 key 缓存最近一次结果；导航切目录即重算。不做文件系统 watch（YAGNI——浏览会话短）。

### 6.1 `:file` 命令（command_neo.go）

## 8. 已知问题与范围外

| # | 项 | 说明 |
|---|----|------|
| 1 | **rename 的 `-z` 两段解析** | `-z` 下 rename 是 `XY new\0old\0`，旧路径片段会被当成独立记录。剥前缀 + 取顶层后旧片段通常映射到 stNone 不入库，副作用大多消失；dir 聚合会显示 M（工作区状态优先），可接受。彻底正确需按记录对解析，未做。 |
| 2 | **聚合优先级注释陈旧** | 源码注释说"M/D/R > A > U"，与实际枚举序（M > U > A > D > R）不符。是否调整优先级是独立产品决策，未改。 |
| 3 | **I（Ignored）状态** | 需额外 `--ignored`，开销大且 dotfile 默认隐藏时触发面小，延后未做。 |
| 4 | **symlink 的 isDir 双来源** | `entry.isDir`（列表渲染用）来自 `os.ReadDir`，**不跟随**符号链接；`metaIsDir`（末行用）来自 `os.Stat`，**跟随**符号链接。symlink→dir 的条目列表按文件画、末行按目录布局（size 留空）。已知边界差异，可接受。 |
| 5 | **左截断扩展名启发式** | 用"basename 内最后一个 `.` 之后"判定扩展名；目录名里的 `.` 不当扩展名处理。双宽/组合字符边界按 rune-safe 截断（不写半截双宽字符）。 |
| 6 | **目录条目数未显示** | 目录行末行只显 mtime（size 段留空）；可选显"N items"代替，未做。 |
| 7 | **相对时间未做** | mtime 用绝对时间（`MM-DD HH:MM` / `YYYY-MM-DD`）；相对时间（`3h ago`）会随时间陈旧、需定时刷新，未做。 |

## 9. 后续铺路

架构已为以下方向留好分离，未实施：

- **fuzzy 过滤（v1.1 优先项）**：State/View/Controller 三层已分离，v1.1 只在 State 加 `filter string` + 过滤后的条目视图，View 几乎不动，Controller 加"可打印字符 → 追加 filter"。
- **移除 `:open`**（F0 §10.3）：`:file` 稳定后执行。`:file` 纯浏览、不接路径参数；`FileCmd` 与 `OpenCmd` 在 `OpenBuffer` 落点上一致，迁移成本低（需给 keybindings / 配置里写死的 `:open <file>` 改用 `:file` 的迁移说明）。
- **第二个使用方**（如 `:save-as` 复用路径选择）：`FileSelector.Open(pane, startDir, onSelect)` 已是通用路径选择器形态，可直接复用。

明确不做：元数据列（size/mtime 作列，窄宽放不下，已挪到末行）、持续高亮 active file（光标起始锚点已够）、面包屑分段跳转（GUI 产物，TUI 复杂度高）、记忆上次目录（不存在该用户场景）、树状展开/折叠（与 `←/→` 平铺导航冲突）、`:file <path>` 接参数（纯浏览心智）。
