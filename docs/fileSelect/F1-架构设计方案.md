**状态**：已实现并测试通过。本文是 microNeo FileSelector 子系统的**唯一权威架构文档**，自包含——读完即可建立全貌心智模型：三入口模型、文件分层、FileSelector 与 FloatFrame 实现、git 子模块、关键契约与已知边界。原 F4（isNoName 模型）、F5（Size 过小）、F6（执行者/调用者分层）三份过程稿已归并删除，本文吸收其全部定论。产品交互细节以 F0 为准。

演进脉络：FileSelector 主体 → 子目录 git 透传 → 末行元数据（F1 早期稿）→ isNoName 模型 + 三入口（F4a-d）→ Size 过小 Reason 拆分（F5）→ 执行者/调用者分层（F6）。各阶段过程稿已删除，本文是合并总结，对齐当前主干代码。

## 0. 一句话

microNeo 给每个 pane 引入「出身」概念（`isNoName`）：以无名空 buffer 诞生的 pane，诞生时弹文件选择器让你挑文件（birth selector），按 `Ctrl-q` 弹选择器让你挑「换入文件 or 退出」（quit selector）；`Ctrl-o` 同样用选择器替代原生 `:open` 文本输入（browse selector）。三个入口的选择器换入全部复用 micro 原生 `OpenCmd`，modified 检查零自写。

## 1. 整体分层

```
┌──────────────────────────────────────────────────────────────┐
│  调用者层   command_neo.go                                       │
│   InitNeoCommands / InitNeoBindings（注册）                     │
│   handler：FileCmd / QuitNeo / QuitNeoCmd / 6 spawn 包装         │
│   职责：识别意图 → 委托执行者（薄委托，不内联 selector 细节）      │
├──────────────────────────────────────────────────────────────┤
│  执行者层   filemanager.go（零热键/命令注册）                     │
│   OpenBirthSelector / openBrowseSelector / openQuitSelector     │
│   职责：算起始目录 → NewFileSelector().Open(pane, dir, onSelect) │
├──────────────────────────────────────────────────────────────┤
│  选择器     FileSelector（fileselector.go）State/View/Controller │
│   布局 · 面包屑 · 文件列表 · 末行元数据 · git 标志                 │
├──────────────────────────────────────────────────────────────┤
│  容器       FloatFrame（floatframe.go）模态路由 · 画框 · resize即关│
├──────────────────────────────────────────────────────────────┤
│  底座       micro 主循环（cmd/micro/micro.go，零改动）            │
└──────────────────────────────────────────────────────────────┘
```

一次完整的「打开 → 选中 → 开文件」数据流：

```
用户按 Ctrl-o（或 Ctrl-q / spawn 新 pane）
  → handler（command_neo.go）：FileCmd / QuitNeo / spawn 包装
       └ 委托 → 执行者（filemanager.go）：openBrowseSelector / openQuitSelector / OpenBirthSelector
            ├ 算起始目录（startDirOf：AbsPath 父目录，否则 cwd）
            └ NewFileSelector().Open(pane, dir, onSelect 闭包)
                 ├ State 初始化（os.ReadDir，μs 级同步）
                 ├ computeLayout 预检（放不下 → onSelect(ReasonSize) 透明返回）
                 ├ TheFloatFrame.Open(spec)（失败 → onSelect(ReasonSize)）
                 └ 成功 → 首帧渲染 + 后台 go fetchGit(dir)
[运行期] 键事件 → FloatFrame.HandleEvent → FileSelector.handleEvent
   ↑↓ 移动 / ←→ Enter 导航 / Enter(文件) pick / . 切显隐 / Esc 关 / Ctrl-q 关
resize → FloatFrame 拦截 → Close + onSelect(ReasonResize)
选中文件 → pick → Close + onSelect(Picked{Path})
  → onSelect 闭包：pane.OpenCmd([path])（复用原生 :open，自带 modified 检查）
```

## 2. isNoName 出身模型

### 2.1 isNoName 是 sticky 出身，不是缓存

`BufPane.isNoName`（`bufpane.go:260`，bool）记录 pane **怎么诞生的**，赋值一次终身不变：

- `true` = 以无名空 buffer 诞生（如 `Ctrl-t` 开新 tab）
- `false` = 以文件诞生（如 `:vsplit foo.md`）

用户在 birth selector 选文件换入后，`isNoName` **保持 true 不清**——它是「出身」语义，不是「当前是否无名」。精确的一句话定义：**isNoName = pane 是否用 quit selector 替代原生 Quit**。它**只管 Ctrl-q**：`true` 的 pane 按 Ctrl-q 先弹 quit selector，`false` 的 pane 直接走原生 Quit。Ctrl-o 对所有 pane 一视同仁，不看 isNoName。

### 2.2 isNoNameBuf 三条件 AND（门控函数）

`isNoNameBuf`（`filemanager.go`）判定 buffer 是否「noName」，作 OpenBirthSelector 的门控：

```go
buf.AbsPath == "" && buf.Type == buffer.BTDefault && buf.Size() == 0
```

- `AbsPath==""`：无路径（非文件 buffer）
- `Type==BTDefault`：排除 Help/Log/Raw/Info/Scratch
- `Size()==0`：排除管道内容（管道有内容 → Size>0 → 非 noName）

`buffer.Size()`（buffer.go）逐行字节求和、末行不加分隔符。**勿改成 `len` 判空**（语义不同）。

## 3. 三个入口与 onSelect 分流矩阵

### 3.1 三入口总表

| 入口 | 触发 | 执行者 | 起始目录 |
|---|---|---|---|
| **birth** | spawn 包装 / micro.go 启动段 | `OpenBirthSelector(pane, dir)` | 父 pane 目录（spawn 捕获），空则 cwd |
| **browse** | `Ctrl-o` / `:file`（FileCmd） | `openBrowseSelector(h)` | `startDirOf(h)`：当前 pane AbsPath 父目录，否则 cwd |
| **quit** | `Ctrl-q`（QuitNeo，noName pane） | `openQuitSelector(h)` | `startDirOf(h)` |

三者结构一致：算起始目录 → `NewFileSelector().Open(h, dir, onSelect)`，各自手写 onSelect 闭包。

### 3.2 onSelect 分流矩阵（权威行为规约）

三个入口的 onSelect 回调按 `SelectResult` 分流。**这是三入口行为的唯一权威规约**：

| 入口 | Picked | ReasonEsc | ReasonQuit | ReasonSize | ReasonResize |
|---|---|---|---|---|---|
| **quit (Ctrl-q)** | OpenCmd | 取消（回编辑） | h.Quit() | **h.Quit()（破死锁）** | 取消（回编辑） |
| **browse (Ctrl-o)** | OpenCmd | 取消 | h.Quit() | 取消 | 取消 |
| **birth (spawn)** | OpenCmd | 取消 | h.Quit() | 取消 | 取消 |

（birth 的退出实为 `pane.Quit()`——pane 是新 pane；语义一致。）

要点：
- 所有 Picked 换入都走 `OpenCmd([]string{path})`（原生 `:open`，自带 modified 检查），且都有 R7 防御 `if h.Buf == nil { return }`（OpenCmd 首行访问 `h.Buf.Modified()`，nil 会 panic）。
- `ReasonQuit`（selector 内按 Ctrl-q）三入口统一退出 pane。
- 三入口**唯一**的行为差异：quit 入口的 `ReasonSize` 也退出（破死锁，见 §3.4），其余入口的 ReasonSize/ReasonResize 都是 no-op。
- 三份闭包**刻意各自手写、不抽公共函数**：三入口语义语境不同（birth 挑文件填空 pane、browse 换文件、quit 换文件或退出），将来各自演化概率高，保持独立更好改。`startDirOf`（纯数据变换，不含意图识别）则抽公共函数。

### 3.3 ReasonSize vs ReasonResize

拆成两个语义不同的 CloseReason，区分「没机会显示」和「显示过但被打断」：

| Reason | 含义 | 触发时机 | 用户感知 |
|---|---|---|---|
| **ReasonSize** | 打开时窗口已过小，selector 从未显示 | `computeLayout` 预检失败 / `TheFloatFrame.Open` 失败 | 按键只看到 InfoBar 提示，selector 没弹出 |
| **ReasonResize** | selector 已正常显示，运行中被窗口缩小打断 | FloatFrame `OnCancel`（强制关） | selector 浏览中消失，回编辑界面 |

另两个 CloseReason：`ReasonEsc`（用户按 Esc）、`ReasonQuit`（用户按 Ctrl-q），与 size 无关。旧的单一 `ReasonResize` 混用三种来源，导致「运行中 resize 也强制退出」的假崩溃问题——拆分后运行中 resize 不再退出。

### 3.4 ReasonSize 破死锁（场景 1）

若 quit 入口对 ReasonSize 也 no-op，窄窗口下 noName pane 会死循环：

```
Ctrl-q → QuitNeo → isNoName=true → 开 quit selector
      → computeLayout 预检失败（pane 太窄）→ 回调收到 ReasonSize
      → 若 no-op → 回编辑 → 再按 Ctrl-q → 又开不出……无限循环
```

化解：quit 入口的 ReasonSize 走 `h.Quit()`——Ctrl-q 本就是退出键，selector 只是可选中间层，开不出就执行底层退出意图。空 buffer 无修改直接退；若有未保存内容，原生 `h.Quit()` 弹存盘提示，数据不丢。只按一次即退出。

运行中 resize（ReasonResize）则不退出：用户已在浏览，缩窗是意外，强制退出像崩溃。关 selector 回编辑，放大窗口后可重试，`isNoName` 保持 true，再按 Ctrl-q 还能重开。

### 3.5 六场景（回归清单）

三入口 × 两种 size 情况 = 六场景，是改动后的回归测试矩阵：

| 场景 | 入口 | size 情况 | Reason | 调用者行为 |
|---|---|---|---|---|
| 1 | Ctrl-q（noName pane） | 打开时过小 | ReasonSize | **h.Quit()（破死锁，重点验证一次退出）** |
| 2 | Ctrl-q（noName pane） | 运行中 resize | ReasonResize | no-op（取消退出） |
| 3 | Ctrl-o | 打开时过小 | ReasonSize | no-op |
| 4 | Ctrl-o | 运行中 resize | ReasonResize | no-op |
| 5 | birth | 打开时过小 | ReasonSize | no-op（新 pane 留空 buffer） |
| 6 | birth | 运行中 resize | ReasonResize | no-op（回空 buffer） |

## 4. 文件分层：执行者 / 调用者

### 4.1 判定准则（唯一）

**一个函数是执行者还是调用者，只看它是否被绑到键位或命令上：**

- 被赋给 `BufKeyActions[...]` / `commands[...]` / 被 `MakeCommand` / `BindKey` 引用 → **调用者**，进 command_neo.go
- 只被别的函数「调用」、从不被「绑定」 → **执行者**，进 filemanager.go

### 4.2 文件职责

| 文件 | 角色 | 函数 |
|---|---|---|
| `filemanager.go` | **纯执行者** | `isNoNameBuf`（门控）/ `birthDir` / `startDirOf`（目录 helper）/ `OpenBirthSelector` / `openBrowseSelector` / `openQuitSelector` |
| `command_neo.go` | **调用者** | `InitNeoCommands` / `InitNeoBindings`（注册）/ `FileCmd` / `QuitNeo` / `QuitNeoCmd` / `ThemeCmd` / 6 spawn 包装 |

`OpenBirthSelector`、`InitNeoBindings` 保持导出大写——micro.go 启动段跨包调用（micro.go:407 / 485）。

### 4.3 干净执行者不变量（硬约束）

`filemanager.go` **不得出现**任何绑定/注册符号。review 时 grep 验证（应无任何命中）：

```
rg -n 'BufKeyActions|BindKey|MakeCommand|commands|InitNeo' internal/action/filemanager.go
```

handler 全是薄委托：command_neo.go 里的 handler 只做「识别意图 → 委托执行者」，不内联 selector 细节。例：

```go
func (h *BufPane) FileCmd(args []string)    { h.openBrowseSelector() }
func (h *BufPane) QuitNeoCmd(args []string) { h.QuitNeo() }
func (h *BufPane) QuitNeo() bool {
    if !h.isNoName { return h.Quit() }   // file-born → 原生 Quit
    h.openQuitSelector()                  // noName → 执行者
    return true
}
```

## 5. spawn 包装 + birth selector

### 5.1 为什么 birth selector 开在 spawn 包装里（不碰 Resize）

早期设计曾想把 birth selector 塞进 `finishInitialize`/`Resize`，造成 neo 逻辑污染纯几何职责。重设计后：birth selector 开在「spawn 包装」里——调完原生 spawn 之后、返回前：

```
spawn 包装 = birthDir(捕获父目录) → 原生 spawn() → MainTab().Resize()(必要时) → OpenBirthSelector(新pane)
```

`Resize` 保持**纯几何职责**，零 neo 耦合。

三种 spawn（`VSplitIndex`/`HSplitIndex`/`AddTab` 及对应 `*Cmd`）末尾都同步调 `SetActive`+`Resize`，故 spawn 包装返回时新 pane = `MainTab().CurPane()`（已 active）、BWindow 已有真实几何（预检不会 0×0 误判），`OpenBirthSelector` 可立即开。

### 5.2 六个 spawn 包装

3 个 key action + 3 个 command，覆盖所有「新建 pane」路径：

| 包装 | 替代的原生 | 快捷键 | 备注 |
|---|---|---|---|
| `neoAddTabAction` | AddTab | `Ctrl-t` | AddTab 不内部 Resize，需补 `MainTab().Resize()` |
| `neoVSplitAction` | VSplit | （无默认键，备用） | VSplit 内部已 Resize |
| `neoHSplitAction` | HSplit | （无默认键，备用） | 同上 |
| `neoNewTabCmd` | NewTabCmd | `:tab` 命令 | 补 Resize |
| `neoVSplitCmd` | VSplitCmd | `:vsplit` 命令 | |
| `neoHSplitCmd` | HSplitCmd | `:hsplit` 命令 | |

带文件参数的 `:vsplit foo` 开 file-born pane（`isNoNameBuf=false`），`OpenBirthSelector` 直接 bail。

### 5.3 启动段（micro.go:406-409）

```go
action.InitBindings()
action.InitNeoBindings()      // Ctrl-q→QuitNeo
action.InitCommands()
action.InitNeoCommands()      // :file/:quit 命令 + spawn 覆盖（最后调用，不被 clobber）
```

主窗口首个 pane 若是 noName，由 micro.go 启动段跨包调 `OpenBirthSelector(pane, "")`。

## 6. modified 检查：复用原生（零自写）

三个入口的 modified 检查**全部复用 micro 原生**，不自写 `closePrompt`/`YNPrompt`：

| 出口 | 检查机制 | 原生位置 |
|---|---|---|
| Enter 换入 | `OpenCmd` 内部 `closePrompt("Save", open)` | command.go |
| Ctrl-q 退出 | `h.Quit()` 内部 `closePrompt("Quit", ForceQuit)` | actions.go |
| Esc 取消 | 无（不丢数据） | — |

关键：**不在「开 selector 前」预检**。早期 QuitNeo/FileCmd 在开 selector 前有预检，与出口处的原生检查串联成双重提示。修复：去掉预检，让 modified 检查推迟到具体出口，每个丢数据出口恰好一次检查，全部原生。

## 7. FileSelector 主体（State / View / Controller）

三层同处 `fileselector.go`、共享 `FileSelector` 接收器，职责清晰分离：

| 层 | 职责 | 不做什么 |
|----|------|---------|
| **State** `fileSelectorState` | 当前目录、条目切片（已排序/已过滤）、cursor、topIdx（滚动）、showHidden、pickerW、listH、git 状态、元数据缓存 | 不碰屏幕、不读键 |
| **View** `display(area)` | 读 State 画面包屑 + 文件列表 + 末行元数据；左截断、Reverse 光标、配色 | 不改 State |
| **Controller** `handleEvent(ev)` | 读键 → 改 State 或触发选中/取消 | 不直接画屏（只 Redraw） |

**生命周期 / 回调契约**：每次 Open 恰好一次 onSelect 回调。选中 → `Picked{Path}`；Esc → `ReasonEsc`；Ctrl-q → `ReasonQuit`；打开放不下 → `ReasonSize`；运行中 resize → `ReasonResize`。

**Controller 键位**（resize 不会到达，FloatFrame 已拦截）：

| 键 | 行为 |
|----|------|
| `↑`/`↓` | 移动光标（不循环，clamp `[0,len]`），滚动跟随 |
| `Enter`（面包屑行） | 回父级目录 |
| `Enter`（目录条目） | 进入该目录 |
| `Enter`（文件条目） | Close + onSelect(Picked) |
| `←` | 回父级目录 |
| `→` | 光标在目录 → 进入 |
| `.` | 切换 dotfile 显隐（绕过 micro 键位绑定，modal default 分支直接匹配 rune） |
| `Esc` | Close + onSelect(ReasonEsc) |
| `Ctrl-q` | Close + onSelect(ReasonQuit) |
| 其它键 | 吞掉（modal） |

进入子目录光标落首条目；回父级落在「刚离开的目录」上（ranger/lf 约定）。每次 chdir 重置 git（新目录首帧无 git 列）并后台重查。

## 8. FloatFrame 容器（floatframe.go）

microNeo 自有的统一浮窗容器：单例 `TheFloatFrame`、模态、单浮窗（全局同时最多一个）、画框、清屏、生命周期、事件路由、「后写胜出」（主循环里 FloatFrame 最后画）。FileSelector 与 SelectPane（`:theme` 用）都是它的具体浮窗。

**`Open(spec) bool`** 经 options 模式收参（`FloatOpenSpec`：Anchor / ContentSize / Title / FrameColor / Display / HandleEvent / OnCancel / AutoExpand）。返回 false = 没开成（已有浮窗在开 / 放不下屏），调用方应 `onSelect(ReasonSize)` 透明返回。

**`AutoExpand` layout 开关**：

| `AutoExpand` | `Anchor` 语义 | 行为 | 谁用 |
|---|---|---|---|
| `true` | 展开中心点（屏幕参考点） | FloatFrame 跑 `expandAnchor` 自适应上下左右 | SelectPane（贴光标/贴 statusLine 弹） |
| `false` | **外矩形左上角（含边框）** | 直接 `fx=Anchor.X, fy=Anchor.Y`，跳过 `expandAnchor` | FileSelector（pane 派生） |

FloatFrame 只做失败前置检查（outer 是否放得下屏）+ 画框 + 生命周期 + 事件路由，**不碰 pane 边界**；FileSelector 自算 pane 派生左上角 + 尺寸，「不溢出到别的 pane」100% 由 FileSelector 把 outer 尺寸钉成 pane 的 avail / pickerW 保证。

**resize 即关 + OnCancel**：`HandleEvent` 收到 `EventResize` 时直接 `Close()` + 调 `onCancel`，不再转发——所有浮窗统一「resize 即关」。处理顺序「先关再回调」：先存 cb → Close（清 onCancel）→ 调 cb。`OnCancel` 内部**不得再调 `TheFloatFrame.Close()`**（重入），只做「调业务回调」。`Close()` 幂等。

**绘制**：HideCursor → 清外矩形 → 4 角 + 边 → title 嵌入上边框（`──<title>──...─`）→ 委托 `display(contentArea)`（contentArea = `{fx+1, fy+1, contentSize.W, contentSize.H}`，已扣边框）。具体浮窗完全不需要知道边框存在。

## 9. Layout（pane-local，computeLayout）

```
view  := pane.BWindow.GetView()
avail := view.Height - 1                      // 留 statusLine
pickerW := int(fileselectwidth * view.Width)  // 默认 0.4
pickerW clamp 到 [fsMinWidth(20), paneW]
```

预检（拒开条件）：`paneW < 20` → InfoBar 提示太窄；`avail < 10` → 太矮。换算 `FloatOpenSpec`：`Anchor={paneX,paneY}`、`ContentSize={pickerW-2, avail-2}`、`Title="Open File"`、`AutoExpand=false`、`OnCancel=func(){ onSelect(ReasonResize) }`。常量 `fsMinWidth=20`、`fsMinHeight=10`（内容区 ≥8 行 = 面包屑 + ≥6 行列表 + 末行）。

## 10. 渲染（View，display）

内容区高 = `area.H`（恒 ≥8），自上而下三段：

1. **行 0 面包屑**（`drawBreadcrumb`）：左截断全路径、恒保留「当前目录/」；根目录显 `/`；光标在此行时该行 Reverse。截断在路径段边界，截断处用 `…/` 标记。
2. **行 1..H-2 文件列表**（`drawEntry`）：视口 `[topIdx, topIdx+visibleH)`；每行 `[标记] 名[/] [git]`。标记：目录 `▸`、文件空格。名：目录带 `/` 后缀；超长**左截断保留扩展名**（basename 内最后一个 `.` 之后；rune-safe，不写半截双宽字符）。选中行 Reverse；溢出右侧 `▲`/`▼`。
3. **末行元数据**（`buildMetaLine`，§10.1）：光标条目的实时元数据，**完全不显** Enter/`.`/Esc 按键提示。

配色：目录名用 colorscheme `type` 色；文件名 `default`；git 状态按 §11.5 表；边框色传零值 → FloatFrame 用 `config.DefStyle`。写屏一律按 CJK 显示列宽推进列坐标（`runeWidth`/`stringWidth`），放不下完整双宽字符则停。

### 10.1 末行元数据（懒查 + 按条目名缓存）

guard 放 display 画末行前（`refreshMetaIfStale`）：名字没变命中缓存、零 stat；变了才 stat 一次。display 是所有渲染唯一入口，一处 guard 覆盖全部 cursor 变化路径（↑↓/chdir/toggleHidden/Open），重复 Redraw 不会变 stat 风暴。

| 光标位置 | 末行 |
|---|---|
| 文件 | `size   mtime`（如 `12.3K   01-15 14:30`） |
| 子目录 | size 段留空但保留固定宽 + mtime（与文件行同列对齐） |
| 面包屑行 | 空白 |

`humanSize`（1024 进制，恒 ≤5 列，对齐 `ls -lh`）；`formatMtime`（同年 `MM-DD HH:MM`，跨年 `YYYY-MM-DD`）。stat 失败字段显 `—`（U+2014，宽 2 列）补齐定宽，不破坏列对齐。数据来源 `os.Stat`（跟随符号链接，对齐 `ls`）；`entry.isDir` 来自 `os.ReadDir`（不跟随），见 §16.3。

## 11. git 状态子模块（fileselector_git.go）

FileSelector 里唯一一项「外来」能力——micro 全仓库无任何 git 调用经验。封装为独立、可降级的子模块，经接口解耦，任何环节失败都静默降级为「不显示状态列」，绝不阻塞浏览。

### 11.1 接口

`gitStatusCache.statusFor(dir) (map[string]statusKind, bool)`；false = 不可用（非仓库/超时/diffgutter 关），调用方降级隐藏。FileSelector 持 `gitCache` 字段（`NewFileSelector()` 注入），便于 mock。FileSelector 对 git 的全部工作就是 `os/exec` 起 git → 读 stdout → 解析两字符状态码 → 显示，**从不读文件内容、不算哈希、不碰 `.git/index`**。

### 11.2 机制：fork 系统 git + pathspec 限定

cache miss 时跑两条 git（都用 `-C <dir>`），零新依赖、版本跟用户一致、自动尊重 `.gitignore`/submodule：

```
git -C <dir> status --porcelain=v1 -z -- .   # 哪些文件有变更（pathspec 限定子树，大仓库不扫全仓）
git -C <dir> rev-parse --show-prefix          # 当前目录相对仓库根的前缀（亚毫秒）
```

`-- .` 把工作限定到当前目录子树；`-z` NUL 分隔（文件名含空格/换行也安全）。评估过 libgit2（cgo）/ go-git（纯 Go 几 MB），省的只是 1–5ms 进程启动，不划算。

### 11.3 prefix 剥离 + 取顶层分量（子目录状态透传）

status 恒输出**仓库根相对**路径（与 cwd / `-z` / `-- .` / `relativePaths` 全无关），需用 prefix 换算成相对当前目录，再取顶层分量——子目录内任意深度有变更时，当前列表里对应的顶层条目亮状态：

| 浏览位置 | prefix | porcelain 路径 | 剥后 (rel) | 顶层分量 | 列表条目 |
|---|---|---|---|---|---|
| 根 | `""` | `docs/x.md` | `docs/x.md` | `docs` | `docs/` ✓ |
| `internal/` | `internal/` | `internal/action/a.go` | `action/a.go` | `action` | `action/` ✓ |
| `internal/` | `internal/` | `internal/flat.go` | `flat.go` | `flat.go` | `flat.go` ✓ |

跨平台只切 `/`：git porcelain / rev-parse 在**所有平台（含 Windows）都发 `/`**；勿用 `\` / `filepath.Separator`（Unix 上 `\` 是合法文件名字符，双切会误伤）。

### 11.4 状态码映射与聚合优先级

porcelain 两字符 `XY`（索引 + 工作区状态），主要看第二字符（工作区），也回退查索引（如 `M ` 暂存区已改、工作区干净）：

| porcelain | 显示 |
|---|---|
| `??` | **U**（未追踪） |
| 含 `M`/`T`（` M`/`M `/`MM`/`MT`…） | **M**（已修改/类型变更） |
| `A`/`AM` | **A**（已暂存新增） |
| `D`/` D` | **D**（删除） |
| `R` | **R**（重命名） |
| 全空格 | 无标记 |

多状态聚合取最高优先级：**M > U > A > D > R**（枚举值最小 = 最高）。

### 11.5 颜色

| M | U | A | D | R |
|---|---|---|---|---|
| Yellow | Blue | Green | Red | Purple |

### 11.6 异步时序与降级

git 查询**绝不阻塞首次渲染**：

```
1. Open() 同步：State init（gitStatus 空）
2. 首帧 Display：列表可见、无 git 标志（用户可立即浏览）
3. go func() 后台：fork git status（带 2s ctx）
4. git 返回 → setGitStatus → screen.Redraw()
5. 下一帧 Display：列表补 git 标志（光标/滚动在 State，重画不打断浏览）
```

降级 ≠ 干等 2s：超时指 goroutine 到 2s 没回来，列表在第 2 步早已显示、git 列始终没填（静默隐藏）。`screen.Redraw()` 线程安全。超时降级后会话内该目录不再重试（cache 记失败结果）。缓存以 currentDir 为 key，导航切目录即重算。显示 git 需同时满足：`diffgutter=true` + 在 git 仓库内 + 2s 内返回，任一不满足整列隐藏。

## 14. 关键不变量（勿违反）

1. **isNoName 是 sticky 出身，不是缓存**。Picked 换入后不清。QuitNeo 用 isNoName（出身）判分流。只管 Ctrl-q，不看 Ctrl-o。
2. **isNoNameBuf 三条件用 AND，含 Size()==0**。勿改 `len` 判空（§2.2）。
3. **Resize 保持纯几何**。任何 neo 逻辑不得进 Resize/finishInitialize。birth selector 只在 spawn 包装里开。
4. **spawn 包装必须补 Resize（AddTab/NewTabCmd）**。VSplit/HSplit 内部已 Resize。
5. **modified 检查走原生，不自写**。Enter→OpenCmd、Ctrl-q→h.Quit、Esc→不检查。不在开 selector 前预检。
6. **ReasonSize 破死锁**：quit 入口的 ReasonSize 走 h.Quit()（窄窗口 noName pane 一次退出，§3.4）。运行中 ReasonResize 不退出。三入口行为以 §3.2 分流矩阵为准。（旧的「ReasonResize 必须退出」规约已废止。）
7. **R7 防御保留**：每个 onSelect 闭包里 `if h.Buf == nil { return }`。
8. **干净执行者**：filemanager.go 不得出现 BufKeyActions/BindKey/MakeCommand/commands/InitNeo（§4.3 grep 验证）。
9. **handler 是薄委托**：command_neo.go 里的 handler 只做识别+路由+委托，不内联 selector 细节。

## 15. 已知坑（务必注意）

**BindKey 缓存机制**（Ctrl-t 曾因此失效）：`BindKey` 把 action 名经 `BufKeyActions[a]` 解析成函数指针，**绑定时缓存进闭包**，运行时按键调缓存闭包、不再查 map。后果：只改 `BufKeyActions["AddTab"]`（spawn 覆盖）而不重新 `BindKey`，按键仍走原版。修复（command_neo.go）：改 map 后必须补 `BindKey("Ctrl-t", "AddTab", Binder["buffer"])`。VSplit/HSplit 无默认快捷键，覆盖仅备用。

**InitNeoCommands 必须最后调用**：注册时序 `InitBindings → InitNeoBindings → InitCommands → InitNeoCommands`，InitNeoCommands 最后，确保不被原生注册 clobber。

**spawn 包装的 Resize 补丁**：AddTab/NewTabCmd 不像 VSplitIndex 那样内部 Resize，spawn 包装里必须手动补 `MainTab().Resize()`，否则新 pane BWindow 几何未就绪，预检误判 0×0。

## 17. 后续可扩展点

- **fuzzy 过滤**：State/View/Controller 三层已分离，加 `filter string` + 过滤后条目视图即可，View 几乎不动，Controller 加「可打印字符 → 追加 filter」。

- **第二个使用方**（如 `:save-as` 复用路径选择）：`FileSelector.Open(pane, startDir, onSelect)` 已是通用路径选择器形态。
- **更多 spawn 路径**：micro 新增 pane 创建方式时，加对应 neo 包装（capture dir → 原生 spawn → Resize → OpenBirthSelector）。

明确不做：元数据列（size/mtime 作列，窄宽放不下，已挪末行）、面包屑分段跳转（GUI 产物）、记忆上次目录、树状展开/折叠（与 `←/→` 平铺导航冲突）、`:file <path>` 接参数（纯浏览心智）。
