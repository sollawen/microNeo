# F3 · 实施方案

对应需求：`F3-思路.md`（无文件名参数启动 → welcome 状态 + FileSelector）。

---

## 1. 目标与思路

无文件名参数启动 microNeo 时进入 **welcome 状态**：底层一个空 buffer（铺满编辑区），上面盖着现成的 FileSelector 浮窗（贴 pane 左缘、40% 宽、上下顶满）。welcome 是**根状态**：

- 选文件 → 进入编辑；
- 编辑中关掉最后一个文件 → **回到 welcome**，而非退出程序；
- 只有在 welcome 按 `Ctrl-q` 才真正退出。

welcome 不新建任何组件，100% 复用现成的 FileSelector（`:file` / `Ctrl-o` 那个）。只给它加一个 `isWelcome` 开关（控制 `Esc` / `Ctrl-q` 在 welcome 态的行为），并把它的回调契约换成带「关闭原因」的 `SelectResult`，让调用方能区分「选中文件」和各种「关掉」。

---

## 2. 状态机

只有两个状态：

```
   启动(无文件名参数)
        │
        ▼
 ┌──────────────┐  选中文件      ┌──────────────┐
 │   WELCOME    │ ────────────▶ │   EDITING    │
 │  空 buffer   │               │   (某文件)    │
 │  + 选择器    │ ◀──────────── │              │
 └──────┬───────┘  Ctrl-q       └──────┬───────┘
        │         (最后pane/tab)       │ Ctrl-q
        │                             │ (非最后)
        ▼                             ▼
     退出程序                      关 pane/tab
                                  (原生 Quit)
```

**W（welcome）**：当前 pane 是空 buffer，上盖 welcome 态 FileSelector。
- 选中文件 → 进 E。
- `Ctrl-q` → 退出程序。
- `Esc` → no-op（不关选择器、不退出）。
- resize → 选择器被强制关 → 立即重开（恒在屏）。

**E（editing）**：正常编辑某文件。
- `Ctrl-q`：非最后一个 pane/tab → 原生 `Quit`（关掉这个 split / tab）；最后一个 → 回 W。
- `Ctrl-o` / `:file` → 打开「普通态」选择器（选中→开进 pane；取消→留在 E）。行为与今天完全一致。

---

## 3. FileSelector 回调契约：SelectResult

本方案的核心改动。原来 FileSelector 的回调是 `func(*string)`（`nil`=没选、非 nil=路径），关掉的原因无法区分。改成带「关闭原因」的结构：

```go
// 结果大类
type ResultKind uint8
const (
    Picked ResultKind = iota  // 选中文件（Path 有效）
    Closed                    // 没选就关了（Reason 有效）
)

// 关闭的具体原因（仅 Kind==Closed 时有效）
type CloseReason uint8
const (
    ReasonEsc    CloseReason = iota
    ReasonQuit
    ReasonResize
)

type SelectResult struct {
    Kind   ResultKind   // Picked | Closed
    Path   string       // Kind==Picked 时：选中的文件绝对路径
    Reason CloseReason  // Kind==Closed 时：为什么关
}

// onSelect 的签名变成：func(SelectResult)
```

selector 所有关闭路径统一走 `onSelect`、带上原因：

| 关的来源 | onSelect 收到 |
|---------|--------------|
| Enter 选中文件 | `SelectResult{Kind: Picked, Path: 路径}` |
| Esc | `SelectResult{Kind: Closed, Reason: ReasonEsc}` |
| Ctrl-q | `SelectResult{Kind: Closed, Reason: ReasonQuit}` |
| resize（OnCancel） | `SelectResult{Kind: Closed, Reason: ReasonResize}` |

两个调用方各自解释：

**普通 `:file` 调用方**（只关心「选没选文件」）：
```go
onSelect: func(r SelectResult) {
    if r.Kind == Picked { openFile(r.Path) }
    // Closed（任何原因）→ 当取消，留在编辑态
}
```

**welcome 调用方**（要看关闭原因）：
```go
onSelect: func(r SelectResult) {
    switch r.Kind {
    case Picked:
        openFile(r.Path)                         // W → E
    case Closed:
        switch r.Reason {
        case ReasonQuit:   exitProgram()          // 退出
        case ReasonResize: openWelcomeSelector()  // 重开
        // ReasonEsc 不会到：welcome 态 Esc 在 selector 内是 no-op（见 §4.1c）
        }
    }
}
```

## 4. 各组件实施

### 4.1 FileSelector 改动（`internal/action/fileselector.go`）

a. 定义 `SelectResult` / `ResultKind` / `CloseReason` 类型（见 §3）。

b. `Open` 加 `isWelcome` 参数：
```go
func (fs *FileSelector) Open(pane *BufPane, startDir string,
                             onSelect func(SelectResult), isWelcome bool)
```
- 普通调用点（`:file` / `Ctrl-o`）传 `isWelcome=false`。
- welcome 调用点（`openWelcomeSelector`）传 `isWelcome=true`。

c. `handleEvent` 按 `isWelcome` 分支——**只有 Esc / Ctrl-q 两键**，其余原样：
```go
case tcell.KeyEscape:
    if !fs.isWelcome {
        fs.finish(SelectResult{Kind: Closed, Reason: ReasonEsc})  // 普通：关
    }
    // welcome：no-op（不关、不报）
case tcell.KeyCtrlQ:
    if fs.isWelcome {
        fs.finish(SelectResult{Kind: Closed, Reason: ReasonQuit}) // welcome：关→退出
    }
    // 普通态：吞掉（维持现状）
// 上下/左右/Enter/`.` 等原样复用
```

d. `pick()` 改成上报 `Picked`：
```go
fs.finish(SelectResult{Kind: Picked, Path: full})
```

e. `finish` 小帮手（统一「关 FloatFrame + 调回调」，替换原 `pick`/`cancel` 里重复的收尾）：
```go
func (fs *FileSelector) finish(r SelectResult) {
    cb := fs.onSelect
    TheFloatFrame.Close()
    if cb != nil {
        cb(r)
    }
}
```

f. `buildSpec` 的 `OnCancel`（resize 时被 FloatFrame 调）改成上报 `ReasonResize`——**与 `isWelcome` 无关、统一**（welcome 回调自己重开、普通回调自己当取消）：
```go
OnCancel: func() { fs.finish(SelectResult{Kind: Closed, Reason: ReasonResize}) },
```

这样 FileSelector 里跟 welcome 有关的只有 c 那「Esc/Ctrl-q 两键在 welcome 态怎么处理」，其余全是中性的「上报原因」。原 `cancel()` 方法可由 `finish(...)` 取代。

### 4.2 启动进 welcome（`cmd/micro/micro.go`）

`micro.go` 启动序列里加 3 个 neo 调用（都在已有 neo 调用的 `main()` 里，不改原生逻辑）：

```
b := LoadInput(args)
...
action.InitBindings()          // 原生
...
action.DetectWelcomeMode(args) // neo：置 action.WelcomeMode
action.InitNeoBindings()       // neo：按 WelcomeMode 决定是否重绑 Ctrl-q/F4/F10
...
action.InitTabs(b)
...
if action.WelcomeMode {        // neo
    action.EnterWelcome()      // 对 MainTab().CurPane() 开 welcome 态选择器
}
```

- `DetectWelcomeMode`：`action.WelcomeMode = (len(args)==0 && isatty(stdin))`。带文件名 / 管道 / `-clean` 等情况置 false，一切照旧。
- `EnterWelcome()`：`openWelcomeSelector(MainTab().CurPane(), "")`（`""` → 用 cwd）。

**顺序约束**：`DetectWelcomeMode` / `InitNeoBindings` 必须在原生 `InitBindings()` 之后；`EnterWelcome()` 必须在 `InitTabs(b)` 之后（要先有 pane 才能开选择器）。micro.go 现有顺序（`InitBindings` → `InitTabs`）天然满足，分别插在其后即可。

（若嫌 3 个触点多，可把 `DetectWelcomeMode` + `InitNeoBindings` 合成 `SetupWelcomeMode(args)`，剩 2 个触点——实施时定。）

### 4.3 QuitNeo / gotoWelcome（`internal/action/welcome_md.go`，新文件）

```go
func (h *BufPane) QuitNeo() bool {
    if !action.WelcomeMode { return h.Quit() }   // 非 welcome 会话：完全等价原生 Quit
    isLast := len(Tabs.List)==1 && len(MainTab().Panes)==1
    if !isLast { return h.Quit() }               // 多 pane/tab：照常关这一个
    // welcome 会话 + 最后一个 → 回 welcome
    if h.Buf.Modified() && !h.Buf.Shared() {
        h.closePrompt("Close", h.gotoWelcome)    // 复用原生 closePrompt（保存提示）
        return true
    }
    h.gotoWelcome()
    return true
}

func (h *BufPane) gotoWelcome() {
    dir := filepath.Dir(h.Buf.AbsPath)           // 刚关文件所在目录
    if h.Buf.AbsPath == "" { dir = "" }          // 空路径 → 用 cwd
    h.Buf.Close()
    h.OpenBuffer(buffer.NewBufferFromString("", "", buffer.BTDefault))
    h.openWelcomeSelector(dir)
}

func (h *BufPane) openWelcomeSelector(startDir string) {
    // 屏幕太小装不下选择器 → 放弃 welcome、回退原生：
    //   置 WelcomeMode=false、不开选择器，留下 micro 原生的空 buffer。
    //   此后 Ctrl-q / :quit 走 QuitNeo，但 WelcomeMode 已 false → 原生 Quit 退出。
    //   （EnterWelcome 启动 / gotoWelcome 回 welcome 共用此预检，一处拦住所有路径）
    fs := NewFileSelector()
    if _, _, ok := fs.computeLayout(h); !ok {
        action.WelcomeMode = false
        return
    }
    if startDir == "" { startDir, _ = os.Getwd() }
    fs.Open(h, startDir, func(r SelectResult) {
        switch r.Kind {
        case Picked:
            // 选中文件 → 开进 pane（W → E）
            b, err := buffer.NewBufferFromFile(r.Path, buffer.BTDefault)
            if err != nil { InfoBar.Error(err); return }
            h.OpenBuffer(b)
        case Closed:
            switch r.Reason {
            case ReasonQuit:
                exitProgram()
            case ReasonResize:
                h.openWelcomeSelector(startDir)  // 重开（同目录）
            }
        }
    }, true)  // isWelcome=true
}

func exitProgram() {
    screen.Screen.Fini()
    InfoBar.Close()
    runtime.Goexit()   // 对齐 actions.go::QuitAll 的 quit() 闭包
}
```

- `gotoWelcome` 起始目录用「刚关文件所在目录」（用户大概率想开同目录的兄弟文件）；启动 `EnterWelcome` 用 cwd。
- `exitProgram` 照抄原生 `QuitAll` 的退出路径，不动原生。

### 4.4 键位重绑（`welcome_md.go` + `command_neo.go`）

```
// welcome_md.go::InitNeoBindings()
BufKeyActions["QuitNeo"] = (*BufPane).QuitNeo   // 无条件注册（惰性）
if action.WelcomeMode {
    BindKey("Ctrl-q", "QuitNeo", Binder["buffer"])  // 仅 welcome 会话重绑
    BindKey("F4",     "QuitNeo", Binder["buffer"])  // 现状也绑 Quit，一并重绑
    BindKey("F10",    "QuitNeo", Binder["buffer"])  // 同上
}
```

`:quit` 命令在 `command_neo.go::InitNeoCommands` 里同样重指到 `QuitNeo` 逻辑，保持与 `Ctrl-q` 一致——welcome 会话最后一个 pane 回 welcome，否则原行为。

（备注：未来计划整体移除 F 键默认绑定，属独立清理任务，不属 F3。）

## 6. 边界与风险

- **resize（welcome）**：FloatFrame 强制关选择器 → 走 `OnCancel` → 上报 `ReasonResize` → welcome 回调重开。关 + 重开在一次事件处理内同步完成（`Display` 在 `DoEvent` 循环开头跑），中间无可见的空 buffer 帧。代价：重开会重读目录、光标回顶（v1 可接受；要保留光标位置属增强项）。若 resize 后终端已被拖到极小（装不下选择器），重开时的预检会**降级**而非硬重开（见下条）。
- **屏幕太小（graceful degradation）**：`openWelcomeSelector` 开选择器前先复用 `computeLayout` 预检；pane 装不下（宽 `< 20` 或矮 `< 10`）→ 置 `WelcomeMode=false`、不开选择器，留下 micro 原生的空 buffer。此后 `Ctrl-q` / `:quit` 走 `QuitNeo`，但 `WelcomeMode` 已 false → 原生 `Quit` 退出，整体行为等同「没进 welcome」。既避免「预检失败 → 重开 → 再失败」的死循环，也不必为「打开失败」新增一类关闭原因。`EnterWelcome`（启动）与 `gotoWelcome`（Ctrl-q 回 welcome）共用这个预检。
- **多 tab / split 的「最后一个」**：`isLast = len(Tabs.List)==1 && len(MainTab().Panes)==1`，同时看 tab 和 pane 两层。
- **未保存文件**：`gotoWelcome` 前用原生 `closePrompt` 弹保存提示，与正常关闭一致。
- **bindings.json 自定义 Ctrl-q**：welcome 会话里运行时重绑会覆盖它一次（welcome 本就是特殊模式）。要尊重用户配置，重绑前检查 `config.Bindings["buffer"]` 是否含该键即可——增强项。
- **普通选择器行为不变**：`isWelcome=false` 时 Esc 关、Ctrl-q 仍吞掉（维持现状），F3 不改 `Ctrl-o` 的既有体验。

## 7. 决策记录

1. **welcome 退出键**：只有 `Ctrl-q` 退出程序；`Esc` no-op（不蹲出空 buffer）；resize 重开选择器保证恒在屏。
2. **Ctrl-q 级联**：保持原生逐级收尾——多 split 关 pane、多 tab 关 tab，只有最后一个 pane/tab 才从「退出程序」改为「回 welcome」。welcome 会话下 `Ctrl-q` / `F4` / `F10` 三键一并重绑到 `QuitNeo`（现状三者都绑 `Quit`）；`Ctrl-o` 保持现状。（未来整体移除 F 键默认绑定，属独立任务。）
3. **`:quit`**：与 `Ctrl-q` 一致——welcome 会话最后一个 pane 回 welcome。
4. **回调契约**：用 `SelectResult`（`Kind: Picked/Closed` + `Path` + `Reason: Esc/Quit/Resize`）替代 `func(*string)`，关闭原因显式、可区分、编译器可查错。普通调用方只看 `Picked`，welcome 调用方按 `Reason` 分流。
5. **`isWelcome` 开关**：`Open(..., isWelcome)`。welcome 态捕获 Ctrl-q、Esc no-op；普通态维持现状。welcome 的决策（开/退/重开）全在回调里。
6. **命名与语义**：flag 叫 `action.WelcomeMode`（原 `FileMode` 易误解，已弃用）。语义 = 「welcome 模式当前是否激活」：启动按「无文件名」置 true；屏幕太小装不下选择器时降级为 false（回退原生）。
7. **gotoWelcome 起始目录**：用「刚关文件所在目录」（用户大概率想开同目录文件）；启动 `EnterWelcome` 用 cwd。
8. **不新建 welcome 组件**：复用现成 FileSelector + 空 buffer，零额外渲染代码。
9. **屏幕太小降级**：装不下选择器时直接放弃 welcome、回退 micro 原生行为（空 buffer + 原生退出键），不为「打开失败」新增一类关闭原因。降级在 `openWelcomeSelector` 的预检处发生（`EnterWelcome` 与 `gotoWelcome` 共用）。
