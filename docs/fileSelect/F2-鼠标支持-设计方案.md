# F2 · FileSelector 鼠标支持 设计方案

**状态**：草案（待评审）
**依据**：源码 `internal/action/fileselector.go`（`handleEvent` / `display` / `computeLayout` / `Open`）+ `floatframe.go`（`HandleEvent` 转发）+ `cmd/micro/micro.go:554`（事件路由）
**前置**：F1b 已落地、`:file` 三段布局（面包屑 / 文件列表 / 末行）已稳定，键盘交互完整。
**交付**：FileSelector 支持（1）左键点击 → 光标移到该行对应的条目；（2）滚轮上下 → 等同按上下键移动光标。
**原生侵入**：零。改动仅限 microNeo 自有文件 `internal/action/fileselector.go`，不改 FloatFrame、不改主循环、不改 micro 原生代码。

---

## 1. 背景与目标

### 1.1 现状

FileSelector 的交互目前**纯键盘**：`handleEvent` 只识别 `*tcell.EventKey`（↑↓Enter/←→/Esc/.）。面对几百个条目的大目录，键盘逐行移动效率低，用户自然会想用鼠标点。

### 1.2 关键发现：鼠标事件其实已经到达 FileSelector

事件路由（浮窗打开时）：

```
micro.go:554  TheFloatFrame.HandleEvent(event)     // 转发所有事件（含鼠标）
  → floatframe.go  f.handleEvent(event)            // 不区分事件类型，直接转给具体浮窗
  → fileselector.go  fs.handleEvent(event)
```

FloatFrame 的 `HandleEvent` 只拦截 `EventResize`（resize 即关），**其余所有事件（含 `*tcell.EventMouse`）原样转发**给具体浮窗。所以鼠标事件已经到了 `handleEvent` 门口。

它被丢弃的唯一原因——入口处的类型断言：

```go
ev, ok := event.(*tcell.EventKey)
if !ok { return }   // ← *tcell.EventMouse 在此被直接 return 掉
```

**结论**：不需要改事件路由、不需要碰 FloatFrame、不需要新增任何回调钩子。只需在 `fileselector.go` 内部加一个鼠标分支即可。这是 microNeo 浮窗里**第一个**接鼠标的（selectpane / notepane 尚未接），本方案同时为它们建立范式。

### 1.3 目标

- **点击**：左键点在某个条目行上 → 光标移到该条目（和键盘 ↑↓ 落位一致）。点面包屑行 → 光标落面包屑（cursor=0）。
- **滚动**：滚轮上/下 → 等同按上/下键（光标上下移一行，越界时自动滚视口）。

---

## 2. 唯一难点：屏坐标 → 行号

### 2.1 FileSelector 目前不知道自己在屏幕的哪里

`handleEvent` 拿到的是**屏坐标**（`ev.Position()` 返回整个终端的列行），而 FileSelector 只持有逻辑状态（`cursor` / `topIdx` / `listH`）。外框位置 `fx, fy` 只存在 FloatFrame 内部，FileSelector 看不到。

### 2.2 解法：Open 时存内容区原点（位置恒定，可安全缓存）

因为 `AutoExpand=false`（F1 D2），浮窗位置在 Open 时就钉死为 `anchor`，**之后永不改变**（pane resize 会触发 `EventResize` → FloatFrame 直接关浮窗，见 `floatframe.go` HandleEvent）。所以位置是「写一次、终身有效」的，可以在 Open 时算好内容区原点存下来：

| 量 | 值 | 来源 |
|----|----|------|
| 内容区左上 `contentX` | `anchor.X + 1` | FloatFrame contentArea = `{fx+1, fy+1, ...}`，`AutoExpand=false` 时 `fx=anchor.X` |
| 内容区左上 `contentY` | `anchor.Y + 1` | 同上 |
| 内容宽 | `state.pickerW` | 已存在（Open 里 `fs.state.pickerW = contentSize.W`，即去边框后的内容宽） |
| 文件列表行数 | `state.listH` | 已存在（Open 里 `fs.state.listH = contentSize.H - 2`） |

即：**宽度/列表高都已缓存在 state，只差一个屏坐标原点**。新增两个字段 `contentX, contentY` 即可。

### 2.3 行映射（`rel = my - contentY`）

内容区自上而下三段（对齐 F1 §7.5 三段布局）：

| rel | 含义 | 点击行为 |
|-----|------|---------|
| `0` | 面包屑行 | `cursor = 0` |
| `1 .. listH` | 文件列表第 `topIdx + (rel-1)` 个条目 | 命中且索引在 `[0, len(entries))` 内 → `cursor = 索引+1`；该行无条目（列表比视口短）→ 忽略 |
| `listH+1` | 末行（元数据 hint，F1d） | 忽略 |
| 其它 / 横向越界 | 边框、内容区外 | 忽略 |

横向命中判定：`contentX ≤ mx < contentX + state.pickerW`，把边框那一列排除掉。

## 3. 滚轮方向（对齐 micro 全局约定）

micro 原生把滚轮绑到滚动动作（`defaults_*.go`）：

```
MouseWheelUp   → ScrollUp      // StartLine -= n，视图上移、看到更早的内容
MouseWheelDown → ScrollDown    // StartLine += n
```

对「文件列表用光标导航」的场景，等价映射为：

| tcell 按钮 | 等同按键 | 动作 |
|-----------|---------|------|
| `WheelUp` | `KeyUp` | `moveCursor(-1)` |
| `WheelDown` | `KeyDown` | `moveCursor(+1)` |

即滚轮上 → 光标往上走、看更靠前的条目，与全编辑器一致，零学习成本。每次 1 行（严格对应「上下键」语义）。

## 4. 改动清单

全部在 `internal/action/fileselector.go`，约 +30 行、改 3 处。

### 4.1 struct 加两个字段

```go
type FileSelector struct {
	state    *fileSelectorState
	pane     *BufPane
	onSelect func(*string)
	gitCache gitStatusCache
	contentX, contentY int  // 新增：内容区屏坐标原点（鼠标命中判断用）
}
```

### 4.2 Open 里存原点

在 `fs.state.listH = ...` 之后补一行：

```go
fs.state.pickerW = contentSize.W
fs.state.listH = contentSize.H - 2
fs.contentX, fs.contentY = anchor.X+1, anchor.Y+1   // 新增
```

### 4.3 handleEvent 入口加鼠标分支

在原 `EventKey` 断言**之前**插入：

```go
func (fs *FileSelector) handleEvent(event tcell.Event) {
	s := fs.state
	if s == nil {
		return
	}
	if ev, ok := event.(*tcell.EventMouse); ok {   // 新增分支
		fs.handleMouse(ev)
		return
	}
	ev, ok := event.(*tcell.EventKey)
	if !ok {
		return
	}
	switch ev.Key() {
	// ... 不变
```

### 4.4 新增两个方法

```go
// handleMouse 处理鼠标：左键点击→光标移到该行；滚轮等同上下键。
func (fs *FileSelector) handleMouse(ev *tcell.EventMouse) {
	switch ev.Buttons() {
	case tcell.Button1:
		mx, my := ev.Position()
		fs.clickTo(mx, my)
	case tcell.WheelUp:
		fs.moveCursor(-1)
	case tcell.WheelDown:
		fs.moveCursor(+1)
	}
}

// clickTo 把点击位置映射为光标移动（仅内容区内有效；末行/边框/空行忽略）。
func (fs *FileSelector) clickTo(mx, my int) {
	s := fs.state
	if mx < fs.contentX || mx >= fs.contentX+s.pickerW {
		return // 横向越界（含边框列）
	}
	rel := my - fs.contentY // 0=面包屑, 1..listH=文件列表, listH+1=末行
	switch {
	case rel == 0:
		s.cursor = 0
	case rel >= 1 && rel <= s.listH:
		i := s.topIdx + (rel - 1)
		if i < 0 || i >= len(s.entries) {
			return // 该行无条目（列表比视口短）
		}
		s.cursor = i + 1
	default:
		return // 末行 / 越界
	}
	screen.Redraw()
}
```

### 4.5 复用现有逻辑（不重复造轮子）

- 滚轮直接复用 `moveCursor(±1)`：它已做 clamp `[0, len]`（F0 §5.1）+ `ensureVisible()` 拉视口 + `screen.Redraw()`。
- 点击落位后**无需** `ensureVisible`：被点的行按定义就在视口内。之后若按方向键，`moveCursor` 会正常滚视口。
- 末行元数据（F1d 的 `refreshMetaIfStale`）在 display 内按 `cursorEntryName()` guard，点击改了 cursor 后下一次重绘自然刷新，**无需额外触发**。

### 4.6 不改

- `entry` 结构、`listDirEntries`、排序 / 过滤、git 状态列、面包屑、文件列表 / 末行渲染。
- FloatFrame（不加 getter、不改 `FloatOpenSpec`）。
- 主循环 `micro.go`。
- 任何 micro 原生文件。

## 5. 设计取舍

| # | 决定 | 理由 |
|---|------|------|
| 1 | **点击只移光标，不打开** | 严格按需求「把光标移上去」。打开仍走 Enter；避免单击误开文件。双击打开属范围外（§7）。 |
| 2 | **按住左键拖动会连续移光标** | 拖动期间 `Button1` 持续触发，`clickTo` 逐行跟随——免费送的选择跟随特性，无害且符合直觉。 |
| 3 | **滚轮每次 1 行** | 严格对应「上下键」。若实测嫌慢，把 `moveCursor(±1)` 改成 `±N`（一处常量）即可，不影响结构。 |
| 4 | **位置在 Open 时算、不在 display 里读** | `AutoExpand=false` 保证位置终身不变；Open 时算一次是确定性的，不在 `display` 里留隐式副作用。resize 会关浮窗，不存在「位置变了但缓存没更新」的窗口。 |
| 5 | **末行不可点** | 末行是只读元数据（F1d），点它无语义，忽略。 |
| 6 | **`switch ev.Buttons()` 范式** | 与 micro 原生 `tab.go` 鼠标处理一致（`case Button1 / WheelUp / WheelDown`），风格统一、可读。 |

## 6. 验证

`make build` 后手测：

- [ ] **点文件行**：光标落到该文件（高亮反转条移过去）。
- [ ] **点子目录行**：光标落到该目录（不进入、不打开，仅定位）。
- [ ] **点面包屑行**：光标落面包屑（cursor=0，再按 Enter 回上级）。
- [ ] **点末行 / 边框 / 内容区外**：无反应，不崩。
- [ ] **点视口内但该行无条目**（列表比视口短的尾部空行）：无反应。
- [ ] **滚轮上/下**：光标逐行上下移；到顶/底 clamp 不越界（对齐 ↑↓）。
- [ ] **滚轮越过视口底**：视口自动下滚跟随（`ensureVisible` 生效），与按 ↓ 一致。
- [ ] **按住左键拖动**：光标连续跟随鼠标所在行。
- [ ] **点击后按 Enter / ← / → / `.`**：行为与键盘定位后完全一致（chdir / pick / toggleHidden 正常）。
- [ ] **点击后末行元数据**：随新光标条目刷新（F1d guard 生效）。
- [ ] **CJK 文件名**：点击命中行不受双宽字符影响（行映射只看 `my`，与列宽无关）。
- [ ] **窄 pane / 大目录**：点击 + 滚动均正常，不溢出、不覆盖边框。

## 7. 范围外（记录，不在本次做）

| # | 项 | 说明 |
|---|----|------|
| 1 | **双击打开文件** | 单击只定位；双击 = 定位 + Enter 才是文件管理器惯例。需区分 click / double-click 时序，本次不做，单击定位已满足核心需求。 |
| 2 | **滚轮加速 / 多行** | 当前严格 1 行 = 上下键。多行滚动（如 `±3`）属手感调参，留常量化开关。 |
| 3 | **横向滚轮（WheelLeft/Right）** | 文件列表无横向维度，无语义。 |
| 4 | **selectpane / notepane 接鼠标** | 本方案是 microNeo 浮窗接鼠标的首例，范式（Open 存原点 + handleEvent 加分支）可直接套用，但各浮窗按需各自实现，不在本方案范围。 |
| 5 | **点击区域 hover 高亮预览** | 当前点击即落位 + Redraw，无独立 hover 态。若要「悬停预览、点击确认」需引入 hover 状态机，本次不做。 |
