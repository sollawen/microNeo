### 1.2 `fileSelectorState`（全量，标注来源）

```go
type fileSelectorState struct {
	// —— 目录与条目（读一次，过滤靠视图）——
	currentDir  string    // 当前目录绝对路径
	allEntries  []entry   // 当前目录全部条目（含 hidden），chdir 时读一次；排序（目录优先）；fetchGit 在此填 gitChar
	showEntries []*entry  // 显示视图 = allEntries 按 showHidden 过滤后的指针子集；cursor/topIdx 索引它

	// —— git（后台 goroutine 写、UI 读，mu 保护，见 §5）——
	isRepo    bool   // 当前目录是否在 git 仓库内。独立于 gitBranch：detached/unborn 时 true 但 gitBranch=""
	gitBranch string // 分支名；"" = detached / unborn / 非仓库。记录用，显示待后续（状态栏/标题）
	// 注：每文件的 git 状态字符在 entry.gitChar 上（§1.1），不另存 map

	// —— 显示模式（字段本期留好；切换功能以后做，见 §2）——
	rightMode  rightMode // 每行右侧列：size / time / none。本期恒 rightSize
	sortMode   sortMode  // 排序键：name / size / time。本期恒 sortName
	sortDesc   bool      // 排序方向：false=正（升序）/ true=反（降序）。本期恒 false
	showHidden bool      // dotfile 显隐

	// —— 视图（不变）——
	cursor  int
	topIdx  int
	pickerW int
	listH   int

	mu sync.RWMutex
}
```

## 2. 显示模式（字段本期留好；切换功能以后做）

`rightMode` / `sortMode` / `sortDesc` 三个字段**本期加进 state**，但**循环切换的按键处理本期不实现**（§2.3）。本期固定：右列显 size（`rightSize`）、排序字母升序（`sortName` + `sortDesc=false`）。以后接切换时字段已就位，只加按键。下面是完整设计（供以后实现参考）。

### 2.1 rightMode（右侧列：size / time / none）

- `rightSize`：5 列 `humanSize`，目录留 5 空格（**本期行为**）。
- `rightTime`：11 列 `formatMtime`（`info.ModTime()`），目录留 11 空格（以后）。
- `rightNone`：无右侧列，name 延伸到 scroll 列前（以后）。

`rightW`：`rightNone→0 / rightSize→5 / rightTime→11`。布局见 §6。

### 2.2 sortMode + sortDesc（排序键 + 方向）

目录组恒先、恒按名（大小写不敏感）排；mode 只决定文件组内的排序键，方向由 `sortDesc` 统一控制（与 mode 解耦）：

- 键：`sortName`=名 / `sortSize`=size / `sortTime`=mtime。
- 方向：`sortDesc=false`（正/升序）→ name a..z、size 小..大、time 旧..新；`sortDesc=true`（反/降序）→ 反之。
- 并列（同 size / 同 mtime）一律回退字母升序，不受 `sortDesc` 影响（保稳定）。

**本期**：`sortName` + `sortDesc=false`（字母升序，即当前行为）。实现见 §7。

### 2.3 切换按键（以后实现，本期不做）

本期不接按键。以后在 modal 的 `default` 分支里加（当前已用：方向/Enter/Left/Right/Esc/CtrlQ + `.`隐藏）：

- `s` —— 循环 sortMode：`name → size → time → name`
- `S`（Shift-s）—— 翻转 sortDesc：正 ⇄ 反
- `c` —— 循环 rightMode：`size → time → none → size`

切换后按操作类型处理（见 §5.3）：
- `s` / `S`（排序）→ `sortEntries(s.allEntries, sm, desc)` 原地重排 + `rebuildShow()`（不重读、不重取）+ Redraw
- `c`（右列）→ entries 不动，直接 Redraw

## 3. git 数据层（`fileselector_git.go` 整文件重写）

原文件（`gitStatusCache` 接口 / `gitStatus` struct / `NewGitStatus` / `statusFor` / 缓存）全删，重写成两个函数 + 一个内部枚举。**无缓存**：每次 selector 打开/chdir 都重新 fork git（工作树可能被别的进程改，缓存会陈旧；后台几十 ms 不卡 UI）。

### 3.1 `getGitStatus`（fork 包装，无缓存）

```go
// getGitStatus fork git 取某目录的状态。返回 (isRepo, branch, chars)；
// isRepo=false 时 branch=""、chars=nil（非仓库/超时/diffgutter 关/git 不存在）。
func getGitStatus(dir string) (isRepo bool, branch string, chars map[string]rune) {
	if !config.GetGlobalOption("diffgutter").(bool) { // 总开关（F1 §10.4 降级链）
		return false, "", nil
	}
	ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
	defer cancel()
	out, err := exec.CommandContext(ctx, "git", "-C", dir,
		"status", "--porcelain=v1", "-z", "-b", "--ignored=traditional", "--", ".").Output()
	if err != nil {
		return false, "", nil
	}
	prefix := ""
	if p, e := exec.CommandContext(ctx, "git", "-C", dir, "rev-parse", "--show-prefix").Output(); e == nil { // 复用 status 的 2s ctx，防 goroutine 泄漏
		prefix = strings.TrimSpace(string(p))
	}
	chars, branch = parsePorcelain(out, prefix)
	return true, branch, chars
}
```

`-b` 让输出首条记录是 `## ` 分支头（见 §3.2）。导入留 `config`/`context`/`os.exec`/`strings`/`time`，删 `sync`，不加 `tcell`（颜色不进数据）。`chars`（name→rune）只是 parse 到 fetchGit 的中转，不进 state——进 state 的是各 `entry.gitChar`（§5.3）。

### 3.2 `parsePorcelain`（纯解析，含分支头捕获，可单测）

记录解析逻辑（indexSt/workSt、剥 prefix、取顶层分量、statusKind 映射、优先级聚合）**沿用现状**，只加两处：循环开头捕获 `## ` 分支头（4 刀 + detached 特判）；结尾 `agg → chars`（statusKind→rune，颜色归显示层）。

```go
func parsePorcelain(out []byte, prefix string) (chars map[string]rune, branch string) {
	chars = make(map[string]rune)
	agg := make(map[string]statusKind) // 解析内部：name → 赢家状态（优先级聚合，不外泄）
	records := strings.Split(string(out), "\x00")
	for _, rec := range records {
		// —— 分支头（## ...）——
		if strings.HasPrefix(rec, "## ") {
			b := strings.TrimPrefix(rec, "## ")
			if strings.HasPrefix(b, "HEAD (no branch)") {
				continue // detached：无分支名，留空
			}
			b = strings.TrimPrefix(b, "No commits yet on ") // unborn：去掉前缀
			if i := strings.Index(b, "..."); i >= 0 {        // "main...origin/main" → "main"
				b = b[:i]
			}
			if i := strings.IndexByte(b, ' '); i >= 0 {      // "main [ahead 1]" → "main"
				b = b[:i]
			}
			branch = b
			continue
		}
		if len(rec) < 4 { // 至少 "XY " + 路径首字符
			continue
		}
		// —— 记录解析（沿用现状）——
		indexSt, workSt := rec[0], rec[1]
		path := rec[3:]
		if path == "" {
			continue
		}
		rel := strings.TrimPrefix(path, prefix)
		var name string
		if i := strings.IndexByte(rel, '/'); i >= 0 {
			name = rel[:i] // 变更在子目录内 → 取顶层目录名
		} else {
			name = rel
		}
		if name == "." || name == "" {
			continue
		}
		var st statusKind
		if indexSt == '!' && workSt == '!' { // ignored 优先（F7）
			st = stIgnored
		} else {
			switch workSt {
			case '?':
				st = stUntracked
			case 'M', 'T':
				st = stModified
			case 'A':
				st = stAdded
			case 'D':
				st = stDeleted
			case 'R':
				st = stRenamed
			default:
				switch indexSt { // 工作区干净但索引有变化（如 "M "）
				case 'M':
					st = stModified
				case 'A':
					st = stAdded
				case 'D':
					st = stDeleted
				case 'R':
					st = stRenamed
				}
			}
		}
		if st != stNone {
			if existing, has := agg[name]; !has || st < existing { // 优先级聚合
				agg[name] = st
			}
		}
	}
	// —— agg → chars（statusKind→rune，一个 switch 内联，颜色归显示层 §6.4）——
	for name, st := range agg {
		ch := ' '
		switch st {
		case stModified:
			ch = 'M'
		case stUntracked:
			ch = 'U'
		case stAdded:
			ch = 'A'
		case stDeleted:
			ch = 'D'
		case stRenamed:
			ch = 'R'
		case stIgnored:
			ch = 'I'
		}
		chars[name] = ch
	}
	return chars, branch
}
```

`statusKind` 枚举 + `stModified/stUntracked/stAdded/stDeleted/stRenamed/stIgnored` 常量**保留**，降级为本文件内部（聚合 `st < existing` 要靠它比大小）。不出现在 state / entry / 返回值里。

## 5. state git 字段与并发

### 5.1 字段

见 §1.2 的 `isRepo` / `gitBranch`；每文件状态在 `entry.gitChar`（§1.1）。

### 5.2 并发访问（`mu` 护 allEntries + showEntries + git 字段，load-bearing）

`allEntries`（含 `gitChar`）、`showEntries`（指针）、`isRepo`/`gitBranch` 都是 fetchGit（后台写）与 display（UI 读）的共享结构，**`mu` 全保护**：

- 写（持 `Lock`）：chdir 重赋 `allEntries` + `rebuildShow()`；toggleHidden / sort 切换调 `rebuildShow()`（重排 `showEntries` 指针）；fetchGit 写 `allEntries[i].gitChar` + `isRepo`/`gitBranch`。
- 读（持 `RLock`）：display 读 `showEntries` + `isRepo`（**直读字段**——display 自己持锁，不另设 getter；锁边界见 §6.5）。

drawEntry 不再有 `gitOf`——`isRepo` 由 display 在 RLock 内**读一次**（直读 `s.isRepo`），连同可见 entry 的值副本（`*s.showEntries[i]`，含 `gitChar`）一起拷出，**RUnlock 后**才画；`gitOn bool` 作为参数传进 drawEntry。画的过程不持锁、不嵌套 RLock——彻底避开 Go RWMutex 不可重入的死锁。

（`gitBranch` 暂无读取处——记录待显示，不需要 getter。）

### 5.3 `fetchGit`（把 chars 合并进 allEntries）+ 重建 / rebuildShow 触发

```go
func (fs *FileSelector) fetchGit(dir string) {
	isRepo, branch, chars := getGitStatus(dir) // chars: name→rune
	if fs.state == nil {
		return
	}
	s := fs.state
	s.mu.Lock()
	if s.currentDir != dir { // 检查放锁内，防 TOCTOU：chdirTo 可能刚换 dir + allEntries
		s.mu.Unlock()
		return // 已导航离开，丢弃（旧 chars 不污染新 entries）
	}
	s.isRepo = isRepo
	s.gitBranch = branch
	for i := range s.allEntries { // 把 chars 按文件名合并进各 entry（含 hidden）
		if ch, ok := chars[s.allEntries[i].name]; ok {
			s.allEntries[i].gitChar = ch
		}
	}
	s.mu.Unlock()
	screen.Redraw()
}
```

**何时重读 / 重取**（关键：分清「重建 entries」vs「重排/重画」）。

| 操作 | allEntries / showEntries 怎么动 | 重读目录 | 重取 git |
|---|---|---|---|
| chdir | allEntries 换新（读新目录全部）+ rebuildShow | 是 | 是 |
| toggleHidden | **只 rebuildShow**（allEntries 不动，从它重新过滤取指针） | 否 | 否 |
| sortMode / sortDesc 切换 | sortEntries(allEntries 原地) + rebuildShow | 否 | 否 |
| rightMode 切换 | **纹丝不动**，只换布局参数 | 否 | 否 |

**关键：目录只在 chdir 时读一次**（→ `allEntries`，含 hidden），`fetchGit` 给 `allEntries` 全部条目（含 hidden 文件——`git status` 本来就报它们）填好 `gitChar`。之后 toggleHidden 只是 `rebuildShow()` 从 `allEntries` 重新过滤取指针，**露出来的 hidden 条目早已有 `gitChar`，无需重读、无需重取**。这就消掉了原本「toggleHidden 要重取 git」的问题。

- 只有 **chdir** 重建 `allEntries`（新 `gitChar=0`）才需 `go fs.fetchGit`（后台几十 ms）。代价：chdir 后 git 列短暂空白再填充——换来新鲜数据。
- **toggleHidden / sort 切换**：`allEntries` 不换、`gitChar` 全保留；前者 `rebuildShow()` 重过滤，后者 `sortEntries` 原地重排 + `rebuildShow()`。零重读、零重取。
- **rightMode 切换**：`allEntries`/`showEntries` 都不动，`s.rightMode = ...; screen.Redraw()` 即可。

`rebuildShow()`：遍历排好序的 `allEntries`，按 `showHidden` 跳过 hidden，`append(&s.allEntries[i])` 重建 `showEntries` 指针切片。chdir / toggleHidden / sort 切换三处都调它（实现见 §7）。

### 5.4 `chdirTo` / `toggleHidden` 编排（把读盘、过滤、git 串起来）

这两处是 `readDirEntries` / `rebuildShow` / `fetchGit` 的编排点（§5.3 的表落到代码），故写明新形态——关键是同步 / `go` 之分、cursor 重定位索引 `showEntries`、删掉 `setGitStatus`。

```go
func (fs *FileSelector) chdirTo(target, focusName string) {
	s := fs.state
	target = filepath.Clean(target)
	if info, err := os.Stat(target); err != nil || !info.IsDir() {
		return
	}
	s.mu.Lock()
	s.currentDir = target
	s.allEntries = readDirEntries(target, s.sortMode, s.sortDesc) // 读全部（含 hidden）+ 排序；gitChar 全 0
	s.rebuildShow()                                                // 过滤 → showEntries
	s.topIdx = 0
	s.cursor = 1
	if focusName != "" { // 回上级：落到原目录名上
		for i, e := range s.showEntries {
			if e.name == focusName { s.cursor = i + 1; break }
		}
	}
	if len(s.showEntries) == 0 { s.cursor = 0 }
	fs.ensureVisible()
	s.mu.Unlock()

	screen.Redraw()        // 首帧：git 列空
	go fs.fetchGit(target) // 异步填 allEntries[*].gitChar → 内部 Redraw（§5.3）
}

func (fs *FileSelector) toggleHidden() {
	s := fs.state
	s.mu.Lock()
	oldName := ""
	if s.cursor >= 1 && s.cursor-1 < len(s.showEntries) {
		oldName = s.showEntries[s.cursor-1].name
	}
	s.showHidden = !s.showHidden
	s.rebuildShow() // 只重过滤；allEntries + 各 gitChar 原封不动 → 不重读、不重取
	// 光标尽量留同名条目，否则 clamp 到最近可见索引（沿用现状 found/clamp，索引改 showEntries）
	fs.ensureVisible()
	s.mu.Unlock()

	screen.Redraw()
	// 无 go fetchGit：露出的 hidden 条目早有 gitChar（chdir 时 fetchGit 填过）
}
```

要点：
- **chdirTo**：`readDirEntries`（同步读盘，μs）+ `rebuildShow` + cursor → `Redraw`（首帧无 git）→ `go fetchGit`（异步）。原 `setGitStatus(nil,false)` 那行**删掉**（不再有独立 map；fetchGit 直接写 `allEntries`）。
- **toggleHidden**：**只 `rebuildShow`**，不调 `readDirEntries`、不 `go fetchGit`——这就是 §5.3 表里 toggleHidden「不重读、不重取」的落地。
- 持 `Lock` 段内**不能再拿锁**（RWMutex 不可重入，§5.2）——要 `isRepo` 就在锁内直读字段，别另开 RLock。
- `cursor` / `topIdx` 现在索引 `showEntries`（可见列表），不是 `allEntries`。

## 6. 行布局：右侧固定预留区（参数化 rightMode）

**本期范围**：`rightMode` 恒 `rightSize`，实际只走 size 分支；`rightTime`/`rightNone` 分支随切换以后接。下面是完整设计（参数化写好，以后接切换零改动）。

### 6.1 列结构（从左到右）

```
[marker] [name…] [sep?] [right?] [scroll(1)] [git(1)?]
```

- `marker`：1 列（目录 `▸` / 文件空格）。
- `name`：超长**中间截断**（`head…ext`，保留头部 + 扩展名，对齐 yazi）；目录无扩展名 → 保留头部 + `…`（`truncateNameKeepExt`）。
- `sep`：name 与 right 之间恒留 1 空格；**rightNone 时无 sep**（name 直接到 scroll 前）。
- `right`：`rightW` 列（0 / 5 / 11），区内右对齐；目录留空。
- `scroll`：1 列，**恒空格**；溢出时由 `display` 在首/末可见行覆盖 `▲/▼`。
- `git`：1 列，**仅 `isRepo` 目录预留**；脏文件画状态字符，干净文件画空格。

### 6.2 列坐标推导（右边缘 `R = x + w`）

```go
const sizeW, timeW = 5, 11
R := x + w
// gitOn 为入参：display 在 RLock 内直读 s.isRepo 后传入（§6.5）。drawEntry 自己不拿锁（§5.2 死锁坑）
var gitCol, scrollCol int
if gitOn {
	gitCol    = R - 1
	scrollCol = R - 2
} else {
	scrollCol = R - 1
}
rightW := 0
switch s.rightMode {
case rightSize:
	rightW = sizeW
case rightTime:
	rightW = timeW
}
var rightStart, nameLimit int
if rightW > 0 {
	rightStart = scrollCol - rightW      // right 区 [rightStart, scrollCol)
	nameLimit  = rightStart - (x + 1) - 1 // -1 = sep
} else {
	nameLimit = scrollCol - (x + 1)       // 无 right、无 sep
}
if nameLimit < 0 {
	nameLimit = 0
}
```

### 6.3 `drawEntry`（重排，写入顺序固定）

```go
func (fs *FileSelector) drawEntry(x, y, w int, e entry, gitOn bool, selected bool, revStyle tcell.Style) {
	s := fs.state
	const sizeW, timeW = 5, 11
	R := x + w

	var gitCol, scrollCol int
	if gitOn {
		gitCol, scrollCol = R-1, R-2
	} else {
		scrollCol = R - 1
	}
	rightW := 0
	switch s.rightMode {
	case rightSize:
		rightW = sizeW
	case rightTime:
		rightW = timeW
	}
	rightStart := scrollCol
	nameLimit := scrollCol - (x + 1)
	if rightW > 0 {
		rightStart = scrollCol - rightW
		nameLimit = rightStart - (x + 1) - 1
	}
	if nameLimit < 0 {
		nameLimit = 0
	}

	nameStyle := config.DefStyle
	if e.isDir {
		nameStyle = config.GetColor("type")
	}
	fillStyle := config.DefStyle
	gitStyle := gitCharStyle(e.gitChar) // e 是 display 在 RLock 内拷出的值副本（含 gitChar），§6.5
	if selected {
		nameStyle, fillStyle, gitStyle = revStyle, revStyle, revStyle
	}

	// marker
	col := x
	marker := ' '
	if e.isDir {
		marker = '▸'
	}
	screen.Screen.SetContent(col, y, marker, nil, nameStyle)
	col += runeWidth(marker)

	// name（截断）+ 填充到 rightStart（或 scrollCol）
	dispName := e.name
	if e.isDir {
		dispName += string(filepath.Separator)
	}
	dispName = truncateNameKeepExt(dispName, nameLimit, e.isDir)
	nameCap := rightStart
	if rightW == 0 {
		nameCap = scrollCol
	}
	for _, r := range dispName {
		rw := runeWidth(r)
		if col+rw > nameCap {
			break
		}
		screen.Screen.SetContent(col, y, r, nil, nameStyle)
		col += rw
	}
	for col < nameCap {
		screen.Screen.SetContent(col, y, ' ', nil, fillStyle)
		col++
	}

	// right 区 [rightStart, scrollCol)：目录留空；文件右对齐
	if rightW > 0 {
		for c := rightStart; c < scrollCol; c++ {
			screen.Screen.SetContent(c, y, ' ', nil, fillStyle)
		}
		if !e.isDir {
			var rs string
			if s.rightMode == rightTime {
				rs = formatMtime(e.info.ModTime())
			} else {
				rs = humanSize(e.info.Size())
			}
			sw := stringWidth(rs)
			c := scrollCol - sw
			if c < rightStart {
				c = rightStart
			}
			for _, r := range rs {
				screen.Screen.SetContent(c, y, r, nil, fillStyle)
				c += runeWidth(r)
			}
		}
	}

	// scroll 列：恒空格（display 在溢出时覆盖首/末行）
	screen.Screen.SetContent(scrollCol, y, ' ', nil, fillStyle)

	// git 列：有标志画标志，否则空格（e.gitChar=0=干净/非仓库）
	if gitOn {
		if e.gitChar != 0 {
			screen.Screen.SetContent(gitCol, y, e.gitChar, nil, gitStyle)
		} else {
			screen.Screen.SetContent(gitCol, y, ' ', nil, fillStyle)
		}
	}
}
```

要点：
- 原 F10 的 `showGit := ghas` 逐行分支**删除**：git 列是否预留绑全局 `isRepo`，干净文件那格显空格 → name 右边缘每帧对齐。
- 每列显式写入，**不再需要尾部 `for col < x+w` 兜底**。
- selected 行：marker/name/fill/right/scroll/git 全取 `revStyle`，整行 `[x, R)` 反转连续。

### 6.4 颜色映射（显示层 `gitCharStyle`，按 colorscheme 名解色）

```go
func gitCharStyle(c rune) tcell.Style {
	switch c {
	case 'M', 'R': // modified / renamed → 都算"改了"
		return config.GetColor("diff-modified")
	case 'A', 'U': // added / untracked → 都算"新的"
		return config.GetColor("diff-added")
	case 'D': // 含被删文件的目录（单个被删文件不在磁盘、不进 entries）
		return config.GetColor("diff-deleted")
	}
	return config.DefStyle // 'I'（ignored）默认
}
```

- `M`/`R` → `diff-modified`；`A`/`U` → `diff-added`；`D` → `diff-deleted`：micro 9/9 colorscheme 都有，主题感知。
- `I`：数据里有，显示默认色。
- `D` 只会出现在**含被删文件的目录**上（porcelain 取顶层分量 → 目录名命中 `'D'`）；单个被删文件不在磁盘上、不进 entries，所以文件行永不会是 `'D'`。

### 6.5 `display`（锁边界：RLock→拷值→RUnlock→画）

display 是 fetchGit（后台写 gitChar/isRepo）的唯一并发读端，**锁边界必须严格**：一把 RLock 内读 `isRepo` + 拷出可见 entry 值（含 `gitChar`），**释放锁后才画**；画的过程不持锁（否则 RWMutex 不可重入 → 死锁，§5.2）。`gitOn` 读一次，传进 drawEntry 和滚动段。

```go
func (fs *FileSelector) display(area Rect) {
	s := fs.state
	if s == nil {
		return
	}
	revStyle := config.DefStyle.Reverse(true)
	// 行 0：面包屑（同现状，bcStyle 按 cursor==0 取反）……

	listTop := area.Y + 1
	listH := area.H - 2
	if listH < 0 {
		listH = 0
	}

	// —— 锁内：读 isRepo + cursor/topIdx + 拷出可见 entry 值（gitChar 含在内）——
	s.mu.RLock()
	gitOn := s.isRepo
	total := len(s.showEntries)
	visibleH := min(total, listH)
	vis := make([]entry, visibleH)
	for vi := 0; vi < visibleH; vi++ {
		i := s.topIdx + vi
		if i >= total {
			break
		}
		vis[vi] = *s.showEntries[i] // 解引用 → 值副本，gitChar 被原子捕获
	}
	cursor, topIdx := s.cursor, s.topIdx
	s.mu.RUnlock()

	// —— 锁外：逐行画（drawEntry 拿 gitOn 入参，自己不拿锁）——
	for vi := 0; vi < visibleH; vi++ {
		fs.drawEntry(area.X, listTop+vi, area.W, vis[vi], gitOn, topIdx+vi+1 == cursor, revStyle)
	}

	// 滚动指示符（用锁内拷出的 gitOn/cursor/topIdx/total，不碰锁）
	scrollCol := area.X + area.W - 1
	if gitOn {
		scrollCol = area.X + area.W - 2
	}
	if total > visibleH && visibleH > 0 {
		topStyle := config.DefStyle
		if topIdx+1 == cursor {
			topStyle = revStyle
		}
		botStyle := config.DefStyle
		if topIdx+visibleH == cursor {
			botStyle = revStyle
		}
		if topIdx > 0 {
			screen.Screen.SetContent(scrollCol, listTop, '▲', nil, topStyle)
		}
		if topIdx+visibleH < total {
			screen.Screen.SetContent(scrollCol, listTop+visibleH-1, '▼', nil, botStyle)
		}
	}

	// 末行：perms+size+mtime（同现状 buildMetaLine）……
}
```

`cursor` / `topIdx` 索引 `showEntries`（可见列表）。指示符画在 drawEntry 写好空格的 scroll 列上，零信息损失，git on/off、rightMode 任意都成立。

## 7. 排序实现（`readDirEntries`）

**本期范围**：恒 `sortName + sortDesc=false`（字母升序，即当前行为）。下面是完整参数化设计（以后接切换零改动）。

```go
// readDirEntries 读目录【全部】条目（含 hidden，不过滤）、建 entry（gitChar=0）、
// d.Info() 失败则跳过、排序返回。只在 chdir 时调一次（不再每次 toggle 重读）。
func readDirEntries(dir string, sm sortMode, desc bool) []entry {
	dirEntries, _ := os.ReadDir(dir)
	all := make([]entry, 0, len(dirEntries))
	for _, d := range dirEntries {
		info, err := d.Info() // lstat，对齐 ls -l；失败跳过
		if err != nil {
			continue
		}
		all = append(all, entry{name: d.Name(), isDir: d.IsDir(), info: info /* gitChar=0 */})
	}
	sortEntries(all, sm, desc)
	return all
}

// sortEntries 原地排序：目录恒在文件前；目录组恒按名升序；文件组主键随 sm + desc，
// 并列恒回退字母升序（不受 desc 影响，保稳定）。readDirEntries 与 sort 切换都调它。
func sortEntries(all []entry, sm sortMode, desc bool) {
	lessName := func(a, b entry) bool { return strings.ToLower(a.name) < strings.ToLower(b.name) }
	sort.SliceStable(all, func(i, j int) bool {
		ai, aj := all[i], all[j]
		if ai.isDir != aj.isDir {
			return ai.isDir // 目录恒在文件前
		}
		if ai.isDir {
			return lessName(ai, aj) // 目录组恒按名升序
		}
		switch sm { // 文件组
		case sortSize:
			si, sj := ai.info.Size(), aj.info.Size()
			if si == sj {
				return lessName(ai, aj)
			}
			if desc {
				return si > sj
			}
			return si < sj
		case sortTime:
			ti, tj := ai.info.ModTime(), aj.info.ModTime()
			if ti.Equal(tj) {
				return lessName(ai, aj)
			}
			if desc {
				return ti.After(tj)
			}
			return ti.Before(tj)
		default: // sortName
			if strings.EqualFold(ai.name, aj.name) {
				return ai.name < aj.name // 大小写并列：原样升序
			}
			if desc {
				return lessName(aj, ai)
			}
			return lessName(ai, aj)
		}
	})
}

// rebuildShow 从排好序的 allEntries 按 showHidden 过滤，重建 showEntries（指针指向 allEntries 元素）。
// chdir / toggleHidden / sort 切换都调它。allEntries 不被重赋期间指针稳定（chdir 整体换新切片后会重调）。
func (s *fileSelectorState) rebuildShow() {
	s.showEntries = s.showEntries[:0]
	for i := range s.allEntries {
		if !s.showHidden && isHiddenName(s.allEntries[i].name) {
			continue
		}
		s.showEntries = append(s.showEntries, &s.allEntries[i])
	}
}
```

三个函数分工：`readDirEntries` 只在 chdir 时调（读目录 → `allEntries`，不再每次 toggle 重读）；`sortEntries` 在 `readDirEntries` 内和 sort 切换时都调（原地排 `allEntries`）；`rebuildShow` 在 chdir / toggleHidden / sort 切换时都调（从 `allEntries` 过滤出 `showEntries`）。sort 切换**不调 `readDirEntries`**（不重读目录），只 `sortEntries` + `rebuildShow`。chdir 后 `go fs.fetchGit`（§5.3）。
