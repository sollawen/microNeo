## 1. 问题

MD 渲染文件中，光标按 ↓ 到底边（statusLine 上方 1~3 行）时**不上滚**，光标直接飞出视口。
symmetric 方向：按 ↑ 到顶边会**偶发多余滚动**。

micro 原生路径在同样场景下正常 → 问题出在 microNeo 接入 MD 渲染后引入的断层。

---

## 2. 根因

micro 的滚动模型（`Relocate / Scroll / Diff / SLocFromLoc`）在非 softwrap 模式下假设
**1 个 buffer 行 = 1 个屏幕行**。MD 装饰行（标题下划线、表格 frame、代码块边框、HR 等）
打破了这个 1:1 假设 → `cursor.Y` 这个 buffer 行号不再代表光标的真实屏幕行号 → 底边
`cursor.Y > StartLine.Line + height-1-scrollmargin` 这个判定**系统性低估**光标的真实位置
→ 该触发上滚时不触发 → 光标跑出屏幕。

两个客观约束决定了修复方向：

- **时序坑**（`actions.go:264-271`）：按键先 `MoveCursor` 再 `Relocate` 再 `Display()`，
  → `Relocate` 时刻的映射数据是**上一帧**的。任何查"光标新位置"的方案在跳转场景失效。
- **`StartLine` 粒度**（`softwrap.go:313-317`）：非 softwrap 下 `Row` 恒 0，
  `Line` 精确到 buffer 行级；装饰行通过 `firstContentBufLine` / `lastContentBufLine`
  绑定到内容行，随内容行一并正确显示 → 滚动目标只需确定到 buffer 行级。

---

## 3. 最终方案

### 3.1 核心数据结构：2D `viewportRowmap []SLoc`

`SLoc` 是 micro 原生类型（`{Line, Row}`，复用 `StartLine` 零转换）：

```go
// bufwindow.go:41
viewportRowmap []SLoc
// viewportRowmap[i].Line = -1: 装饰行（标题下划线、表格 frame 等）
// viewportRowmap[i].Line = -2: 空白填充区域（buffer 内容不够填满 viewport）
// viewportRowmap[i].Line >= 0: 内容行；.Row = 该屏行是此 buffer 行的第几个 softwrap 段
```

为什么 2D 而不是 1D：1D 数组里装饰行 / softwrap 续行 / 内容行三者的取值都是**绑定的内容行号**
（非负），无法在数组层面区分 → 倒序反查会命中装饰行或续行末尾 → 判定偏差。2D 把
"段内第几个 softwrap 续行"作为第二维度，二元组精确匹配天然排除装饰行（`-1 ≠ c.Line`）
和错误段（`Row` 不一致）。

### 3.2 写入路径（`bufwindow_md.go`，每帧 `displayBufferMD`）

- 段开始时：分配/重置为 `SLoc{Line: -2}`。
- 光标所在段（`editMode && hasCursorInside`）：走 `renderSegmentNative`，
  segmentRow = `screenRow - lineStartVY`（天然处理 `StartLine.Row` 偏移）。
- 其他段：走 `renderSegmentMD`，按 `row.BufLine` + 同行累加推算 segRow，**装饰行存 `{Line:-1, Row:-1}`**。
- 剩余空间自动是 `-2`（无需额外写入）。

### 3.3 滚动判定（`Relocate` 分发）

`bufwindow.go:249` 加一个 `if IsMD` 分发缝（与 `LocFromVisual` 已有同构约定一致）：

```go
if w.Buf.IsMD {
    ret = w.relocateVerticalMD(c, scrollmargin, height)
} else {
    // micro 原生 Relocate（原样，仅缩进）
}
```

MD 分支 `relocateVerticalMD` 核心逻辑（`bufwindow_md.go:867`）：

```go
cursorRow, ok := w.LineToScreenRow(c.Line, c.Row)
if !ok { return nativeFallback(c, scrollmargin, height) }   // 光标跳出视口

if cursorRow > botMarginRow {
    delta := cursorRow - botMarginRow
    loc := w.viewportRowmap[delta]
    for loc.Line < 0 && delta+1 < n {                       // delta 落装饰/空白：向下找首个内容行
        delta++; loc = w.viewportRowmap[delta]
    }
    if loc.Line < 0 { return nativeFallback(...) }
    w.StartLine = loc                                        // ← 精确，含 segmentRow
    return true
}
if cursorRow < scrollmargin {
    w.StartLine = w.Scroll(c, -scrollmargin)                // 向上：段内 1:1
    return true
}
```

`relocateVerticalNativeFallback` 复刻 micro 原生垂直 Relocate（1:1 假设），供首帧 / 光标跳出视口 /
delta 落空白尾等边界场景兜底——这些场景本来就该用原生逻辑。

## 5. 已知限制

| 限制 | 来源 | 影响 |
|------|------|------|
| **跨段切换时光标偶发"消失一帧"**（用户实测：table 下方光标飞出） | "光标段原生渲染特权"在跨段时让屏行预算分配突变（涌现装饰行挤占光标行） | 多按一两次就回来 |
| **跨段进入 MD 段的首次按键**走 fallback（1:1 近似） | MD 段 segmentRow 按 MD 渲染器 softwrap 计数，与 micro 原生 `c.Row` 不一致 | 1 帧近似，下一帧自纠正 |
| **`bufwindow_md_test.go` 引用旧 API 暂未更新** | 实施 commit 已知问题 | `go test ./internal/display/` 编译失败；手测核心场景已通过 |

前两条都是"光标段原生渲染特权"方案级缺陷的衍生——要彻底解决需 prefix-sum 或
放弃该特权。本次未做，留待滚动场景再次出现可感知问题时再评估。
