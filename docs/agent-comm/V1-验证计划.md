# V1 · 验证计划：外部消息能否变成 prompt 触发 LLM

> **目的**:抛开之前所有技术选型争论,只验证一件事——**从 opencode 进程外部,能不能把一段文本送进去,让它作为 prompt 触发 LLM 回复。** 哪个 API 能跑通用哪个。
>
> 这是 aibp-opencode 的唯一命脉。验证通过前,不写任何正式实现。

---

## 已确认的事实(不用再查)

1. **TUI 模式的 opencode 不监听任何 TCP/HTTP**(实测 PID 80171,零 LISTEN)。所以**外部进程无法直接 HTTP POST**——必须靠**插件**在 opencode 进程内调用 client API。
2. **插件拿到的 `client` 是进程内的**(`PluginInput.client`),`client.session.promptAsync` / `client.tui.appendPrompt` 都在进程内可达。
3. **本地插件加载位置**:`~/.config/opencode/plugins/*.ts`(全局),opencode 启动时自动加载。**无需发包**,丢文件 + 重启即可。
4. **外部(我们的 microNeo/nc)→ 进程内插件的通道**:unix socket(插件起 socket server,外部 `nc -U` 连)。

## 待验证的两条路径

| 路径 | API | 消息是否过输入框 | 需不需要 session ID |
|------|-----|-----------------|---------------------|
| B(直接给 server) | `client.session.promptAsync({body:{id, parts:[{type:"text",text}]}})` | 否,直接进对话流 | **需要** |
| A(经输入框) | `client.tui.appendPrompt` + `submitPrompt` | 是,模拟填+回车 | 不需要 |

## 边界

- 路径 B 必须 `session.list()` 拿一个 sessionID。先选「最近更新的一个」,看是否就是用户当前看的活动 session。
- 路径 A 的 `appendPrompt` 操作「TUI 当前显示的输入框」。

## 探针插件(完整代码)

**位置**:`~/.config/opencode/plugins/aibp-probe.ts`(全局自动加载)

```typescript
// AIBP 最小探针插件 —— 验证「外部消息能否变成 prompt 触发 LLM」
//
// 用法:
//   1. 重启 opencode(加载此插件)
//   2. echo '{"text":"说一句话证明你能收到"}' | nc -U /tmp/aibp-probe.sock
//   3. 看 opencode 是否开腔回复 + 看 /tmp/aibp-probe.log

import type { PluginInput } from "@opencode-ai/plugin";
import * as net from "node:net";
import * as fs from "node:fs";

const SOCKET_PATH = "/tmp/aibp-probe.sock";
const LOG = "/tmp/aibp-probe.log";

function log(msg: string) {
  const line = `[${new Date().toISOString()}] ${msg}`;
  fs.appendFileSync(LOG, line + "\n");
  console.error("[aibp-probe]", msg);
}

export default async function plugin(input: PluginInput) {
  const client = input.client;
  log(`插件加载。有 session?=${!!client.session}，有 tui?=${!!client.tui}`);

  // 拿一个 sessionID(最近更新的)
  let sessionId: string | undefined;
  try {
    const res = await client.session.list();
    const sessions = (res as any).data ?? [];
    sessions.sort((a: any, b: any) => (b.time?.updated ?? 0) - (a.time?.updated ?? 0));
    sessionId = sessions[0]?.id;
    log(`session.list 返回 ${sessions.length} 个，选最近: ${sessionId} (${sessions[0]?.title})`);
  } catch (e: any) {
    log(`session.list 失败: ${e?.message ?? e}`);
  }

  try { fs.unlinkSync(SOCKET_PATH); } catch {}
  const server = net.createServer((conn) => {
    let buf = "";
    conn.on("data", async (chunk) => {
      buf += chunk.toString("utf8");
      const nl = buf.indexOf("\n");
      if (nl < 0) return;
      const line = buf.slice(0, nl).trim();
      buf = buf.slice(nl + 1);
      if (!line) return;
      log(`收到: ${line}`);

      let parsed: any;
      try { parsed = JSON.parse(line); } catch { parsed = { text: line }; }
      const text: string = parsed.text ?? parsed.message ?? String(parsed);

      // —— 路径 B:session.promptAsync ——
      if (sessionId) {
        try {
          const r = await client.session.promptAsync({
            body: { id: sessionId, parts: [{ type: "text", text }] } as any,
          });
          log(`✅ session.promptAsync 返回: ${JSON.stringify((r as any).data ?? (r as any).error ?? r).slice(0, 200)}`);
        } catch (e: any) {
          log(`❌ session.promptAsync 抛错: ${e?.message ?? e}`);
        }
      }

      // —— 路径 A:tui.appendPrompt + submitPrompt ——
      try {
        await client.tui.appendPrompt({ body: { text: `[probe] ${text}` } } as any);
        log(`✅ tui.appendPrompt 成功`);
        await client.tui.submitPrompt();
        log(`✅ tui.submitPrompt 成功`);
      } catch (e: any) {
        log(`❌ tui.* 抛错: ${e?.message ?? e}`);
      }

      conn.write("done\n");
      conn.end();
    });
  });

  server.listen(SOCKET_PATH, () => {
    log(`unix socket 监听: ${SOCKET_PATH}`);
  });
  return {};
}
```

3. **请你重启 opencode**(在 opencode TUI 里退出再 `opencode` 重新进)。这一步**必须你手动做**——我没法操作你的 TUI。
4. **我发测试消息** → `echo '{"text":"请回复：探针收到"}' | nc -U /tmp/aibp-probe.sock`
5. **看结果**:
   - `/tmp/aibp-probe.log` 里路径 B / A 各自的 ✅/❌
   - **你的 opencode TUI 是否真的开腔回复了**(最关键,日志只能证明 API 不报错,真正触发 LLM 得肉眼看)
6. **你告诉我**:TUI 里看到消息了吗?LLM 回复了吗?

## 判定标准

- **路径 B 成功**:TUI 对话流出现「请回复:探针收到」+ LLM 开始回复 → `session.promptAsync` 可用。
- **路径 A 成功**:TUI 输入框被填入 `[probe] 请回复...` 并自动发送 → `tui.appendPrompt+submitPrompt` 可用。
- **两者都成**:任选一个(选更贴合 aibp-pi 体验的)。
- **都失败**:回到调研报告,考虑 TUI 插件的 `TuiPromptRef.set+submit`(原调研 §7.1 骨架)。
