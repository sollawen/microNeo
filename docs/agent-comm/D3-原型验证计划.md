# 原型验证计划（D3）

> 本文档是 `D2-通信协议方案.md` 的**实施文档**：把已定稿的协议用最小成本端到端跑通，在写进 microNeo 核心之前验证风险点。
>
> **性质**：分两部分。`eabp-receivers/pi/` 是仓库长期资产（当前原型验证 M1→M2）；`proto/eabp-sender/` 是一次性原型（模拟发送端），验证完退役，但 `registry.go`/`message.go` 将来搬进 `internal/eabp/`。
>
> **实现代码的修改按项目规则先请求许可。**

---

## 一、目标

把 D2 定稿的协议全链路跑通，**完全不碰 microNeo 核心**（符合"对 micro 原生零侵入"原则）。重点验证两类靠"读文档"无法消除的风险：

1. **pi 扩展能不能起 Unix socket server 并活一整个 session**——注册/注销生命周期（D2 §8.1）从未实测。
2. **从 socket `'data'` 回调调 `pi.sendUserMessage` 能否 reentrant**——这是 D2 §6.3「带 message 立即递送」的核心，且 `sendUserMessage` 官方示例只在命令处理里用过，未在 socket 回调里验证过。

---

## 二、关键事实：pi 扩展能力摸底（方案地基）

动手前已核实 pi 文档（`extensions.md` / `tui.md` / `packages.md`）。以下每条都直接决定方案可行性：

| 需求 | pi API | 核实结论 |
|------|--------|---------|
| session 注册/注销 | `pi.on("session_start", (e, ctx))` / `pi.on("session_shutdown", (e, ctx))` | ✅ 配对钩子，`/new`/`/resume`/`/reload` 时旧实例 shutdown → 重载 → 新实例 start。注册在这里做 |
| 起 socket server | `node:net`（Node 内建，扩展可直接 import） | ✅ factory 里建 `net.Server`，`session_start` 里 `listen`，`session_shutdown` 里 `close`+`unlink` |
| 分帧 | 手写 buffer 拼接 + `indexOf("\n")` | ✅ Node 无内置逐行 socket reader，自己拼（见 §五骨架） |
| UI 提示 | `ctx.ui.notify(msg, "info"\|"warning"\|"error")` | ✅ |
| 待用上下文 widget | `ctx.ui.setWidget(key, string[]\|undefined)` | ✅ 传 `undefined` 即清除 |
| 带 message 立即递送 | `pi.sendUserMessage(text)`（**顶层 `pi`，非 `ctx`**） | ✅ "always triggers a turn"；**可在任何闭包里调，包括 socket 回调**——这是 M2 可行的根据 |
| streaming 时排队 | `pi.sendUserMessage(text, { deliverAs: "followUp" })` | ✅ 流式中不带 `deliverAs` 会**抛错**；`"followUp"` 等当前回合结束再送 |
| 判断是否 streaming | `ctx.isIdle()` | ✅ 空闲 true，据此选「立即发」还是「followUp 排队」 |
| 纯上下文隐式带上 | `before_agent_start` 返回 `{ message: { customType, content, display:true } }` | ✅ 该 message 进 session 并送 LLM；是「纯上下文」的理想落点 |

**最重要的结论**：D2 §6 的两条递送路径（带 message / 纯上下文）**pi API 全部原生支持**，无需 hack。M2 的唯一真未知收窄为「socket 回调里调顶层 `pi.sendUserMessage` 的 reentrancy」（见 §八 风险 2）。

### `eabp-receivers/pi/index.ts` 骨架（M1 形态，M2 增量加）

```typescript
import type { ExtensionAPI } from "@earendil-works/pi-coding-agent";
import * as net from "node:net";
import * as fs from "node:fs";
import * as path from "node:path";

const PROTOCOL = "eabp-1";
const WIDGET = "mnab";

export default function (pi: ExtensionAPI) {
  let server: net.Server | null = null;
  let name = "", socketPath = "", regFile = "";
  let pending: any = null;            // M2：纯上下文暂存

  pi.on("session_start", async (_e, ctx) => {
    name = `pi-${process.pid}`;        // PID 保证唯一，规避冲突处理（见 §八 风险3）
    const dir = registryDir();
    fs.mkdirSync(dir, { recursive: true, mode: 0o700 });
    socketPath = path.join(dir, `receiver-${name}.sock`);
    regFile    = path.join(dir, `receiver-${name}.json`);

    server = net.createServer((conn) => {
      let buf = "";
      conn.on("data", (chunk) => {
        buf += chunk.toString("utf8");
        let nl;
        while ((nl = buf.indexOf("\n")) >= 0) {
          handleLine(buf.slice(0, nl), ctx);
          buf = buf.slice(nl + 1);
        }
      });
    });
    await new Promise<void>((res, rej) => {
      server!.listen(socketPath, () => res());
      server!.on("error", rej);
    });
    fs.chmodSync(socketPath, 0o600);
    fs.writeFileSync(regFile, JSON.stringify({
      name, pid: process.pid, transport: "unix",
      socket: socketPath, protocol: PROTOCOL,
      startedAt: Math.floor(Date.now()/1000), cwd: process.cwd(), labels: ["default"],
    }));
  });

  pi.on("session_shutdown", async () => {
    if (server) {
      await new Promise<void>((res) => server!.close(() => res()));
      server = null;
    }
    try { fs.unlinkSync(regFile); } catch {}
    try { fs.unlinkSync(socketPath); } catch {}
    pending = null;
  });

  function handleLine(line: string, ctx: any) {
    let env: any;
    try { env = JSON.parse(line); } catch { return; }
    if (env.v !== 1 || env.type !== "context") return;   // §7.2 主版本校验
    onContext(env, ctx);                                 // 传完整信封（M1 要读 sender）
  }

  function onContext(env: any, ctx: any) {
    const p = env.payload, s = env.sender;
    // —— M1：纯 UI 展示 ——
    const from = s?.instance ?? s?.name ?? "?";
    const sel = p.selection ? ` [${p.selection.start.line}-${p.selection.end.line}]` : "";
    const msg = p.message ? ` → "${p.message}"` : "";
    ctx.ui.notify(`${from}: ${p.path}:${p.cursor?.line}${sel}${msg}`, "info");

    // —— M2 增量（替换上面五行）：见 §七 ——
  }

  // —— M2 增量：before_agent_start 注入纯上下文 ——
  // pi.on("before_agent_start", async (_e, ctx) => {
  //   if (!pending) return;
  //   const block = formatContext(pending);
  //   pending = null;
  //   ctx.ui.setWidget(WIDGET, undefined);
  //   return { message: { customType: "mnab-context", content: block, display: true } };
  // });

  // —— M2 增量：/mnab 命令 ——
  // pi.registerCommand("mnab", { ... });   // paste/send/discard/status
}
```

### `proto/eabp-sender/go/message.go`（包 eabp）

D2 §五 schema 的 Go 镜像。用指针表达可选字段（`selection`/`message`/`visible_lines`），marshal 时 nil 自动省略（`omitempty`）。

```go
package eabp

const Protocol = "eabp-1"

// Envelope — D2 §5.1 公共信封
type Envelope struct {
	V       int             `json:"v"`       // 主版本，当前=1
	Type    string          `json:"type"`    // "context"（v1仅）/ "bye"(预留)
	Sender  Sender          `json:"sender"`
	TS      float64         `json:"ts"`      // Unix 浮点秒
	Payload json.RawMessage `json:"payload"` // 原样透传；调用方按 Type 自行反序列化
}

type Sender struct {
	PID      int    `json:"pid"`
	Name     string `json:"name"`     // "microNeo"
	Instance string `json:"instance"` // 窗口/实例标识
}

// Position — D2 §5.3，被 Cursor / Selection.Start / Selection.End 复用。0-based
type Position struct {
	Line int `json:"line"`
	Col  int `json:"col"`
}

// ContextPayload — D2 §5.3 `context` 报文 payload
type ContextPayload struct {
	Path         string     `json:"path"`
	Cursor       Position   `json:"cursor"`
	Selection    *Selection `json:"selection,omitempty"`    // 无选区则省略（不是 null）
	Message      string     `json:"message,omitempty"`      // 有无决定递送路径（D2 §6.1）
	VisibleLines string     `json:"visible_lines,omitempty"`
}

type Selection struct {
	Start Position `json:"start"`
	End   Position `json:"end"`
	Text  string   `json:"text,omitempty"`
}

// MarshalLine 序列化为单行 JSON + \n（D2 §4.3 分帧）。调用方写入 socket 后关闭连接。
func (e *Envelope) MarshalLine() ([]byte, error) {
	b, err := json.Marshal(e)
	if err != nil {
		return nil, err
	}
	return append(b, '\n'), nil
}
```

### `proto/eabp-sender/go/registry.go`（包 eabp）

D2 §三 注册表的纯逻辑。`RegistryDir` 与 TS 端逐字符一致（§六）；`Discover` 做 connect 权威 + PID 旁证 + 僵尸 GC（D2 §3.5）。

```go
package eabp

import (
	"encoding/json"
	"fmt"
	"net"
	"os"
	"path/filepath"
	"strconv"
	"strings"
	"syscall"
	"time"
)

// RegistryDir — D2 §3.1。MNAB_REG_DIR 覆盖优先（调试便利，见 D3 §六）
func RegistryDir() string {
	if d := os.Getenv("MNAB_REG_DIR"); d != "" {
		return d
	}
	base := os.Getenv("XDG_RUNTIME_DIR")
	if base == "" {
		base = os.Getenv("TMPDIR")
	}
	if base == "" {
		base = "/tmp"
	}
	return filepath.Join(base, fmt.Sprintf("microneo-agent-bridge-%d", os.Getuid()))
}

// RegFile — D2 §3.2 注册文件内容
type RegFile struct {
	Name      string   `json:"name"`
	PID       int      `json:"pid"`
	Transport string   `json:"transport"` // "unix"
	Socket    string   `json:"socket"`
	Protocol  string   `json:"protocol"` // "eabp-1"
	StartedAt int64    `json:"startedAt"`
	Cwd       string   `json:"cwd,omitempty"`
	Labels    []string `json:"labels,omitempty"`
}

// Discover — 扫描注册表，返回存活 receiver。目录不存在/空目录均返回空 slice，不报错。
//   顺手 GC 僵尸（D2 §3.5）。
//   存活判据：connect socket 为权威；connect 成功 → 活。connect 失败 → 再看 PID：
//   PID 已死 → GC（删注册文件），不计入。PID 仍活 → 保留但不推荐（socket 可能暂不可用）。
func Discover() ([]RegFile, error) {
	dir := RegistryDir()
	entries, err := filepath.Glob(filepath.Join(dir, "receiver-*.json"))
	if err != nil {
		return nil, err
	}
	var live []RegFile
	for _, p := range entries {
		var rf RegFile
		if b, e := os.ReadFile(p); e != nil {
			continue
		} else if json.Unmarshal(b, &rf) != nil {
			continue
		}
		// 主版本不符 → 跳过（D2 §7.2）。字符串形如 "eabp-1"
		if major(rf.Protocol) != 1 {
			continue
		}
		if alive(rf.Socket) {
			live = append(live, rf)
			continue
		}
		if !pidAlive(rf.PID) {
			_ = os.Remove(p) // GC 僵尸
		}
	}
	return live, nil
}

// alive — connect 试探为权威判据（D2 §3.5）
func alive(socket string) bool {
	c, err := net.DialTimeout("unix", socket, 200*time.Millisecond)
	if err != nil {
		return false
	}
	_ = c.Close()
	return true
}

// pidAlive — 跨平台 kill(0) 旁证。Windows 上 syscall 用法不同，v1 仅 Unix
func pidAlive(pid int) bool {
	if pid <= 0 {
		return false
	}
	return syscall.Kill(pid, 0) == nil
}

// major — 解析 "eabp-1" 取 1（D2 §7.2）
func major(protocol string) int {
	i := strings.LastIndexByte(protocol, '-')
	if i < 0 {
		return -1
	}
	n, err := strconv.Atoi(protocol[i+1:])
	if err != nil {
		return -1
	}
	return n
}
```

package main

import (
	"encoding/json"

	"fmt"

func main() {

		fmt.Fprintln(os.Stderr, "用法: send -name <receiver> -file <path> [-msg ...] [选区flags]")
		os.Exit(2)
	}

	payload := eabp.ContextPayload{
		Path:    *path,
		Cursor:  eabp.Position{Line: *line, Col: *col},
		Message: *msg,
	}
	if *selText != "" {
		payload.Selection = &eabp.Selection{
			Start: eabp.Position{Line: *selSLine, Col: *selSCol},
			End:   eabp.Position{Line: *selELine, Col: *selECol},
			Text:  *selText,
		}
	}
	// TODO: 生产化前加 payload 合法性校验（D2 §5.3：line/col 非负，selection.end >= start 等）
	payloadJSON, _ := json.Marshal(payload)

	env := eabp.Envelope{

		TS:      float64(time.Now().UnixNano()) / 1e9,
		Payload: payloadJSON,
	}

	// 找 socket：先 Discover 验活，再按 name 取（不能盲目用注册表里的 sock，可能已僵）
	receivers, err := eabp.Discover()

## 六、registryDir 算法（两端必须完全一致）

伪码（D2 §3.1 的落地）：

```
base = $XDG_RUNTIME_DIR ? $TMPDIR ? /tmp          // 优先级回落
dir  = base + "/microneo-agent-bridge-" + $UID
```

### Go 实现（已在 `registry.go` 里，此处不重复）

### TS 实现（须与 Go 逐字符一致）

```typescript
function registryDir(): string {
  const override = process.env.MNAB_REG_DIR;
  if (override) return override;
  const base = process.env.XDG_RUNTIME_DIR
    || process.env.TMPDIR
    || "/tmp";
  return path.join(base, `microneo-agent-bridge-${process.getuid?.() ?? 0}`);
}
```

> `process.getuid()` 仅 Unix 可用。原型只跑 macOS/Linux，`?? 0` 纯兜底。

**调试便利（D2 §3.1 的可选覆盖）**：两端都认 `MNAB_REG_DIR` 环境变量——设了就用它作 dir（跳过算法），调试时指个短路径（如 `/tmp/mnab`）方便手查注册文件。默认走算法。**建议实现**——生产也用得上。

## 七、两个里程碑

### M1 — 协议骨干验证

pi 扩展收到报文后**只做 UI 展示**（`notify` + `setWidget`），**不进 LLM**。Go discover/send 跑通。

**要建**：
- `eabp-receivers/pi/index.ts`（上面骨架，`onContext` 只 notify）
- `proto/eabp-sender/go/{registry,message}.go`
- `cmd/discover`（glob `receiver-*.json` → 解析 → connect 试探 + PID → GC 僵尸）
- `cmd/send`（组信封 + payload → connect → 写一行 → close；支持 `--msg` / 纯上下文两种）

**验收清单**：
- [ ] pi 扩展：`session_start` 起 server + 写注册文件（`receiver-pi-<pid>.json`，`protocol:"eabp-1"`）

```typescript
function onContext(p: any, ctx: any) {
  if (p.message) {                       // §6.1 带 message
    const text = formatContextWithMessage(p);
    if (ctx.isIdle()) pi.sendUserMessage(text);                 // 立即触发

    ctx.ui.setWidget(WIDGET, [
      `📌 来自 ${p.sender?.instance ?? "?"} 的上下文待用`,
      `   ${p.path}:${p.cursor?.line}  /mnab paste|send|discard`,
    ]);
    // 不自动触发对话
  }
}
```

加 `before_agent_start`（见骨架注释）注入纯上下文；加 `/mnab paste|send|discard|status` 命令。

**验收清单**：
- [ ] 带 message 报文 → `sendUserMessage`，agent **真的开口回复**
- [ ] agent streaming 时收到带 message 报文 → `followUp` 排队，不打断，结束后触发

## 八、技术风险与对策

### 风险 1：socket 生命周期在 session 切换时"闪"

`/new`/`/resume`/`/reload` 会先 `session_shutdown` 旧实例再 `session_start` 新实例，socket server 在此窗口被拆重建。

**对策**：按 D2 §8.1 老实做配对清理即可。发送端 connect 探测 + GC 能自愈（连不上就 GC 重选）。
**已知限制（原型可接受）**：闪窗口内到达的报文会丢失——M1/M2 都属于"人手触发、可重发"，丢失由发送端 connect 失败提示用户重试。生产化若要消除闪烁，另议。

### 风险 2：socket `'data'` 回调里调 `pi.sendUserMessage` 的 reentrancy（**M2 核心未知**）

`sendUserMessage` 本身有文档有示例、挂在顶层 `pi`、可在任意闭包调——这些已核实。**唯一未验证**：从 `net.Socket` 的 `'data'` 事件（裸 Node 异步上下文，非 pi 事件、非 `ctx`）调它，是否触发未预期的重入或时序问题。

**对策**：这正是 M2 验收第 1 条。退路见 §七 M2 末尾（退到 `before_agent_start`，pi 事件回调 reentrancy 有保障）。

### 风险 3：多 pi 并存的 name 冲突

D2 §3.3 要求后启动者检测旧 pid。原型先做最简：默认 `name = pi-<pid>`，PID 天然唯一，几乎不冲突。冲突处理（覆盖死 pid / 自改名）留到生产前。
