## 二、数据结构设计

全部在 `internal/action/inlineinput.go` 中定义：

```go
package action

import (
    "os"

    "github.com/micro-editor/micro/v2/internal/buffer"
    "github.com/micro-editor/micro/v2/internal/display"
    "github.com/micro-editor/micro/v2/internal/views"
)

// InlineInputInstance 是全局单例，活跃时非 nil
// Tab.SetActive 守卫直接读这个变量
var InlineInputInstance *InlineInput

// InlineInput 管理一次三分屏的完整生命周期
type InlineInput struct {
    active    bool

    // 撕开前的状态快照（用于接合还原）
    origPane  *BufPane       // 原 pane 指针
    origStart display.SLoc   // 原 pane 的 StartLine（还原视口用）
    curLine   int            // 撕开时光标所在行号

    // 三分屏的三个 pane
    topPane    *BufPane
    inputPane  *BufPane
    bottomPane *BufPane

    // 临时文件
    tmpFile    string           // inputPane 对应的临时 .md 文件路径

    // 配置
    maxLines  int            // input 区最大行数（默认 10）
    onClose   func(content string)  // 关闭回调

    // 快照
    origTotalHeight int     // 撕开前 pane 的总内容高度（adjustInputHeight 用）
}

// lowestBoundary 返回 pane 中最下面的编辑边界行号。
// 多光标时取最下光标行；有选区时取选区最下边界行；两者都有时取最下的那个。
// 选区排序方式与 bufwindow_md.go:hasCursorInside 保持一致。
func lowestBoundary(bp *BufPane) int {
    lowest := bp.Cursor.Y
    for _, c := range bp.Buf.GetCursors() {
        if c.Y > lowest {
            lowest = c.Y
        }
        if c.HasSelection() {
            selStart, selEnd := c.CurSelection[0].Y, c.CurSelection[1].Y
            if selStart > selEnd {
                selStart, selEnd = selEnd, selStart
            }
            if selEnd > lowest {
                lowest = selEnd
            }
        }
    }
    return lowest
}

// maxBottomReserve 保证下半 pane 至少露出 1 行（视觉收尾，告诉用户文件没结束）
const maxBottomReserve = 1
```

### 设计说明

- **全局单例 `InlineInputInstance`**：焦点锁守卫在 `Tab.SetActive` 里直接判这个变量，最简最直接。同时保证一个 Tab 同时只有一个 InlineInput 活跃。
- **`origStart`**：保存原 pane 的 `StartLine`，接合时还原用（不只是 `curLine`，因为 softwrap 下 StartLine.Row 也有意义）。
- **`onClose` 回调**：目前签名为 `func(content string)`，把 input 内容返回给调用方。后续对接发送管线时扩展为 `func(content string, bp *BufPane)` 即可带上编辑上下文。
- **不加 `height` 字段**：input 的当前高度直接从 inputPane 的 view 取（`inputPane.GetView().Height`），不冗余存储。

## 三、生命周期函数

### 3.1 `ToggleInlineInput()` — 热键入口

```go
// ToggleInlineInput 是热键绑定的 action。
// 如果当前没有 InlineInput，打开；如果有，关闭。
func ToggleInlineInput() {
    if InlineInputInstance != nil && InlineInputInstance.active {
        InlineInputInstance.close()
    } else {
        bp := MainTab().CurPane()
        if bp == nil {
            return
        }
        ii := &InlineInput{
            origPane:  bp,
            maxLines:  10,
        }
        ii.open(bp)
    }
}
```

### 3.2 `open(bp *BufPane)` — 撕开

目标：在光标行 `curLine` 处撕开，构造三分屏。

```
撕开前：                     撕开后：

Node (leaf, buffer B)        Node (STVert, 三子节点)
                              ├── topPane    (buffer B, 非活跃)
                              ├── inputPane  (新空 buffer, 活跃)
                              └── bottomPane (buffer B, 非活跃)
```

**步骤**：

```go
func (ii *InlineInput) open(bp *BufPane) {
    // ---- 0. 前置检查 ----
    // 获取总可用高度
    totalHeight := bp.BufView().Height
    if totalHeight < maxBottomReserve + 1 {
        // 终端太小，拒绝打开
        return
    }

    // ---- 1. 快照状态 ----
    ii.curLine = lowestBoundary(bp)           // 多光标取最下光标，有选区取选区最下边界
    ii.origStart = bp.GetView().StartLine
    ii.origTotalHeight = bp.BufView().Height  // HSplit 之前快照总高度

    // ---- 2. 第一次 HSplit：bp → [bp, inputPane] ----
    // 创建临时 .md 文件给 input（真实文件，自动获得 MD 渲染管线）
    tmpDir := os.TempDir()
    tmpFile, err := os.CreateTemp(tmpDir, "micro-input-*.md")
    if err != nil {
        InfoBar.Message("InlineInput: failed to create temp file")
        return
    }
    ii.tmpFile = tmpFile.Name()
    tmpFile.Close()  // 关闭文件句柄，让 NewBufferFromFile 重新打开

    inputBuf, err := buffer.NewBufferFromFile(ii.tmpFile, buffer.BTDefault)
    if err != nil {
        os.Remove(ii.tmpFile)
        ii.tmpFile = ""
        return
    }
    ii.inputPane = bp.HSplitIndex(inputBuf, true)  // bottom=true
    // 此时 inputPane 拿到焦点（HSplitIndex 内部 SetActive）

    // ---- 3. 第二次 HSplit（仅当光标不在末行）----
    if ii.curLine < bp.Buf.LinesNum()-1 {
        bottomBuf := bp.Buf  // 共享原 buffer
        ii.bottomPane = ii.inputPane.HSplitIndex(bottomBuf, true)
    }

    // topPane 就是 bp 自身（两次 HSplit 后它还在）
    ii.topPane = bp

    // ---- 4. 精确分配三层高度 ----
    // 两次 HSplit 后 Root 有三个平级子节点，但高度被均分为 H/3。
    // 必须用 ResizeSplit 将高度调整为与 curLine 匹配的值。
    // 此步在焦点锁生效之前执行（内部会触发 SetActive，正常流转）。
    ii.layoutPanes()

    // ---- 5. 设置焦点锁 ----
    ii.active = true
    InlineInputInstance = ii
}
```

### 3.3 `close()` — 接合

目标：移除 inputPane 和 bottomPane，还原到单 pane。

```
[top, input, bottom] → Unsplit input → [top, bottom] → Unsplit bottom → [top] → flatten → 单根
```

**步骤**：

```go
func (ii *InlineInput) close() {
    content := ii.inputPane.Buf.String()

    // ---- 1. 回调 ----
    if ii.onClose != nil {
        ii.onClose(content)
    }

    // ---- 2. 防御性检查：origPane 是否仍在 tab 中 ----
    // （极端情况：InlineInput 活跃期间 Tab 被关闭/重置）
    tab := ii.inputPane.tab

    // 2a. 检查 origPane 对应的 Node 是否有效
    if tab.GetNode(ii.origPane.splitID) == nil {
        // origPane 已从布局树消失，保守处理：只清理状态，不尝试 Unsplit
        ii.active = false
        InlineInputInstance = nil
        return
    }

    // 2b. 查找 origPane 在 Panes 中的索引
    origIdx := -1
    for i, p := range tab.Panes {
        if p == ii.origPane {
            origIdx = i
            break
        }
    }

    // ---- 3. 两次 Unsplit ----
    // 注意：GetPane 找不到时返回 0（非 -1），所以必须用指针比较验证

    // 3a. 移除 bottomPane
    if ii.bottomPane != nil {
        bottomNode := tab.GetNode(ii.bottomPane.splitID)
        if bottomNode != nil {
            bottomNode.Unsplit()               // 从布局树删除
        }
        // 直接遍历指针比较，不用 GetPane（它找不到时返回 0 而非 -1）
        for idx, p := range tab.Panes {
            if p == ii.bottomPane {
                tab.RemovePane(idx)           // 从 Panes 删除
                break
            }
        }
    }

    // 3b. 移除 inputPane
    inputNode := tab.GetNode(ii.inputPane.splitID)
    if inputNode != nil {
        inputNode.Unsplit()
    }
    for idx, p := range tab.Panes {
        if p == ii.inputPane {
            tab.RemovePane(idx)
            break
        }
    }

    // ---- 4. 还原焦点到 origPane ----
    if origIdx >= 0 {
        tab.SetActive(origIdx)
    } else {
        // origPane 已不在 tab 中，设焦点到第一个 pane
        if len(tab.Panes) > 0 {
            tab.SetActive(0)
        }
    }

    // ---- 5. 还原视口 ----
    // origPane 的视口在 Unsplit 后可能被 Relocate 打乱，覆盖回来
    v := ii.origPane.GetView()
    v.StartLine = ii.origStart
    ii.origPane.SetView(v)

    // ---- 6. 清理状态 ----
    ii.active = false
    InlineInputInstance = nil

    // 删除临时 .md 文件
    if ii.tmpFile != "" {
        os.Remove(ii.tmpFile)
        ii.tmpFile = ""
    }
}
```

**安全性**：`Node.Unsplit` 只操作布局树 `children`，`Tab.RemovePane` 只操作 `Panes` 数组，两者都不碰 buffer 内容。原文件 buffer B 在整个过程中内容零修改。

### 3.4 `layoutPanes()` — 构造后精确分配三层高度

两次 HSplit 后，Root 下产生三个平级子节点（`[topPane, inputPane, bottomPane]`），但 `applyNewSize` 将高度均分为 H/3。必须在焦点锁生效之前调用此函数，用 `ResizeSplit` 将三层高度调整到正确值。

**高度分配**：
- `topHeight = curLine + 1`（显示到光标行为止）
- `inputHeight = 1`（初始 1 行，后续由 `adjustInputHeight` 动态调整）
- `bottomHeight = totalHeight - topHeight - inputHeight`（剩余空间）

```go
func (ii *InlineInput) layoutPanes() {
    tab := ii.inputPane.tab
    totalHeight := ii.origTotalHeight

    // 计算各层高度
    topHeight := ii.curLine + 1
    inputHeight := 1
    if topHeight+inputHeight > totalHeight-maxBottomReserve {
        // 空间不够，压缩 topHeight
        topHeight = totalHeight - maxBottomReserve - inputHeight
        if topHeight < 1 {
            topHeight = 1
        }
    }

    // 第一步：Resize topPane
    // topPane 是 Root.children[0]，ResizeSplit 调整 topPane 和 inputPane 之间的分配
    topNode := tab.GetNode(ii.topPane.splitID)
    if topNode != nil {
        topNode.ResizeSplit(topHeight)
        tab.Resize()
    }

    // 第二步：Resize inputPane（仅当有 bottomPane 时需要）
    // inputPane 是 Root.children[1]，ResizeSplit 调整 inputPane 和 bottomPane 之间的分配
    if ii.bottomPane != nil {
        inputNode := tab.GetNode(ii.inputPane.splitID)
        if inputNode != nil {
            inputNode.ResizeSplit(inputHeight)
            tab.Resize()
        }
    }

    // 第三步：设置各 pane 视口
    // topPane 从文件开头显示
    tv := ii.topPane.GetView()
    tv.StartLine = display.SLoc{Line: 0, Row: 0}
    ii.topPane.SetView(tv)

    // bottomPane 从 curLine+1 开始显示
    ii.reStamp()
}
```

**为什么可以连续 ResizeSplit**：micro 的 split 树支持任意数量的子节点（`[]*Node`）。`vResizeSplit` 在相邻兄弟间重新分配高度。在三个平级子节点 `[top, input, bottom]` 中：
- ResizeSplit top → 在 top 和 input 之间重新分配 → top 拿到 topHeight，input+bottom 拿到剩余
- ResizeSplit input → 在 input 和 bottom 之间重新分配 → input 拿到 inputHeight，bottom 拿到剩余
- bottom 的高度自然就是 `totalHeight - topHeight - inputHeight`

### 3.5 `reStamp()` — 下半 pane 视口覆盖

在 Relocate 干完之后，强制把下半 pane 的 `StartLine` 盖回正确位置。

```go
func (ii *InlineInput) reStamp() {
    if ii.bottomPane == nil {
        return
    }
    v := ii.bottomPane.GetView()
    v.StartLine = display.SLoc{Line: ii.curLine + 1, Row: 0}
    ii.bottomPane.SetView(v)
}
```

调用时机：
1. `layoutPanes()` 构造完成后
2. `TabList.Resize()` 完成后（见 §四.2 resize 钩子）
3. `adjustInputHeight()` 调用 `ResizeSplit` 后

### 3.6 `adjustInputHeight()` — 动态高度

input 区随内容行数自动伸缩。只调整 inputPane 和 bottomPane 之间的分配（topPane 高度不变）。

```go
func (ii *InlineInput) adjustInputHeight() {
    if !ii.active || ii.inputPane == nil {
        return
    }

    linesNum := ii.inputPane.Buf.LinesNum()

    // topPane 的实际高度（从 split 树中读取）
    topNode := ii.inputPane.tab.GetNode(ii.topPane.splitID)
    if topNode == nil {
        return
    }
    topHeight := topNode.H

    // inputPane + bottomPane 的可用空间
    availForInput := ii.origTotalHeight - topHeight - maxBottomReserve
    if availForInput < 1 {
        availForInput = 1
    }

    effectiveMax := ii.maxLines
    if availForInput < effectiveMax {
        effectiveMax = availForInput
    }

    desired := linesNum
    if desired < 1 {
        desired = 1
    }
    if desired > effectiveMax {
        desired = effectiveMax
    }

    inputNode := ii.inputPane.tab.GetNode(ii.inputPane.splitID)
    if inputNode != nil {
        currentHeight := inputNode.H
        if desired != currentHeight {
            inputNode.ResizeSplit(desired)
            ii.inputPane.tab.Resize()
            ii.reStamp()
        }
    }
}
```

**为什么用 `node.H` 而非 `GetView().Height`**：`GetView().Height` 是 bufHeight（可能扣掉了 statusline），而 `ResizeSplit` 操作的是 node 的 H。两者不一致会导致判断错误。`node.H` 是 split 树中的实际高度，与 `ResizeSplit` 的语义一致。

func (t *Tab) SetActive(i int) {

## 五、事件处理：Alt-i 切换

InlineInput 的打开和关闭**只用一个热键 `Alt-i`**，作为 toggle。不再使用 Esc 或其他键关闭。

### 为什么不用 Esc

Esc 在编辑器里承担了太多职能（退出搜索、取消操作、关闭弹窗等），绑定它关闭 InlineInput 会产生冲突和混乱。一个专用热键，语义更清晰。

### 为什么选 Alt-i

- ✅ 全平台默认键绑定中空闲
- ✅ i = input，好记
- ✅ 单手可按
- ✅ 不和终端控制字符冲突

### 实现方式

`ToggleInlineInput` 既负责打开也负责关闭：

```go
// ToggleInlineInput 是 Alt-i 绑定的 action。
// 没有活跃 InlineInput 时打开；有则关闭并发送。
func ToggleInlineInput() {
    if InlineInputInstance != nil && InlineInputInstance.active {
        InlineInputInstance.close()  // close 内部会触发 onClose 回调
    } else {
        bp := MainTab().CurPane()
        if bp == nil {
            return
        }
        ii := &InlineInput{
            origPane:  bp,
            maxLines:  10,
        }
        ii.open(bp)
    }
}
```

默认键绑定（`defaults_darwin.go` + `defaults_other.go` 的 `bufdefaults` 中加）：

```go
"Alt-i":  "ToggleInlineInput",
```

### 事件流

```
用户按 Alt-i
  → micro 键绑定系统匹配到 "Alt-i" → "ToggleInlineInput"
  → ToggleInlineInput()
  → InlineInputInstance == nil?
      是 → open()：撕开三分屏，焦点锁生效
      否 → close()：接合，触发 onClose 回调
```

**焦点锁的作用**：当 InlineInput 活跃时，用户按任何其他键（包括其他热键），只有 Alt-i 能触发 ToggleInlineInput（因为它是通过键绑定系统全局触发的，不依赖焦点在哪个 pane）。其他 pane 切换类的热键被 `Tab.SetActive` 守卫拦住。

> 注意：焦点锁不阻止 Alt-i 本身，因为 `ToggleInlineInput` 不走 `Tab.SetActive`，它是全局 action。这正好是我们想要的行为。

## 六、边界情况处理

**统一原则**：input 永远出现在光标所在行的下面。无论光标在哪一行，都是三分屏——不存在退化为二分屏的情况。

### 6.1 光标在首行（curLine == 0）

topPane 只有 1 行（第 0 行），但语义完全正确——用户的光标就在这里，撕纸就该在这里。

```
┌──────────────────────────────┐
│ topPane    原文 line 0（1行）│
├──────────────────────────────┤
│ inputPane  新 buffer         │
├──────────────────────────────┤
│ bottomPane 原文 line 1~end   │
└──────────────────────────────┘
```

### 6.2 光标在末行（curLine == buf.LinesNum()-1）

末行下面没有内容，不构造 bottomPane，退化为二分屏（topPane + inputPane）。`open()` 里用 `if curLine < buf.LinesNum()-1` 判断是否做第二次 HSplit。

```
┌──────────────────────────┐
│ topPane    原文 line 0~N  │
├──────────────────────────┤
│ inputPane  新 buffer      │
└──────────────────────────┘
```

### 6.3 终端高度过小

在 `open()` 开头检查：

```go
if totalHeight < maxBottomReserve + 1 {
    InfoBar.Message("Terminal too small for InlineInput")
    return
}
```

## 七、实现阶段与顺序

### Phase 1：骨架（能撕开、能接合）

目标：`Alt-i` 能撕开三分屏，再按 `Alt-i` 能接合回去。

| 步骤 | 内容 | 文件 |
|------|------|------|

## 九、风险与开放问题

1. **两次 HSplit 产生的是扁平三兄弟，不是嵌套**：第二次 HSplit 时，inputPane 的 node 走 `hHSplit` → 找到自己在 Root.children 的 index → 调 `Root.vHSplit(index, true)` → 走非 leaf 分支 → **直接在 Root.children 数组中插入第三个子节点**。最终 `Root.children = [topPane, inputPane, bottomPane]`，三个平级子节点。`ResizeSplit` 在相邻兄弟间分配高度，因此可以分两步精确控制三层：先 resize top（调整 top 与 input+bottom 的分配），再 resize input（调整 input 与 bottom 的分配）。`HSplitIndex` 内部调 `SetActive`，会在构造过程中触发焦点切换。焦点锁守卫在 `open()` 还没设置 `InlineInputInstance` 时不生效，所以构造过程中的 SetActive 是正常的。但需要在 `open()` 的最后一步才设置 `ii.active = true` 和 `InlineInputInstance = ii`。

2. **共享 buffer 的光标问题**：上半和下半 pane 共享同一个 buffer B。buffer 的活动光标停在 `curLine`（上半 pane 的区域内），对下半 pane 的 Relocate 会造成视口被拉回。re-stamp 解决了这个问题，但需要确认 `BufWindow.Resize` 内部的 `Relocate` 调用时机——它在 `Resize` 方法内同步调用（`bufwindow.go:109`），re-stamp 在 `TabList.Resize` 末尾调用，时序上是 Relocate 先执行、re-stamp 后覆盖，**顺序正确**。

3. **softwrap 下 SLoc.Row 的处理**：当前 re-stamp 用 `SLoc{Line: curLine+1, Row: 0}`。如果启用了 softwrap，同一行可能占多个 Row，`Row: 0` 是正确的（从该行的第一个显示行开始显示）。但需实测确认。

4. **input buffer 的类型**：使用真实临时 `.md` 文件 + `BTDefault`（`Syntax=true`），自动获得语法高亮和 MD 渲染管线。关闭时删除临时文件，不触发保存提示（文件已删除，无法保存）。

5. **动态高度的性能**：`adjustInputHeight()` 在每次 input 事件后调用，但内部仅在行数变化时才调 `ResizeSplit`，多数情况是 cheap 比较，无性能问题。

6. **`InlineInputInstance` 并发安全**：micro 的事件循环是单线程的，`InlineInputInstance` 只在主事件循环中被读写（`Tab.SetActive` 守卫、`ToggleInlineInput`、`open/close`、`adjustInputHeight`），不存在并发问题。无需加锁，但所有访问必须保持在主事件循环的串行上下文中。
