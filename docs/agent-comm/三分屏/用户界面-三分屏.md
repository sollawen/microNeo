# InlineInput 浮动窗口技术方案

## 需求概述

用户编辑文件时按快捷键，在**当前光标位置**显示一个浮动窗口：
- 窗口内可输入内容，支持 micro 的各种编辑功能
- 再按一次快捷键，窗口关闭，内容自动存盘到主缓冲区

我想像中的界面是这样的

 用户在 line10的地方按了热键
 - 于是，在这个文件的line11，撕裂开了，就像一张纸撕开了一样
 - line1-10在上半张纸上，line11-end 在下半张纸上
 - 我新输入的信息，在两张纸中间。只不过我输入的东西不送进这个文件的buffer里。
 - 我按某个键后另外处理或保存就行了。然后这个pane就消失了。两张纸又接在一起了
 - 原文件不受我输入的任何影响

原来是一个bufwindow

后来变成了三个
- 最上面的，里面是原文件，显示的是line0-10，失去焦点，变成非活跃窗口
- 最下面的，里面也是原文件，显示的是line11-end，失动焦点，变成非活跃窗口
- 中间的，是我们的input窗口，里面是新文件新buff，是当前活跃窗口。
- input窗口没有关闭的时候，不允许转移焦点到其它窗口去
