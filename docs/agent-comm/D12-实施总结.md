## 三、手动验证指南

### 3.1 快速验证（必做）

**前置**：
```bash
# 1. 编译
make build

# 2. 准备测试环境
# 终端 1：启动 2 个 pi（用于接收消息）
pi &  # Alpha
sleep 0.5
pi &  # Bravo
```

**测试步骤**：
1. 在 microNeo 中按 Alt-Enter
2. 应弹出 SelectPane，显示 "pi-Alpha" 和 "pi-Bravo"
3. 按 ↓ 选择 "pi-Bravo"，按 Enter
4. notePane 打开，上边框显示 `┌─→ pi-Bravo─────...`
5. 输入测试文字，按 Alt-Enter
6. Bravo 窗口应收到消息（不是 Alpha）

### 3.2 缓存验证

**缓存命中测试**：
1. 上述测试后，Bravo 仍在运行
2. 再次按 Alt-Enter
3. 应直接打开 notePane（不弹 SelectPane），边框仍显示 `→ pi-Bravo`

**缓存失效测试**：
1. kill Bravo（只剩 Alpha）
2. 按 Alt-Enter
3. 应弹出 SelectPane（缓存未命中），可选 Alpha

### 3.3 完整测试矩阵

使用测试脚本：

## 六、维护注意

1. **bufpane.go forwarding 是与 micro 原生代码的唯一耦合点**
   - micro 升级时如果重构 `BufPane.HandleEvent` 或 `BufPane.Display`，需要重新插入 forwarding

2. **selectedReceiver 是进程内缓存**
   - microNeo 重启后缓存丢失
   - 用户重启后第一次 Alt-Enter 必然走 Discover 全流程

3. **未来扩展**
   - notePane 内热键主动打开 SelectPane 切换 receiver
   - 需要新的焦点路由（不能复用当前 BufPane forwarding）
