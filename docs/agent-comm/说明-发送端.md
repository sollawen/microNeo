## 二、协议核心（internal/eabp/）

### 2.1 message.go

定义所有协议数据类型 + 一个序列化辅助函数。

| 类型 | 说明 |
|------|------|
| `Envelope` | 公共信封（v/type/sender/ts/payload） |
| `Sender` | 发送端标识（pid/name/instance） |
| `ContextPayload` | `context` 报文 payload（path/cursor/selection/message/visible_lines） |
| `Position` | {line, col}，**1-based** |
| `Selection` | {start, end, text} |
| `Protocol` | 常量 `"eabp-1"` |
| `Envelope.MarshalLine()` | 序列化为单行 JSON + `\n` |

**Payload 字段语义**：

```go
type ContextPayload struct {
    Path         string     `json:"path"`
    Cursor       Position   `json:"cursor"`
    Selection    *Selection `json:"selection,omitempty"`    // 无选区：省略字段（不是 null）
    Message      string     `json:"message,omitempty"`      // 有无决定递送路径
    VisibleLines string     `json:"visible_lines,omitempty"` // v1 不发送
}
```

**MarshalLine 实现**：

```go
func (e *Envelope) MarshalLine() ([]byte, error) {
    b, err := json.Marshal(e)
    if err != nil { return nil, err }
    return append(b, '\n'), nil
}
```

### 2.2 registry.go

注册表的纯逻辑。**无状态**（不持有任何 receiver 缓存），每次调用都重新扫描。

#### RegistryDir()

```go
func RegistryDir() string {
    if d := os.Getenv("MNAB_REG_DIR"); d != "" { return d }
    base := os.Getenv("XDG_RUNTIME_DIR")
    if base == "" { base = os.Getenv("TMPDIR") }
    if base == "" { base = "/tmp" }
    return filepath.Join(base, fmt.Sprintf("microneo-agent-bridge-%d", os.Getuid()))
}
```

返回注册表目录路径。算法见 `说明-EABP.md §3.1`。
`MNAB_REG_DIR` 调试覆盖：设了就用它（跳过算法）。

#### RegFile

```go
type RegFile struct {
    Name      string   `json:"name"`
    PID       int      `json:"pid"`
    Transport string   `json:"transport"`
    Socket    string   `json:"socket"`
    Protocol  string   `json:"protocol"`
    StartedAt int64    `json:"startedAt"`
    Cwd       string   `json:"cwd,omitempty"`
    Labels    []string `json:"labels,omitempty"`
}
```

注册文件内容。发送端读这些字段决定发给谁、连哪个 socket、协议是否匹配。

#### Discover()

**核心：扫描注册表 + 存活检测 + 僵尸 GC**。

```go
func Discover() ([]RegFile, error) {
    dir := RegistryDir()
    entries, err := filepath.Glob(filepath.Join(dir, "receiver-*.json"))
    if err != nil { return nil, err }
    var live []RegFile
    for _, p := range entries {
        var rf RegFile
        if b, e := os.ReadFile(p); e != nil { continue }
        else if json.Unmarshal(b, &rf) != nil { continue }
        // 协议主版本不符 → 跳过
        if major(rf.Protocol) != 1 { continue }
        if alive(rf.Socket) { live = append(live, rf); continue }
        if !pidAlive(rf.PID) { _ = os.Remove(p) }  // GC
    }
    return live, nil
}
```

**关键行为**：

- 目录不存在或空 → 返回空 slice，**不报错**
- JSON 解析失败 → 跳过该文件（不报错）
- 协议主版本不符 → 跳过（不计入 live，不 GC——留给 agent 自行处理）
- connect 成功 → 计入 live
- connect 失败 + PID 死 → **删除注册文件**（GC）
- connect 失败 + PID 活 → 跳过（保留但不用——socket 可能暂不可用）

#### alive() / pidAlive() / major()

```go
func alive(socket string) bool {
    c, err := net.DialTimeout("unix", socket, 200*time.Millisecond)
    if err != nil { return false }
    _ = c.Close()
    return true
}

func pidAlive(pid int) bool {
    if pid <= 0 { return false }
    return syscall.Kill(pid, 0) == nil
}

func major(protocol string) int {
    // 解析 "eabp-1" → 1；解析失败 → -1
}
```

**connect 为权威判据**：进程崩溃后 socket 无人 listen，connect 必失败——这是 GC 的主路径。PID 复用问题根本进不了这条判断链（复用旧 PID 的新进程不会监听旧 socket）。

## 三、notePane 发送动作（internal/action/notepane.go）

发送端 = notePane 上的 `NotePaneSend` action。**不是独立模块**，是 notePane 的一部分。

### 3.1 触发链

```
用户在 notePane 内按 Alt+Enter
  → notePaneBindings 解析（keytree.go）
  → notePaneActions["NotePaneSend"] = NotePaneSend
  → NotePaneSend(h) 执行
```

### 3.2 发送流程（实际代码）

`NotePaneSend` 实际执行 5 步（`notepane.go:402-491`）：

#### 步骤 0：空内容拦截（D10 决策 1）

```go
message := string(n.BufPane.Buf.Bytes())
if strings.TrimSpace(message) == "" {
    InfoBar.Message("✗ 内容为空,未发送")
    n.close()
    return false
}
```

- 用 `TrimSpace` 判断（纯空白也算空）
- 拦截 → InfoBar 提示 + `close()` notePane + return（**不发任何东西**）
- InfoBar 提示与 Esc 关闭行为区分：用户按 Alt-Enter 得到"未发送"反馈

> **顺带效果**：notePane 这条链路保证 `Message` 必非空，pi 端的"无 message 路径"（`setEditorText`）在 notePane 上基本死路。

#### 步骤 1：Discover receivers

```go
receivers, err := eabp.Discover()
if err != nil { InfoBar.Message("✗ discover error: " + err.Error()); return false }
if len(receivers) != 1 {
    if len(receivers) == 0 { InfoBar.Message("✗ no receiver found") }
    else                  { InfoBar.Message("✗ multiple receivers found, need exactly one") }
    return false
}
receiver := receivers[0]
```

**MVP 行为**：

- 0 个 → InfoBar 报错，不发
- 1 个 → 继续
- 多个 → InfoBar 报错，**不选**（多 receiver 选择 UI 是产品功能，后续迭代）

#### 步骤 2：构建 payload（坐标转换）

```go
// 1-based（与 LLM 工具链对齐）
cursorPos := eabp.Position{Line: n.fileCursor.Y + 1, Col: n.fileCursor.X + 1}

payload := eabp.ContextPayload{
    Path:    n.filePath,
    Cursor:  cursorPos,
    Message: message,  // 步骤 0 守卫后必非空
}

if n.fileSelection != nil {
    payload.Selection = &eabp.Selection{
        Start: eabp.Position{Line: n.fileSelection[0].Y + 1, Col: n.fileSelection[0].X + 1},
        End:   eabp.Position{Line: n.fileSelection[1].Y + 1, Col: n.fileSelection[1].X + 1},
        Text:  n.fileSelectionText,  // 见 §3.3 行数阈值
    }
}
```

**坐标转换**：

- `buffer.Loc` = `{X, Y}` = `{col, row}`（**0-based**）
- `eabp.Position` = `{Line, Col}` = `{row, col}`（**1-based**）
- 字段顺序相反 + 基址不同 → **必须显式转换**

**Selection 捕获时机**：`n.fileSelection` / `n.fileSelectionText` 在 `open()` 时一次性从主编辑器抓取（详见 §四）。**不**在发送时重新读——保证"打开时是什么，发送时就是什么"。

#### 步骤 3：序列化为 envelope

```go
payloadJSON, err := json.Marshal(payload)
if err != nil { ... }

env := eabp.Envelope{
    V:       1,
    Type:    "context",
    Sender:  eabp.Sender{PID: os.Getpid(), Name: "microNeo", Instance: "default"},
    TS:      float64(time.Now().UnixNano()) / 1e9,
    Payload: payloadJSON,
}
```

- `Instance: "default"`：**MVP 固定**。microNeo 当前无实例/窗口 ID 概念
- `TS`：Unix 浮点秒（毫秒精度）

#### 步骤 4：连接 socket + 写一行 + 关闭

```go
c, err := net.Dial("unix", receiver.Socket)
if err != nil { InfoBar.Message("✗ send failed: " + err.Error()); return false }
defer c.Close()

lineBytes, err := env.MarshalLine()
if err != nil { ... }

if _, err := c.Write(lineBytes); err != nil {
    InfoBar.Message("✗ send failed: " + err.Error())
    return false
}
// c.Close() via defer
```

**连接策略**：每次发送一次 connect → 写一行 → 关闭。无状态管理，无重试。

#### 步骤 5：成功 → close + 提示

```go
n.close()
InfoBar.Message("✓ sent to " + receiver.Name)
return false
```

- 成功 → 关闭 notePane + InfoBar 确认
- 失败（任何一步）→ **不关闭** notePane，InfoBar 报错，用户可修改后重试

### 3.3 selection 行数阈值（D10 决策 2 + 3）

**位置**：`lowestCursorScreenRow()`（`notepane.go:506`）在遍历光标抓选区时设置 `n.fileSelectionText`。

**常量**：

```go
// MaxSelectionLines 是 selection 文字内联发送的行数上限。
// 超过此行数，selection.Text 不发送（仅发 Start/End 位置），
// 接收端 fallback 到 @path :lineA-lineB 让 LLM 自己读文件。
const MaxSelectionLines = 30
```

**逻辑**：

```go
selText := string(bw.Buf.Substr(start, end))
lineSpan := end.Y - start.Y + 1
if lineSpan > MaxSelectionLines {
    selText = ""  // 文本不发送，但 start/end 位置仍发
}
n.fileSelectionText = selText
```

**关键点**：

- **位置信息**（start/end）**永远发**——`n.fileSelection` 总会被设置
- **文本**仅在 `lineSpan ≤ 30` 时发
- 接收端通过 **`Selection.Text` 是否为空**判断：空 → fallback 到 `@path :lineA-lineB` 让 LLM 读文件；非空 → 内联文字（详见 `说明-接收端.md`）
- **行数对用户可见**（编辑器能数行），字节阈值不可见

> 即便 selText 被清空，payload 仍然有 `Selection.Start/End`——pi 端 fallback 时需要这个位置。

## 四、主编辑器上下文捕获

### 4.1 捕获时机：open() 时一次性抓取

notePane 打开后主编辑器的键盘/鼠标事件被阻断（micro 主循环事件路由：notePane 开着时事件全给它），所以**打开时的状态就是发送时的状态**。

打开时一次性存到 `NotePane` 字段：

```go
type NotePane struct {
    *BufPane
    isOpen            bool
    // ...
    filePath          string       // 主编辑器 buffer.AbsPath
    fileCursor        buffer.Loc   // 最低光标（X=col, Y=line，0-based）
    fileSelection     *[2]buffer.Loc // nil = 无选区；有 = [start, end]（0-based）
    fileSelectionText string       // 选中文本（> MaxSelectionLines 时为空）
}
```

### 4.2 filePath 捕获

`n.filePath = pane.Buf.AbsPath`，放在 `reposition()` 开头（`notepane.go:283`），`pane` 从 `MainTab().CurPane()` 取。

### 4.3 fileCursor + fileSelection 捕获

`reposition()` 调用 `lowestCursorScreenRow(bw, view)` 找**最低的屏幕行**（多光标时取最下面那个；有选区时取选区底端）。**EABP 只关心最低光标**——用户视野的"我正在看的最后一行"。

`lowestCursorScreenRow` 在更新 `lowestRow` 时**顺便**记录 cursor 和 selection（`notepane.go:507-546`）：

```go
for _, cursor := range bw.Buf.GetCursors() {
    var loc buffer.Loc
    if cursor.HasSelection() {
        sel := cursor.CurSelection
        if sel[0].Y > sel[1].Y { loc = sel[0] } else { loc = sel[1] }
    } else {
        loc = cursor.Loc
    }

    screenRow := n.locToScreenRow(bw, view, loc)
    if screenRow > lowestRow {
        lowestRow = screenRow
        // 顺手记录
        n.fileCursor = loc
        if cursor.HasSelection() {
            sel := cursor.CurSelection
            start, end := sel[0], sel[1]
            if start.GreaterThan(end) { start, end = end, start }  // 归一化
            n.fileSelection = &[2]buffer.Loc{start, end}
            selText := string(bw.Buf.Substr(start, end))
            lineSpan := end.Y - start.Y + 1
            if lineSpan > MaxSelectionLines { selText = "" }
            n.fileSelectionText = selText
        } else {
            n.fileSelection = nil
            n.fileSelectionText = ""
        }
    }
}
```

**归一化**：`CurSelection` 可能是反向选择（从右往左），归一化为 `start ≤ end`，方便 EABP payload 也按 `start ≤ end` 发。

**MD 模式特殊处理**：`locToScreenRow`（`notepane.go:494-505`）调用 `bw.LineToScreenRow` 处理表格/代码块等 frame 装饰行的偏移——MD 模式下，buffer 的 row N 不等于屏幕 row N。

### 4.4 reposition() 的重复调用

`reposition()` 在两个时机被调：

1. `open()` 时——算初始位置、捕获主编辑器上下文
2. `HandleEvent` 收到 `EventResize` 时——**已开态下**重新定位（不重抓上下文，因为上下文是"打开时"的状态）

重复执行 `lowestCursorScreenRow` 中的 `lineSpan > MaxSelectionLines` 阈值判断**不影响正确性**——start/end 是 open 时捕获的，不随 resize 变。

## 五、Send 失败的回退行为

按 `NotePaneSend` 实际实现：

| 失败点 | 行为 | notePane 是否关闭 |
|--------|------|-------------------|
| 空内容拦截 | InfoBar 提示 + return | ✅ **关闭**（D10 决策 1） |
| Discover 失败 | InfoBar 报错 + return | ❌ 不关 |
| 0 个 receiver | InfoBar 报错 + return | ❌ 不关 |
| 多个 receiver | InfoBar 报错 + return（MVP） | ❌ 不关 |
| Payload 序列化失败 | InfoBar 报错 + return | ❌ 不关 |
| Dial 失败 | InfoBar 报错 + return | ❌ 不关 |
| Write 失败 | InfoBar 报错 + return | ❌ 不关 |
| 成功 | InfoBar 确认 + close() | ✅ **关闭** |

> 只有"空内容"和"成功"两个分支会关闭 notePane。其他失败都保持打开，用户可修改后重试（注意：空内容已 close，所以"修改后重试"不存在——但空内容本来就无意义）。

## 六、InfoBar 消息一览

| 消息 | 触发 | 含义 |
|------|------|------|
| `✓ sent to <name>` | 发送成功 | 报文已写入 socket |
| `✗ 内容为空,未发送` | 空内容拦截（D10） | 没写任何东西就发 |
| `✗ no receiver found` | Discover 返回空 | 没 agent 在监听 |
| `✗ multiple receivers found, need exactly one` | Discover 返回 > 1 | MVP 暂不支持多选 |
| `✗ discover error: <err>` | Discover 内部错误 | 注册表目录读不了？ |
| `✗ marshal error: <err>` | Payload/Envelope 序列化失败 | 几乎不可能 |
| `✗ send failed: <err>` | Dial 或 Write 失败 | socket 连不上/写不进 |

## 七、原型工具（proto/eabp-sender/）

调试用，**不进 microNeo 主线**（独立 `go.mod`，根 `make build` 不编译它）。

| 工具 | 用途 |
|------|------|
| `cmd/discover` | 扫描注册表 + 存活检测 + 僵尸 GC。可选 `--json` 输出 |
| `cmd/send` | 手工拼报文 + 发送给指定 receiver |

**示例**：

```bash
cd proto/eabp-sender/go
go build ./cmd/discover ./cmd/send

./discover                                # 列出存活 receiver
./discover --json                         # 机器可读
./send -name pi-12345 -file a.md -line 42 -msg "讲解一下"   # 带 message
./send -name pi-12345 -file a.md -line 42                  # 纯上下文
```

**为什么保留**：调试时手工触发发送，不需要开 microNeo 找 notePane，方便协议层单测/回归。

**维护要求**：`message.go` / `registry.go` 与 `internal/eabp/` 字面镜像。改一边要同步另一边。
