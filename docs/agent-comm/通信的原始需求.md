## 1. 需求

### 原始想法
- 用户同时开多个 microNeo，编辑不同文件
- microNeo 可以在**用户需要的时候**，发送信息给 ai agent

### 具体的通信需求

通信的双方：
- 一边是 ai agent（比如 pi、opencode），可能同时有多个进程在处理不同的事情
- 一边是 microNeo，可能同时有多个进程在处理不同的文件

通信的方向：
- 目前我们只考虑 **microNeo → ai agent** 单向推送
- 当同时存在多个 ai agents 时，microNeo 可以指定其中一个（用名字或 PID）

### 一次"发送"意味着什么
用户在 microNeo 里编辑文件，**主动触发一次发送**（命令 / 快捷键）。microNeo 把当前编辑上下文打包，推送给指定的 ai agent。典型场景：

- 选中一段 markdown/代码 → 发送 → 让 agent 看看这段
- 光标停在某个位置 → 发送 → 告诉 agent "我现在在看这里"
- 发送时可以**附带一句话提问**（可选）

> 注意：这是**按需、一次性、用户主动发起**的推送，**不是**对光标移动/选区变化的持续追踪。

### 初步想法
- **在 ai agent 里写专门的扩展程序：**
	- 注册：我开始工作了，我可以接收信息，我叫什么名字
	- 监听：监听有没有人向我发送信息
	- 转发：当收到信息后，提供给 LLM
- **发送端：**
	- 查看有没有监听者，有几个监听者
	- 当存在多个监听者时，可以让用户挑选一个
	- 根据用户的操作，把信息发送给指定的监听者
