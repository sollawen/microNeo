# microNeo 配置文件的优化

## 现状

micro原生的配置文件
- settings.json
- colorschemes/xxxx.micro
- syntax/markdown.yaml

文件位置
- micro的代码包里面内置了一批配置文件，包括几个colorscheme, 还有一堆语法的yaml。
- 用户也可以在 `~/.config/microNeo/` 目录里面写自己的配置文件。

---


## microNeo 的配置文件

### 两个 markdown.yaml 的对比

microNeo主要做md文件的渲染，它使用的markdown.yaml来自 `~/.config/microNeo/syntax/markdown.yaml`（用户自定义配置），而非micro原生的内置版本。


#### 1. 代码块支持

| | micro 原生内置 | microNeo 使用的 |
|---|---|---|
| **Fenced code blocks** | 简单匹配 `^```$` | **支持嵌入 25+ 种语言**的高亮（typescript, javascript, python, java, c, cpp, rust, go, ruby, bash, css, html, json, yaml, sql, kotlin, swift, dart, php, scala, lua, r, dockerfile, graphql, vue, xml 等） |
| **内联代码** | backtick 规则 | backtick 规则 |

#### 2. 语法特性覆盖与规则差异

| 特性 | micro 原生内置 | microNeo 使用的 |
|---|---|---|
| **标题** | `^#{1,6}.*` | `^#{1,6}\s.+$` (要求 # 后有空格) |
| **斜体** | `(^\|[[:space:]])(_[^ ][^_]*_\|\*[^ ][^*]*\*)` (宽松匹配) | `\*[^*]+\*` (严格匹配) |
| **加粗** | `(^\|[[:space:]])(__[^ ][^_]*__\|\*\*[^ ][^*]*\*\*)` | `\*\*.+?\*\*` 和 `__.+?__` (非贪婪) |
| **删除线** | `(^\|[[:space:]])~~[^ ][^~]*~~` | `~~.+?~~` (非贪婪) |
| **引用** | `^>.*` | `^>\s.*$` (要求 > 后有空格) |
| **无序列表** | `^[[:space:]]*[\*+-] ` (结尾是空格) | `^[\\s]*[\*\-]` (结尾是空格) |
| **有序列表** | `^[[:space:]]*[0-9]+\. ` (结尾是空格) | `^[\\s]*\d+\.` (结尾是空格) |
| **任务列表** | ❌ | ✅ `^[\s]*[-*+] \[[ xX]\] ` |
| **水平线** | `^(---+|===+|___+|\*\*\*+)\s*$` (支持 -、=、_、* 四种符号) | `^(---+|===+|___+|\*\*\*+)\s*$` (与原生相同) |
| **链接** | `\\[[^]]+\\]` + `\\[(...)\\]\\([^)]+\\)` | 包含在 URL 规则中 |
| **URL** | `https?://[^ )>]+` | `https?://[^\s]+` (允许更多字符) |
| **表格** | ✅ `.*[ :]\|[ :].*` | ❌ |
| **图片** | ✅ `!\\[...\\](...)` | ❌ |
| **HTML 标签** | ❌ | ✅ `<[/?]?[a-zA-Z][a-zA-Z0-9]*[^>]*>` |
| **HTML 实体** | ❌ | ✅ `&[a-zA-Z0-9#]+;` |
| **HTML 注释** | ❌ | ✅ `<!--.*?-->` |
| **邮箱地址** | ❌ | ✅ email 正则 |

#### 3. 总结对比

| 方面 | micro 原生内置 | microNeo 使用的 |
|---|---|---|
| **复杂度** | 简洁 (~30 行) | 复杂 (~130 行) |
| **代码块内嵌语言** | ❌ | ✅ 支持 25+ 种 |
| **任务列表** | ❌ | ✅ |
| **HTML 语法** | ❌ | ✅ |
| **图片/表格** | ✅ | ❌ |
| **正则风格** | 贪婪匹配 | 非贪婪匹配 |

**结论**：microNeo 使用的 markdown.yaml 来自用户自定义配置，功能更完整，支持 fenced code blocks 内嵌语言高亮、任务列表、HTML 语法等高级特性。但缺少表格和图片支持。
