# 我的想法
- micro原生没有弹窗的概念
- 我们的microNeo,未来会有不少需要弹窗的地方
	- 现在已经有一个selectPane了
	- 以后还会有 fileList
	- md文件的大纲目录选择器
	- color/theme选择器
	- 一段弹出的版本说明
	- 以及更多的应用场景
- 这些应用场景的共同特点
	- 临时的
	- 弹出后,所有焦点都在这个临时的窗口里,直到退出/关闭为止,就回到原来的地方
	- 有边框,title
	- 有触发点,
		- 可能是全局都可以用热键触发
		- 可能是只有在某个pane里面或某些场景下触发
		- 可能是某个ctrl-e后的命令触发
	- 有返回值给原来的调用者
	- 其它的我现在有还没想到的点

所以,我觉得我们需要讨论和研究
- 是不是我们应该新设计一个非常小巧的框架,给未来的这些应用?
- 然后,在这个新的框架内,来实现包括现在的selectPane在内的这些弹窗应用?

我觉得microNeo应该是一个很轻量的软件,所以我设想的弹出式浮窗,我不想设计嵌套的功能。也就是说,
- 同一时间只有一个弹出窗,
- 它打开了之后,就只是它自己的功能,不会再里面再打开一个弹窗出来了。
- 而且焦点的只能在这个打开了的弹窗里了,不能跑到其它地方去
- 直到这个弹窗关闭

## microNeo现在的主循环

**DoEvent 简化版**(micro.go:486-559):

```go
func DoEvent() {
    var event tcell.Event

    // === 1. Display 阶段:按顺序画所有该画的东西 ===
    screen.Screen.Fill(' ', config.DefStyle)
    screen.Screen.HideCursor()
    action.Tabs.Display()                              // 1 Tab 栏
    for _, ep := range action.MainTab().Panes {
        ep.Display()                                   // 2 各 pane(BufPane)
    }
    action.MainTab().Display()                         // 3 main tab 自身
    action.InfoBar.Display()                           // 4 底部 info bar(永远在)
    if action.TheNotePane != nil && action.TheNotePane.IsOpen() {
        action.TheNotePane.Display()                   // 5 notePane(如果开着)
    }
    screen.Screen.Show()

    // === 2. 等事件(select 监听多个 channel)===
    select {
    case event = <-screen.Events:
        // ... 其他 case 略(jobs / autosave / draw / timer / signal)...
    }

    // === 3. 处理事件(关键 if-else 树,分支短路)===
    if event != nil {
        _, resize := event.(*tcell.EventResize)
        if resize {
            // resize 事件:通知所有
            action.InfoBar.HandleEvent(event)
            action.Tabs.HandleEvent(event)
            if action.TheNotePane != nil && action.TheNotePane.IsOpen() {
                action.TheNotePane.HandleEvent(event)
            }
        } else if action.TheNotePane != nil && action.TheNotePane.IsOpen() {
            action.TheNotePane.HandleEvent(event)       // ← modal: notePane 开着就给它
        } else if action.InfoBar.HasPrompt {
            action.InfoBar.HandleEvent(event)           // ← 命令行 prompt
        } else {
            action.Tabs.HandleEvent(event)              // ← 落到 Tab → BufPane
        }
    }
}
```

## 引入 floatFrame

**已拍板**：FloatFrame 容器跟 InfoBar / TheNotePane 同生命周期——microNeo 启动时在 `globals.go` InitGlobals 里 new，永远非 nil，进程退出时才销毁。

```go
// globals.go
var TheFloatFrame *FloatFrame   // 新增

func InitGlobals() {
    InfoBar = NewInfoBar()
    TheNotePane = NewNotePane()
    TheFloatFrame = NewFloatFrame()  // 新增：启动时初始化
}
```

**DoEvent 改造后**（`micro.go:486-559`）：不需要 `!= nil &&` 检查，直接 `if TheFloatFrame.IsOpen()`：

```go
func DoEvent() {
    var event tcell.Event

    // === 1. Display 阶段：按顺序画所有该画的东西 ===
    // ... Tabs / 各 pane / InfoBar 同现状 ...
    if action.TheNotePane != nil && action.TheNotePane.IsOpen() {
        action.TheNotePane.Display()                   // ⑤ notePane（如果开着）
    }
    if action.TheFloatFrame.IsOpen() {                 // ⑥ FloatFrame（如果开着）
        action.TheFloatFrame.Display()                 //    ← 不需要 nil check（启动时已 new）
    }
    screen.Screen.Show()

    // === 2. 等事件（select 监听多个 channel）===
    select {
    case event = <-screen.Events:
        // ... 其他 case 略（jobs / autosave / draw / timer / signal）...
    }

    // === 3. 处理事件（关键 if-else 树，分支短路）===
    if event != nil {
        _, resize := event.(*tcell.EventResize)
        if resize {
            // resize 事件：通知所有
            action.InfoBar.HandleEvent(event)
            action.Tabs.HandleEvent(event)
            if action.TheNotePane != nil && action.TheNotePane.IsOpen() {
                action.TheNotePane.HandleEvent(event)
            }
            if action.TheFloatFrame.IsOpen() {         // resize 也通知 FloatFrame
                action.TheFloatFrame.HandleEvent(event)
            }
        } else if action.TheFloatFrame.IsOpen() {     // ← FloatFrame 最高优先级（modal）
            action.TheFloatFrame.HandleEvent(event)
        } else if action.TheNotePane != nil && action.TheNotePane.IsOpen() {
            action.TheNotePane.HandleEvent(event)       // ← notePane 开着就给它
        } else if action.InfoBar.HasPrompt {
            action.InfoBar.HandleEvent(event)           // ← 命令行 prompt
        } else {
            action.Tabs.HandleEvent(event)              // ← 落到 Tab → BufPane
        }
    }
}
```

microNeo启动后，就创建了floatFrame，但是没有启用。

用户在使用过程中，通过各种方式，通知microNeo我现在要打开一个 selectPane了
- 于是，就创建一个 selectPane
	- selectPane负责计算出来 x,y,w,h,title,甚至frameColor
	- 然后selectPane把这些参数，传给floatFrame，
	- 并把自己的render() and 事件处理函数传给floatFrame
- 于是通过floatFrame就接管了所有的用户交互
- 当selectPane关闭后
	- 就清空floatFrame里面的数据
	- 然后销毁selectPane（或是不销毁，等下次用户再使用？）
	- 这样floatFrame又变成空的了。所有的用户交互又恢复正常了
	
现在floatFrame的底层结构已经完成了
