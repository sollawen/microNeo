# 组合键键码表

Ctrl / Alt 配字母和数字时，终端实际收到的字节。可在 microNeo 里用 `raw` 命令（命令栏输入 `raw`）实时验证。

Alt 部分基于 ESC 前缀约定（`Alt-X` → `\x1b X`）。macOS 默认把 Option 当 compose 键，需在终端设置里把 Option 改为 Meta 才能发下面的字节序列；Linux 终端默认就是 ESC 前缀。

microNeo 的绑定来自 `internal/action/defaults_other.go`（Linux/Windows）和 `internal/action/defaults_darwin.go`（macOS），列出的绑定是这两个文件的并集。

---

## Ctrl + 字母（26）

ASCII 标准：Ctrl 清掉字符第 5、6 位（`AND 0x1F`）。A–Z (0x41–0x5A) → 0x01–0x1A，完全规律，全终端一致。

| 组合键 | 16进制码 | ASCII 含义 | microNeo 绑定 |
|--------|---------|-----------|---------------|
| Ctrl-A | 0x01 | SOH 标题开始 | SelectAll |
| Ctrl-B | 0x02 | STX 正文开始 | ShellMode |
| Ctrl-C | 0x03 | ETX 正文结束（Unix 里作 SIGINT 中断） | Copy, CopyLine |
| Ctrl-D | 0x04 | EOT 传输结束（Unix 里作 EOF） | Duplicate, DuplicateLine |
| Ctrl-E | 0x05 | ENQ 询问 | CommandMode |
| Ctrl-F | 0x06 | ACK 确认应答 | Find |
| Ctrl-G | 0x07 | BEL 响铃 | ToggleHelp |
| Ctrl-H | 0x08 | Backspace | Backspace（与 Backspace 键同字节路径） |
| Ctrl-I | 0x09 | Tab | Autocomplete, IndentSelection, InsertTab |
| Ctrl-J | 0x0A | LF 换行 | **PlayMacro**（不会换行！） |
| Ctrl-K | 0x0B | VT 垂直制表符 | CutLine |
| Ctrl-L | 0x0C | FF 换页 / 清屏 | goto（跳到指定行号） |
| Ctrl-M | 0x0D | Enter | InsertNewline |
| Ctrl-N | 0x0E | SO 切出 | FindNext |
| Ctrl-O | 0x0F | SI 切入 | :file（FileSelector，改造自原生 OpenFile） |
| Ctrl-P | 0x10 | DLE 数据链路转义 | FindPrevious |
| Ctrl-Q | 0x11 | DC1 / XON 恢复输出 | Quit |
| Ctrl-R | 0x12 | DC2 | ToggleRuler |
| Ctrl-S | 0x13 | DC3 / XOFF 暂停输出 | Save |
| Ctrl-T | 0x14 | DC4 | AddTab |
| Ctrl-U | 0x15 | NAK 否认应答 | ToggleMacro |
| Ctrl-V | 0x16 | SYN 同步空闲 | Paste |
| Ctrl-W | 0x17 | ETB 传输块结束 | NextSplit, FirstSplit |
| Ctrl-X | 0x18 | CAN 取消 | Cut, CutLine |
| Ctrl-Y | 0x19 | EM 介质结束 | Redo |
| Ctrl-Z | 0x1A | SUB 替换（Unix 里作 SIGTSTP 挂起） | Undo |

注：Ctrl-H / Ctrl-I / Ctrl-M 这三个字节和 Backspace / Tab / Enter 键共享同字节，microNeo 默认不单独绑 `Ctrl-h/I/m`，而是依赖各自的键名绑定。

## Ctrl + 数字（10）

数字 ASCII（0x30–0x39）不在 Ctrl 的清位作用域内，**无 ASCII 标准**，靠各终端硬编码。规律：数字 2–8 是 `@ [ \ ] ^ _` + DEL 这 7 个非字母控制码的替身，已用 `raw` 实测验证。

| 组合键 | 16进制码 | ASCII 含义 | microNeo 绑定 |
|--------|---------|-----------|---------------|
| Ctrl-1 | 0x31 | - | 未绑定 |
| Ctrl-2 | 0x00 | NUL 空 | 未绑定 |
| Ctrl-3 | 0x1B | ESC 转义 | 未绑定 |
| Ctrl-4 | 0x1C | FS 文件分隔符 | 未绑定 |
| Ctrl-5 | 0x1D | GS 组分隔符 | 未绑定 |
| Ctrl-6 | 0x1E | RS 记录分隔符 | 未绑定 |
| Ctrl-7 | 0x1F | US 单元分隔符 | 未绑定 |
| Ctrl-8 | 0x7F | DEL 删除 | Backspace（同 Backspace2 键字节路径） |
| Ctrl-9 | 0x39 | - | 未绑定 |
| Ctrl-0 | 0x30 | - | 未绑定 |

## Ctrl + 标点（6）

只有 ASCII 0x40–0x5F 范围内的标点（`@ [ \ ] ^ _`）按 Ctrl 才会产生确定的标准控制码。其他标点（如 `# $ !`）按 Ctrl 后终端通常发乱码或忽略，所以下表只列这 6 个。它们与上一节 Ctrl-2 至 Ctrl-7 **字节相同**（同一套控制码）：Ctrl-@ ↔ Ctrl-2 ↔ Ctrl-Space；Ctrl-[ ↔ Ctrl-3 ↔ Esc；… 依次类推。

| 组合键 | 16进制码 | ASCII 含义 | microNeo 绑定 |
|--------|---------|-----------|---------------|
| Ctrl-@ | 0x00 | NUL 空 | 未绑定（与 Ctrl-Space 同字节） |
| Ctrl-[ | 0x1B | ESC 转义 | 未绑定（与 Esc 键同字节） |
| Ctrl-\ | 0x1C | FS 文件分隔符 | 未绑定 |
| Ctrl-] | 0x1D | GS 组分隔符 | 未绑定 |
| Ctrl-^ | 0x1E | RS 记录分隔符 | 未绑定 |
| Ctrl-_ | 0x1F | US 单元分隔符 | 未绑定 |

注：`bindings.json` 里写 `Ctrl-\` 这类字符串含反斜杠/方括号需手动转义（tcell 仍识别为 `CtrlBackslash` / `CtrlLBracket` 等命名）。

## Ctrl + 方向键（4）

方向键本身是 4 字节 CSI 序列：`ESC [ A/B/C/D`。加修饰键时按 xterm 约定，扩展为 `ESC [ 1;修饰符 A/B/C/D`（6 字节）。修饰符码 = 1 + 位掩码，Ctrl 占 bit 4，所以 Ctrl 单独按下是 `1;5`；Alt 单独是 `1;3`；Shift 单独是 `1;2`；Shift+Ctrl 是 `1;6`。

| 组合键 | 键码（字节序列） | microNeo 绑定 |
|--------|-----------------|---------------|
| Ctrl-Up | 0x1B 0x5B 0x31 0x3B 0x35 0x41 (ESC [ 1;5 A) | CursorStart（到文件首）|
| Ctrl-Down | 0x1B 0x5B 0x31 0x3B 0x35 0x42 (ESC [ 1;5 B) | CursorEnd（到文件尾）|
| Ctrl-Left | 0x1B 0x5B 0x31 0x3B 0x35 0x44 (ESC [ 1;5 D) | WordLeft |
| Ctrl-Right | 0x1B 0x5B 0x31 0x3B 0x35 0x43 (ESC [ 1;5 C) | WordRight |

---

## Alt + 方向键（4）

加 Alt 修饰（位 2）→ 序列变 `ESC [ 1;3 A/B/C/D`（6 字节）。

`AltUp`/`AltDown` 在编辑区和命令栏是不同的 action（编辑区是移动整行，命令栏是跳到输入首/尾）；`AltLeft`/`AltRight` 在不同平台不同：Linux/Windows 是类 Home/End，macOS 是 emacs 风（按词移动）。

| 组合键 | 键码（字节序列） | microNeo 绑定（编辑区 / 命令栏） |
|--------|------------------|----------------------------------|
| Alt-Up | 0x1B 0x5B 0x31 0x3B 0x33 0x41 (ESC [ 1;3 A) | MoveLinesUp / CursorStart |
| Alt-Down | 0x1B 0x5B 0x31 0x3B 0x33 0x42 (ESC [ 1;3 B) | MoveLinesDown / CursorEnd |
| Alt-Left | 0x1B 0x5B 0x31 0x3B 0x33 0x44 (ESC [ 1;3 D) | StartOfTextToggle（macOS: WordLeft）/ 同左 |
| Alt-Right | 0x1B 0x5B 0x31 0x3B 0x33 0x43 (ESC [ 1;3 C) | EndOfLine（macOS: WordRight）/ 同左 |

## 字节层的大小写与修饰符

一个组合键实际发多少字节,决定了它能不能区分大小写 / Shift:

| 组合键族 | 字节数 | 大小写或 Shift 在字节中区分吗 | Ctrl-Shift+a 和 Ctrl+a 可区分吗 |
|----------|--------|------------------------------|----------------------------------|
| Ctrl + 字母 | 1 | 否（单字节装不下“哪个字母”+“是否 Shift”两份信息）| 否（按了都发 0x01） |
| Alt + 字母 | 2 | 是（第二字节原样保留大小写）| N/A（Shift 信息已经隐含在第二字节里） |
| Ctrl/Alt + 方向键 | 6 | 是（CSI 参数 `1;修饰符` 编码修饰符）| 是：`1;5` = Ctrl，`1;6` = Ctrl+Shift，`1;7` = Ctrl+Alt 等 |

所以 binding 写法上有区别:
- `Ctrl-a` 和 `Ctrl-A` 在 microNeo 里是**同一个**按键（默认 binding 全用小写字母书写是传统习惯，实际只要是同一字母都识别）
- `Alt-a` 和 `Alt-A` 是**两个**按键，要分别绑（参考 default 里 `Alt-F: FindLiteral` 和 `Alt-f: WordRight` 共存）
- 方向键的修饰全在 CSI 参数里,所以 Ctrl/Ctrl-Shift/Ctrl-Alt 都能独立绑
