我在开发一个terminal markdown editor。

自从半年前开始使用vibe codeing之后，我就删掉了vscode，因为ai coding比我coding要快多了。于是我绝大部份时间是在用markdown与ai对产品设计、方案、架构进行讨论。我想在ai时代里，写代码的人会越来越少，写markdown的人会越来越多的。

但是世面上的 terminal editor 
- 要么就是vim这程为了写代码而生的，对写代码功能非常强大，但是markdown就不舒服了，也太重了。
- 要么就是nano，micro这种轻量级的，对markdown只能是进行源代码编辑，没有渲染效果。
- 还有就是专门在terminal里面渲染显示markdown的，只显示不能编辑
- 全世界都没有一个能把markdown的渲染和编辑整合在一起的terminal editor

于是我自己动手，在GLM-5的帮助下，在micro的基础上改写了一个microNeo，专门优化了markdown的处理。
- 漂亮的渲染和显示
- 渲染和显示在同一个pane里面，不用split占用大量宝贵的屏幕空间。点击直接编辑。编辑完马上渲染

自己使用起来，真的十分方便。

但是我痛苦的是：没有人知道这个工具啊，全世界只有我一个人在使用。

## English Version

I'm developing a terminal Markdown editor.

Since I started using vibe coding six months ago, I've deleted VS Code because AI coding is much faster than my own coding. So most of my time is spent using Markdown to discuss product design, solutions, and architecture with AI. I believe that in the AI era, fewer people will write code, but more people will write Markdown.

However, the terminal editors available today:
- Some are like Vim, designed for coding. They're powerful for code but uncomfortable for Markdown, and they're too heavy.
- Some are lightweight like nano or micro, which can only edit Markdown as raw text without rendering.
- Others can render and display Markdown in the terminal but can't edit it.
- There is simply no terminal editor in the world that integrates Markdown rendering and editing together.

So I decided to build my own. With the help of GLM-5, I modified micro to create microNeo, specifically optimized for Markdown handling.
- Beautiful rendering and display
- Rendering and editing in the same pane — no split views needed, saving precious screen space. Click to edit directly, rendered instantly after.

After using it myself, it's truly convenient.

But here's my pain: nobody knows about this tool. I'm the only user in the world.
