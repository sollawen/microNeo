## 二、各安装方式详解

### 1. 一键 curl 安装（最重要！）

这是用户最常用的方式，类似：
```bash
curl -sL https://get.microneo.io/install.sh | bash
```

**实现方案：**

创建一个静态资源服务，有两种方式：

**方案 A：使用 GitHub Pages + Release 文件（推荐）**

```
sollawen/microneo
├── install.sh              # 安装脚本
├── formula/microneo.rb # Homebrew Formula (可选)
└── .github/workflows/release.yaml  # 发布时自动更新到 gh-pages
```

install.sh 核心逻辑：
```sh
#!/bin/bash
set -e

# 检测操作系统和架构
detect_os() {
  os="$(uname -s)"
  arch="$(uname -m)"
  case "$os" in
    Linux*) os="linux" ;;
    Darwin*) os="osx" ;;
    CYGWIN*|MINGW*|MSYS*) os="win64" ;;
  esac
  case "$arch" in
    x86_64) arch="amd64" ;;
    arm64|aarch64) arch="arm64" ;;
  esac
  echo "${os}-${arch}"
}

# 获取最新版本号
VERSION=$(curl -sL https://api.github.com/repos/sollawen/microneo/releases/latest | grep -o '"tag_name": *"[^"]*"' | cut -d'"' -f4)

# 下载对应平台的二进制
URL="https://github.com/sollawen/microneo/releases/download/${VERSION}/microneo-${os}.tar.gz"
curl -L "$URL" -o /tmp/microneo.tar.gz

# 解压安装
tar -xzf /tmp/microneo.tar.gz -C /tmp
sudo mv /tmp/microneo /usr/local/bin/microneo
rm /tmp/microneo.tar.gz
```

**关键前提：release.yaml 需要开启 push tags触发，并发布 assets 到 GitHub Release**

### 2. Homebrew 安装

用户可以：`brew install microneo/tap/microneo`

**实现方案：**

需要创建独立的 Homebrew tap仓库：

```
homebrew-tap/
├── README.md
├── Formula/
│   └── microneo.rb
└── .github/workflows/
    └── release.yml
```

或集成到主仓库（通过 formula/ 目录）

**Formula 示例：**
```ruby
class Microneo < Formula
  desc "A modern and intuitive terminal-based text editor"
  homepage "https://github.com/sollawen/microneo"
  url "https://github.com/sollawen/microneo/releases/download/vX.X.X/microneo-linux64.tar.gz"
  sha256 "..."

  def install
    bin.install "microneo"
    man1.install "micro.1"
  end
end
```

**自动化方案：** release.yaml 中动态更新 Formula 版本

---

### 3. npm 安装

对于 Go 编写的 CLI工具，可以通过 `pkg.pr.new` 或 `npx`包装：

**方案 A：npm 包包装（不推荐维护成本高）**

```bash
npx microneo file.txt
```

**方案 B：提供 npm 包但内部调用二进制（适合 Node.js 生态集成）**

如果确实需要，可以用 `@vercel/ncc` 打包后发布 npm 包。

**结论：** Go CLI 工具优先级是 curl + brew，npm 非必需。

## 六、注意事项

1. **域名/托管**：建议注册 `microneo.io` 或使用 GitHub Pages
2. **版本同步**：install.sh 需要能够动态获取最新 release 版本
3. **checksum校验**：生产环境必须包含 SHA256 校验
4. **权限处理**：安装脚本需要处理 sudo 权限
5. **卸载支持**：install.sh 最好也支持 uninstall

## 七、release.yaml 详细改进计划

### 当前状态

```yaml
name: Release builds
on:
  workflow_dispatch:  # ❌ 只支持手动触发
  # push:
  #   tags:
  #     - 'v*.*.*'     # ❌ tag 触发被注释掉了
jobs:
  release:
    strategy:
      matrix:
        go-version: [1.23.x]
        os: [ubuntu-latest]  # ❌ 只在 Linux 编译
```

### 改进后状态

```yaml
name: Release builds
on:
  push:
    tags:
      - 'v*.*.*'  # ✅ 打 tag 自动触发

jobs:
  release:
    strategy:
      matrix:
        go-version: [1.23.x]
        os: [ubuntu-latest, macos-latest, windows-latest]  # ✅ 多平台编译
    runs-on: ${{ matrix.os }}
    steps:
    - name: Checkout
      uses: actions/checkout@v4
      with:
        fetch-depth: 0
        fetch-tags: true

    - name: Setup Go
      uses: actions/setup-go@v5
      with:
        go-version: ${{ matrix.go-version }}

    - name: Build
      run: tools/cross-compile.sh

    - name: Publish
      uses: softprops/action-gh-release@v2
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
      with:
        files: binaries/*
        generate_release_notes: true  # ✅ 自动生成 release notes

    - name: Cleanup
      run: rm -rf binaries
```

### 改进点总结

| 项目 | 当前 | 改进后 |
|------|------|--------|
| 触发方式 | 手动（workflow_dispatch） | 自动（打 tag） |
| 编译平台 | 仅 Linux | Linux + macOS + Windows |
| Release Notes | 手动填写 | 自动生成 |

### 使用方式

```bash
git tag v1.0.0
git push origin v1.0.0

```

### 8.1 install.sh 需要完善的点

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| 缺少 SHA256 校验 | ⚠️ 严重 | 必须校验下载文件的完整性，防止中间人攻击 |
| 架构检测不够健壮 | ⚠️ 中 | arm64/aarch64 映射需完善 |
| 安装路径问题 | ⚠️ 中 | macOS SIP 限制，/usr/local/bin 不可用 |
| 缺少卸载功能 | ❌ | 计划提到但未实现 |
| 缺少进度提示 | ❌ | 用户看不到下载进度 |
| 缺少安装前确认 | ❌ | 没有询问用户 |
| 缺少错误处理 | ❌ | 网络/解压失败需优雅处理 |

### 8.2 release.yaml 需要修改的点

| 问题 | 严重程度 | 说明 |
|------|----------|------|

| 缺少 license 声明 | ⚠️ 中 | 应添加 `license "MIT"` |
