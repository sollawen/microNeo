---

## 切分原则

每个 milestone 必须同时满足三条：

1. **只引入一个新风险维度**——不把"新架构 + 新交互"绑在一起做
2. **可独立 demo**——不依赖未来版本就能验证
3. **能完整合并到 main**——不留半截工程

## v0.1 — 架构换骨

**目标**：证明"渲染管线作为统一入口"的架构可行。**用户行为零变化**——和当前完全一样的体验。

**范围**：
- 新建 `internal/display/md/` 子目录
- `pipeline.go`：管线主循环（对每个可见行问："这行你能处理吗"）
- `segment.go`：Segment 接口骨架
- `table.go`：表格 segment（复用现有 `DetectTables` / `calcColWidths`）
- segment 输出 == buffer 行数 1:1（**不引入 mapper**）
- segment 输出纯净 style，**不叠加 Micro 语法高亮**（留到 v0.4）
- `bufwindow.go` 接入 pipeline 作为新路径；旧 inline padding 路径保留作为 fallback
- 配置开关 `mdpipeline`（默认 false）做灰度

**不做**：
- 不动光标行为
- 不动输入
- 不动状态栏
- 不引入模式概念

**Demo**：开关切换新旧路径，视觉一致。

**验收标准**：
- `mdpipeline: false` 时行为与当前 100% 一致
- `mdpipeline: true` 时表格渲染效果与旧路径一致
- 非 `.md` 文件不受任何影响

## v0.2 — Segment 内光标 fallback + 状态栏

**目标**：在 segment 层引入"编辑态"，但仍**完全在 Micro 原生交互模型下**——有光标、方向键移动、可显示行号（暂保留）。

**范围**：
- 光标命中 segment 任一行 → 整片回退原生显示
- 多光标支持：任一光标命中即回退
- 状态栏 READING/EDITING 指示（`mdmodeindicator` 配置项）
- `.md` 文件不显示行号（gutter 移除）

**不做**：
- 不引入"渲染模式"（光标仍可见，方向键仍移动光标而非滚动）
- 不引入鼠标点击的额外逻辑
- 不引入 10 秒计时器

**Demo**：光标移入表格 → 看到原生 markdown；移出 → 看到渲染。状态栏同步切换 READING/EDITING。

**验收标准**：
- 多光标场景行为正确
- 非 `.md` 文件状态栏不变、行号照常显示

## v0.3 — 全局模式切换（首次进入"渲染模式"）

> ⚠️ **全项目风险最高的版本**。如果体量过大，可拆为 v0.3a（模式状态 + 键盘切换）/ v0.3b（鼠标点击）。

**目标**：PRD 描述的核心交互**第一次落地**。

**范围**：
- 打开 `.md` 文件默认进入渲染模式
- 渲染模式：光标隐藏；方向键 ↑↓←→ = 滚动文档（类似 `less`）；其他任意键 = 进入编辑模式
- 编辑 → 渲染：**先用显式快捷键**（如 `Esc`）切换；**10 秒规则推迟到 v0.4**
- 鼠标点击 segment 的**内容字符** → 进入编辑模式（不做装饰字符处理，先不引入 mapper）
- 鼠标点击 segment 的**装饰字符**或非 segment 区域 → 进入编辑模式（行为待定，最简方案是按 Micro 原生处理）

**不做**：
- 不做 10 秒计时器
- 不做鼠标拖选复制
- 不做 Ctrl+F 搜索适配

**Demo**：打开 `.md` 文件看到无光标渲染效果，按任意键立刻回退到 Micro 原生编辑模式；按 `Esc` 回到渲染模式。

**风险点**：
- 输入事件劫持（要拦截 Micro 的 action 层）
- 模式状态机（per-pane 状态、初始状态、文件切换）
- 鼠标点击的 screen → buffer 反向映射（v0.1 的 segment 输出是 1:1，所以这步尚可控）

**验收标准**：
- 切换 pane（Tab/VSplit）时各 pane 状态独立（基础保证，完整多视图测试留 v0.9）
- `.md` 之外文件不受影响

## v0.4 — 10 秒计时器 + inline 渲染器（粗体 / 斜体）

**目标**：补全模式切换的最后一环；引入第二个能力维度（行内渲染）。

**范围**：
- 10 秒空闲计时器（配置项 `mdrenderidle`，默认 10，0 = 从不自动进入渲染模式）
- 鼠标移动不算操作，只有按键和点击重置计时器
- inline 渲染器（`md/inline.go`）：
  - 去除 `**` / `*` / `__` / `_` 标记
  - 应用粗体/斜体终端属性
  - 跨行闭合保守处理：标记未闭合则该行不渲染
- inline 输出**叠加** Micro 语法高亮颜色（每个字符的最终样式 = Micro 颜色 + inline 终端属性）
- 表格 segment 单元格内容改为调 inline 渲染器
- 配置项 `mdbolditalic`

**Demo**：编辑后 10 秒看到完整渲染；表格内的 `**bold**` 显示为粗体无 `**`。

**验收标准**：
- 不闭合的 `**` 不引起渲染错乱
- inline 与表格 segment 叠加正确

---

## v0.5 — 主要结构渲染器（代码块 / 标题 / 列表 / 链接）

**目标**：覆盖最常见的 Markdown 元素。**几乎没有新架构**，只是平行增加模块。

**范围**：
- `md/codeblock.go`：代码块 segment（**不调 inline**，原样输出）
- `md/heading.go`：标题 segment（调 inline）
- `md/list.go`：列表 segment（调 inline）
- 链接 `[text](url)` 加到 inline 渲染器：只显示 `text`，用特殊颜色标识
- 各自独立配置项 `mdcodeblock` / `mdheading` / `mdlist` / `mdlink`

**风险维度**：每个模块独立，可分别合并小 PR。

**Demo**：一份典型 README.md，大部分内容以渲染效果呈现。

---

## v0.6 — mapper（segment 输出行数 ≠ buffer 行数）

**目标**：解锁 segment "长得不一样"的能力。

**范围**：
- `md/mapper.go`：buffer line ↔ screen line 双向映射
- 表格升级：加 ASCII 边框（`mdtableborder` 配置项）、表头分隔，输出多行（含装饰行）
- 代码块升级：加包围框、标题栏（语言标识）
- 列表升级：嵌套缩进按视觉层级输出
- 渲染模式下方向键滚动适配新行数差
- 鼠标点击定位同步升级：处理装饰字符 = 无效点击；内容字符 = 精确定位

**风险维度**：mapper 是全项目第二复杂的子系统（仅次于 v0.3 的模式状态机）。

**Demo**：表格带边框、代码块带框；点击单元格内的文字光标准确定位，点击边框线无效。

**验收标准**：
- mapper 在大文件下性能可接受（目标 < 16ms 每帧）
- 滚动时光标位置追踪正确

## v0.7 — 鼠标拖选复制 + 搜索

**范围**：
- 渲染模式下拖选（按下 + 移动 + 抬起） → 复制剪贴板 + 状态栏闪现 `Copied` + **保持渲染模式**
- 渲染模式下 `Ctrl+F` → 进入编辑模式 + Micro 原生搜索（搜原始 markdown）
- 编辑模式下复制沿用 Micro 原生机制

**待拍板**：拖选复制的具体内容形式（渲染后纯文本 vs 原始 markdown，装饰字符如 `|` 如何处理）。

---

## v0.8 — 全面接管 + 性能

**范围**：
- 引用块（`>`）、水平分割线（`---`）、图片标记（`![alt](url)`）、脚注
- `md/paragraph.go`：段落兜底渲染器（覆盖所有未被其他渲染器接管的行）
- segment 检测缓存机制（仅 buffer 内容变更时重算）
- 性能 benchmark：大文件（10K+ 行）、复杂嵌套、大表格
- 100% 渲染接管演示：渲染模式下无任何 fallback

---

## v0.9 — 多视图、超宽处理、跨终端

**范围**：
- Tab / VSplit / HSplit：各 pane 模式状态、计时器独立维护
- 表格按词换行（参考 glow/leaf），动态调整列宽
- 代码块按字符换行（代码中空格有意义，不能按词断）
- 终端兼容测试矩阵：iTerm2 / Terminal.app / Windows Terminal / Alacritty / kitty
- 真粗体/真斜体不支持时优雅降级
