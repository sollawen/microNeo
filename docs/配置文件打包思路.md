# microNeo 配置文件的优化

## micro 的配置文件位置

micro原生的配置文件
- settings.json
- colorschemes/xxxx.micro
- syntax/markdown.yaml

文件位置
- micro的代码包里面内置了一批配置文件，包括几个colorscheme, 还有一堆语法的yaml。
- 用户也可以在 `~/.config/microNeo/` 目录里面写自己的配置文件。




## microNeo 的配置文件


microNeo主要做md文件的渲染，它使用的markdown.yaml来自 `~/.config/microNeo/syntax/markdown.yaml`（用户自定义配置），而非micro原生的内置版本。如果只使用micro原生的配置，渲染出的效果就没那么漂亮了。

我在想，microNeo以后要编译打包给其它用户使用，那需要把我们的这份markdown.yaml要放在包里面，用户安装了之后要能正确的找到并使用这个yaml里的配置参数。

## Go 程序打包与配置文件分发机制介绍

### 一、Go 程序的打包方式

Go 编译出来的是**单个二进制文件**（静态链接），这是 Go 的一大优势。用户拿到这个二进制文件就能直接运行，不需要安装任何运行时环境。

#### 1. 基本编译
```bash
# 最简单的编译，生成可执行文件
go build -o microneo ./cmd/micro

# 带版本信息的编译（micro 当前的方式）
go build -ldflags "-s -w -X main.version=1.0.0" -o microneo ./cmd/micro
```

#### 2. 交叉编译（跨平台）
```bash
# 编译 Linux 版本
GOOS=linux GOARCH=amd64 go build -o microneo-linux ./cmd/micro

# 编译 Windows 版本
GOOS=windows GOARCH=amd64 go build -o microneo.exe ./cmd/micro

# 编译 macOS 版本
GOOS=darwin GOARCH=arm64 go build -o microneo-mac ./cmd/micro
```

### 二、配置文件的三种分发方式

#### 方式 A：Go Embed（✅ micro 已经在用）

Go 1.16+ 引入了 `//go:embed` 机制，可以把文件在**编译时**嵌入到二进制文件中。

**micro 当前的做法：**

```go
// runtime/runtime.go
//go:embed colorschemes help plugins syntax
var runtime embed.FS
```

这意味着 `runtime/syntax/`、`runtime/colorschemes/` 等目录下的所有文件，在编译时就已经被**打包进二进制文件**了。

**加载优先级**（见 `internal/config/rtfiles.go` 的 `InitRuntimeFiles`）：
1. 先加载用户目录 `~/.config/microNeo/syntax/` 中的文件（用户自定义）
2. 再加载 embed 嵌入的 `runtime/syntax/` 文件（内置）
3. 用户文件**同名覆盖**内置文件

**结论：microNeo 已经自带了 `runtime/syntax/markdown.yaml`，编译后就内嵌在二进制中了。**

但问题是：我们**修改过的 markdown.yaml** 需要放在 `runtime/syntax/markdown.yaml` 里，而不是依赖用户手动放到 `~/.config/microNeo/` 下。

#### 方式 B：安装脚本（作为补充）

如果需要安装时往用户目录放配置文件，可以用 shell 脚本：

```bash
#!/bin/bash
# install.sh

# 复制二进制
cp microneo /usr/local/bin/

# 创建默认配置目录（可选）
mkdir -p ~/.config/microNeoneo/

# 复制默认配置（如果不存在）
if [ ! -f ~/.config/microNeoneo/settings.json ]; then
    cp default-settings.json ~/.config/microNeoneo/settings.json
fi
```

#### 方式 C：首次运行自动初始化

在代码中检测，如果用户配置目录不存在就自动创建：

```go
func ensureConfigDir() {
    if _, err := os.Stat(configDir); os.IsNotExist(err) {
        os.MkdirAll(configDir, 0755)
        // 从 embed 中提取默认配置写入
        data, _ := rt.Asset("runtime/default-settings.json")
        os.WriteFile(filepath.Join(configDir, "settings.json"), data, 0644)
    }
}
```

### 三、发布渠道

| 渠道 | 方式 | 适用场景 |
|------|------|----------|
| **GitHub Releases** | `gh release create` 上传各平台二进制 | 最主流，用户手动下载 |

### 四、针对 microNeo 的建议方案

基于 micro 现有的架构，我们**不需要做额外的打包工作**来分发 markdown.yaml，因为：

1. **micro 已使用 `//go:embed`**，`runtime/syntax/markdown.yaml` 已经编译进二进制
2. 只要我们把**优化后的 markdown.yaml 放到 `runtime/syntax/` 目录**中，编译后就自动带上了
3. 用户的自定义文件（`~/.config/microNeo/`）会覆盖内置文件，所以用户仍然可以自定义

**需要做的事：**
- ✅ 把优化后的 `markdown.yaml` 放在 `runtime/syntax/markdown.yaml`
- ✅ 运行 `make build` 编译（会先 `go generate` 生成 `.hdr` 文件）
- ✅ 发布编译好的二进制文件即可

**可选优化：**
- 如果需要把配置目录从 `~/.config/microNeo/` 改为 `~/.config/microNeoneo/`，需要修改 `ConfigDir` 的定义
- 如果需要首次运行自动创建配置目录，可以在初始化逻辑中添加
