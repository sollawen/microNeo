# 仓库根目录精简方案

## 背景

GitHub 仓库根目录的文件/目录条目过多，会把 README 预览挤到第二屏。
访客从宣传链接进来，首屏看不到项目"卖相"（demo、一句话定位），影响 star 转化。

本方案目标：**在零/低风险前提下，减少 GitHub 仓库根目录显示的条目数**，让 README 重新回到首屏。

---

## 现状

仓库根目录实际提交在 git 里、GitHub 上会显示的条目共 **23 个**：

```
.editorconfig  .github/  .gitignore
AGENTS.md  CHANGELOG.md  LICENSE  LICENSE-THIRD-PARTY
Makefile  README.md
aibp-agents/  assets/  cmd/  data/  docs/
go.mod  go.sum  install.sh  internal/
mkdocs.yml  pkg/  runtime/  snapcraft.yaml  tools/
```

> 注：本地根目录还能看到 `microneo`（13MB 二进制）和 `site/`（mkdocs 构建产物），
> 但两者都已在 `.gitignore` 中，**GitHub 上不显示**，不属于本方案处理范围。

每条目在 GitHub 文件列表中约占 26px 高度，23 条 ≈ 600px，加上 repo header（标题/badges/about），
README 预览确实被推到第二屏才出现。

---

## 现有目录（10 个）

| 目录 | 可否移动/删除 | 理由 |
|---|---|---|
| `.github/` | ❌ | GitHub 强制 `workflows/`、`ISSUE_TEMPLATE/` 在此位置 |
| `aibp-agents/` | ✅ 移 `internal/aibp/aibp-agents/` | 自写的 agent 源码；代码只字符串匹配 `"aibp-agents"`，保留名段即零改动 |
| `assets/` | ✅ 删除 | 打包资源→`tools/packaging/`、展示资源→`docs/website/assets/`、其余为孤儿可删 |
| `cmd/` | ❌ | Go main 入口；Makefile 构建 `./cmd/micro` |
| `data/` | ✅ 删除 | 上游 micro 遗产（metainfo.xml + micro.json），无代码/脚本引用，内容仍是 micro 名号 |
| `docs/` | ❌ | 本方案其他条目的目标归宿 |
| `internal/` | ❌ | Go 私有包，import 路径强依赖 |
| `pkg/` | ❌ | Go 公开包，import 路径强依赖 |
| `runtime/` | ❌ | `go:embed` 源且是被 import 的 package，移动需改 embed 路径与所有 import，影响太大 |
| `tools/` | ❌ | 本方案其他条目的目标归宿 |

## 现有文件（13 个）

| 文件 | 可否移动/删除 | 理由 |
|---|---|---|
| `.editorconfig` | ❌ | 编辑器约定根目录 |
| `.gitignore` | ❌ | git 约定根目录 |
| `AGENTS.md` | ❌ | AI agent（Claude Code 等）默认查找根目录 `AGENTS.md` |
| `CHANGELOG.md` | ❌ | 开源约定根目录 |
| `LICENSE` | ❌ | 开源约定 + GitHub 自动识别根目录 LICENSE |
| `LICENSE-THIRD-PARTY` | ✅ 移 `docs/` | 第三方许可证要求随二进制分发；仅被 2 个发布脚本 `cp` |
| `Makefile` | ❌ | make 默认 cwd 查找；内部全为根相对路径 |
| `README.md` | ❌ | 开源约定 + GitHub 自动渲染根目录 README |
| `go.mod` | ❌ | Go 模块根 |
| `go.sum` | ❌ | Go 模块根 |
| `install.sh` | ✅ 移 `tools/` | 仅被 raw URL 引用（README + 文档站） |
| `mkdocs.yml` | ❌ | 根目录是 mkdocs 社区默认，移动留长期心智负担 |
| `snapcraft.yaml` | ✅ 删除 | 上游 micro 遗产，从未产出过 snap 包，CI 未接，国内 Snap Store 不可用 |
