# C3 光标滚动 —— 最小修复

> 关联：调研依据见 `docs/C0-光标滚动-现状调研.md`。

---

## 1. 问题边界

| 场景 | 现状 | 本次范围 |
|------|------|---------|
| viewport 中间，↑↓ 移动光标 | ✅ 正常 | — |
| ↑ 键引发滚动 | ⚠️ 体验可接受但不精确（见 §6.3） | ✅ 修 |
| 翻页（PageUp/Down、HalfPage） | ✅ 正常（buffer 行级翻页，不依赖屏幕行精度） | — |
| click | ✅ 正常（走 `viewportRowBufLine`） | — |
| **↓ 键在 viewport 底边，不触发上滚** | ❌ **本次 bug** | ✅ 修 |

修法核心：让 `Relocate` 的边界判定从"按 line 算"改成"按 row 查"。不构建全文 row→line 数据。

---

## 2. 根因（精确到表达式）

`Relocate`（`internal/display/bufwindow.go:248-255`）的顶边/底边 margin 判定：

```go
c := w.SLocFromLoc(activeC.Loc)   // 非 softwrap = SLoc{cursor.Y, 0}

// 顶边
if c.LessThan(w.Scroll(w.StartLine, scrollmargin))                    // (1) 顶边阈值
    && c.GreaterThan(w.Scroll(bStart, scrollmargin-1)) {
    w.StartLine = w.Scroll(c, -scrollmargin)                          // 动作
    ret = true
} else if c.LessThan(w.StartLine) {
    w.StartLine = c
    ret = true
}
// 底边
if c.GreaterThan(w.Scroll(w.StartLine, height-1-scrollmargin))        // (4) 底边阈值
    && c.LessEqual(w.Scroll(bEnd, -scrollmargin)) {
    w.StartLine = w.Scroll(c, -height+1+scrollmargin)                 // 动作
    ret = true
} else if c.GreaterThan(w.Scroll(bEnd, -scrollmargin))
    && c.GreaterThan(w.Scroll(w.StartLine, height-1)) {
    w.StartLine = w.Scroll(bEnd, -height+1)
    ret = true
}
```

非 softwrap 下，`Scroll(s, n)` 就是 `SLoc{s.Line + n, 0}`。两个阈值：

| 阈值 | 1:1 算出的值 | 实际想要的值 |
|------|------------|------------|
| (1) 顶边 margin | `StartLine.Line + scrollmargin` | "屏幕第 `scrollmargin` 行显示的那个 buffer 行" |
| (4) 底边 margin | `StartLine.Line + height-1-scrollmargin` | "屏幕第 `height-1-scrollmargin` 行显示的那个 buffer 行" |

MD 装饰行打破 1:1 → 算出的 buffer 行**偏大**（装饰行吃掉屏幕行，真实位置对应的 buffer 行更小）。而"实际想要的值"，`viewportRowBufLine[]` **已经有了**——它记录视口每个屏幕行实际显示的 buffer 行。

## 3. 修法详解

### 3.1 改动位置：实质逻辑放 `bufwindow_md.go`，`Relocate` 只加分发缝

只有 MD 文件有此问题，其它文件走 micro 原生。实质 MD 逻辑收拢到 `bufwindow_md.go`，`Relocate` 只加一个 `if IsMD` 分发缝——与项目里**已有的** `LocFromVisual`（`bufwindow.go:289`）约定同构：

```go
// bufwindow.go:289 LocFromVisual —— 已有的同构先例
if w.Buf.IsMD && w.mdConfig.MDRender {
    ... = w.screenOffsetToBufferLine(...)   // 委托给 bufwindow_md.go
} else {
    ... = w.Scroll(...)                     // 原生逻辑，完全不动
}
```

> `Relocate` 的 248-260 行 `git blame` 全是 micro 原生作者（Zachary Yedidia 2019 / Dmitry Maluka 2021-2022），microNeo 至今未碰。本方案保持其判定结构和所有动作语句不变，仅把两个内联阈值提成变量 + 加 `if/else` 分发缝。

### 3.2 新增 helper（`bufwindow_md.go`，纯新增）

```go
// mdScrollMargins 返回 MD 模式下顶/底 margin 阈值的真实 buffer 行。
// viewportRowBufLine 已计入装饰行，比原生 1:1 计算更准。
// 查不到（短文件 / 视口未填满 / 极矮窗口）时回退原生 Scroll。
func (w *BufWindow) mdScrollMargins(scrollmargin, height int) (topMargin, botMargin SLoc) {
    topMargin = w.Scroll(w.StartLine, scrollmargin)           // 先按原生 1:1
    botMargin = w.Scroll(w.StartLine, height-1-scrollmargin)
    if scrollmargin >= height {                               // 极矮窗口：索引会为负，零成本防护
        return
    }
    if t := w.viewportRowBufLine[scrollmargin]; t >= 0 {      // -2 = 未填充 → 保留 fallback
        topMargin = SLoc{t, 0}
    }
    if t := w.viewportRowBufLine[height-1-scrollmargin]; t >= 0 {
        botMargin = SLoc{t, 0}
    }
    return
}
```

> `scrollmargin >= height` 同时保证 `[scrollmargin]` 和 `[height-1-scrollmargin]` 两个索引非负（后者 ≥ 0 等价于 `scrollmargin < height`）。

### 3.3 `Relocate` 的改动（diff 前缀：`+` 新增 / `~` 修改 / `·` 不变）

```go
+   // MicroNeo: MD 模式下查 viewportRowBufLine（实质逻辑见 bufwindow_md.go）
+   var topMargin, botMargin SLoc
+   if w.Buf.IsMD && w.mdConfig.MDRender {
+       topMargin, botMargin = w.mdScrollMargins(scrollmargin, height)
+   } else {
+       topMargin = w.Scroll(w.StartLine, scrollmargin)            // 搬运：原本内联在下条 if 条件里
+       botMargin = w.Scroll(w.StartLine, height-1-scrollmargin)   // 搬运：同上
+   }
~   if c.LessThan(topMargin) && c.GreaterThan(w.Scroll(bStart, scrollmargin-1)) {   // → topMargin
·       w.StartLine = w.Scroll(c, -scrollmargin)
·       ret = true
·   } else if c.LessThan(w.StartLine) {
·       w.StartLine = c
·       ret = true
·   }
~   if c.GreaterThan(botMargin) && c.LessEqual(w.Scroll(bEnd, -scrollmargin)) {     // → botMargin
·       w.StartLine = w.Scroll(c, -height+1+scrollmargin)
·       ret = true
·   } else if c.GreaterThan(w.Scroll(bEnd, -scrollmargin)) && c.GreaterThan(w.Scroll(w.StartLine, height-1)) {
·       w.StartLine = w.Scroll(bEnd, -height+1)

```

**改动量构成**：

| 类别 | 行数 | 内容 |
|------|------|------|
| **新增（实质）** | 3 行 | MD 分支缝（注释 + `var` 声明 + `if/else` 分发）|
| **搬运** | 2 行 | 两行 fallback 赋值——右侧 `w.Scroll(w.StartLine, ...)` 原本内联在上面的 `if` 条件里，现拆出来赋给变量 |
| **修改** | 2 行 | 两个主 `if` 的条件，各把内联表达式换成变量名 |
| **不变** | 8 行 | 所有动作语句（`w.StartLine = w.Scroll(c, ...)`）+ 两个 `else if` 兜底 |

## 6. 正确性论证

### 6.1 非 MD 路径零影响

`Scroll` 是纯函数（`internal/display/softwrap.go:292-296`）：非 softwrap 分支只读 `LinesNum()`、返回新 `SLoc`，无副作用。

```go
func (w *BufWindow) Scroll(s SLoc, n int) SLoc {
    if !w.Buf.Settings["softwrap"].(bool) {
        s.Line = util.Clamp(s.Line+n, 0, w.Buf.LinesNum()-1)
        return s
    }
    return w.scroll(s, n)
}
```

非 MD 文档：`w.Buf.IsMD == false` → `&&` 短路 → `mdScrollMargins` 不被调用、`viewportRowBufLine` 不被读取。else 分支的两行赋值，右侧表达式与原内联表达式相同、输入相同、`Scroll` 纯函数 → 变量值等于原内联表达式的值。后续判定和动作一字不动。

把纯函数从"内联在 if 条件"重排为"先赋值再比较"（`if f(x)` → `y := f(x); if y`），对纯函数严格等价。**非 MD 文档运行时行为 100% 一致。**

### 6.2 MD 路径的时序正确性

C0 §4.1 的时序坑（`viewportRowBufLine` 滞后于 `Relocate`）针对的是"查光标新位置"。本修法**不查光标**——查视口的阈值行：`viewportRowBufLine[scrollmargin]` 和 `viewportRowBufLine[height-1-scrollmargin]`。

按下键 / 上键时：视口内容（`StartLine`）**没变**，只有光标移动 → "视口里哪个屏幕行显示哪个 buffer 行"这个映射**完全没变** → 这两个阈值对本次 `Relocate` 判定是当前有效的。

**连续快速按 ↓ 键**：符合 micro 原生"先 MoveCursor 再 Relocate 最后 Display"流水线——第 N 帧 `Relocate` 触发滚动（`StartLine` 改变）后，外部 `Display()` 重建 `viewportRowBufLine` 为新视口，第 N+1 帧 `Relocate` 自然拿到新数据。本修法依赖的"旧视口数据 + 旧 `StartLine`"在每次 `Relocate` 判定期间的一致性，在该流水线下总成立。

### 6.3 顶边与底边对称修复的依据

底/顶边的 1:1 误差方向相反，但都是错的：

- **底边**：阈值偏大 → 条件**该触发时不触发** → 光标飞出视口（灾难，本次 bug）
- **顶边**：阈值偏大 → 条件**不该触发时触发了**（premature）→ 视口过早向上滚 → 光标被推离顶边比 scrollmargin 更远（轻度，用户难察觉，故体验"可接受"）

修法精确对称（顶/底边各加一个查表，同一个 `viewportRowBufLine` 数组、同一对索引），一并修让顶边滚动也精确，消除偶发的多余滚动。

### 6.4 两个辅助论证

- **动作不用改**：动作是 buffer 行级的滚动（`StartLine.Line = cursor.Y ± ...`）。把 `StartLine.Line` 设到正确的 buffer 行后，该行前置装饰行会随它一并正确显示（C0 §4.2 已论证）。滚动目标精度只需到 buffer 行级——顶/底边一致。
- **两个 `else if` 不动**：顶边 `c.LessThan(w.StartLine)` 和底边 `c.GreaterThan(w.Scroll(bEnd, -scrollmargin))` 是极端上下界兜底（光标已超出视口顶/底），与 MD 装饰无关，1:1 假设够用。
  - **一个已知的边界**：底边 `else if` 里的 `c.GreaterThan(w.Scroll(w.StartLine, height-1))` 同样基于 1:1 假设——MD 下若视口底部堆了大量装饰行，光标 buffer 行可能已超出 `StartLine.Line + height-1`，该条件算出的阈值偏小，理论上可能漏触发。但这是"光标已完全飞出视口底边"的极端兜底，触发概率极低，且触发后的动作是设到 buffer 末尾（`w.Scroll(bEnd, -height+1)`），行为可接受。本次不改，留待未来滚动场景发现可感知问题时再评估。

---

## 7. 结论

`Relocate` 的顶边/底边 margin 阈值在 MD 下从"按 line 算"改成"查 `viewportRowBufLine`"——实质逻辑放进 `bufwindow_md.go` 的 `mdScrollMargins` helper，`Relocate` 只加一个与 `LocFromVisual` 同构的 `if IsMD` 分发缝。两个阈值对称改、动作不动、实质新增 3 行——即可同时修复 ↓ 底边不滚 + ↑ 顶边偶发多余滚动，对非 MD 文档零影响，且不引入任何新数据结构或时序改动。
